  <action>
Add a `HEALTHCHECK` instruction to the `production` stage of the Dockerfile (per CONT-02). The `curl` binary is already installed in the base image.

Locate the production stage block (starts with `FROM base AS production`). Add the HEALTHCHECK instruction AFTER the `EXPOSE 3100` line and BEFORE the `ENTRYPOINT` line:

```dockerfile
HEALTHCHECK --interval=10s --timeout=5s --start-period=60s --retries=5 \
  CMD curl -sf http://localhost:3100/health || exit 1
```

Parameters rationale:
- `--interval=10s`: Poll every 10 seconds — frequent enough to detect outages quickly without excessive overhead
- `--timeout=5s`: HTTP request must complete within 5 seconds
- `--start-period=60s`: Grace period for server startup and database migration on first boot (migrations can take 10-30s; this prevents false unhealthy states during startup)
- `--retries=5`: Container is marked unhealthy only after 5 consecutive failures (50 seconds of failure), avoiding flaps
- `curl -sf`: `-s` silent (no progress), `-f` fail with non-zero exit on 4xx/5xx HTTP responses

The HEALTHCHECK must be placed only in the `production` stage (the final `FROM base AS production` block), NOT in the `base`, `deps`, or `build` stages. Adding it to earlier stages would have no effect and would be confusing.

Do not change any other line in the Dockerfile.
  </action>
