```
# =============================================================================
# Paperclip AI — Environment Variables
# =============================================================================
# Copy this file to .env in the same directory:
#   cp .env.template .env
# Fill in every required variable before running:
#   docker compose up -d
# This file should be committed. .env must NOT be committed (add to .gitignore).
# =============================================================================


# -----------------------------------------------------------------------------
# Authentication (REQUIRED)
# -----------------------------------------------------------------------------

# Secret key used by BetterAuth to sign sessions and tokens.
# SECURITY: If this is missing or wrong, anyone can forge session tokens.
# Generate with: openssl rand -hex 32
# Never reuse across environments. Never commit the actual value.
BETTER_AUTH_SECRET=


# -----------------------------------------------------------------------------
# PostgreSQL Credentials (REQUIRED)
# -----------------------------------------------------------------------------

# Database username. Used by both the db service (POSTGRES_USER) and
# the server's DATABASE_URL connection string.
POSTGRES_USER=paperclip

# Database password. Choose something strong for production.
# Generate with: openssl rand -hex 16
POSTGRES_PASSWORD=

# Database name. Typically matches the project name.
POSTGRES_DB=paperclip


# -----------------------------------------------------------------------------
# Application URL (REQUIRED)
# -----------------------------------------------------------------------------

# The exact URL users will type in their browser to access the dashboard.
# Must include scheme (http/https) and hostname. Include port if non-standard.
# CRITICAL: If this does not match the actual browser URL, BetterAuth login
# will silently fail or redirect to an unreachable origin.
#
# Examples:
#   http://192.168.1.100:3100     (LAN access by IP, before Traefik)
#   https://paperclip.example.com (after Phase 2 Traefik + SSL setup)
#
# For Phase 1 (no reverse proxy yet), use the host IP and port 3100.
PAPERCLIP_PUBLIC_URL=


# -----------------------------------------------------------------------------
# Host User Identity (REQUIRED for bind mount permissions)
# -----------------------------------------------------------------------------

# UID and GID of the user on the Docker host (docker-001) that owns the
# bind mount directories /mnt/paperclip/pgdata and /mnt/paperclip/data.
# The container entrypoint remaps the internal 'node' user to these values
# to avoid permission errors on NFS-backed storage.
#
# Find the correct values by running on docker-001:
#   id
# Look for uid=XXXX and gid=XXXX in the output.
#
# If USER_UID/USER_GID do not match the host owner, every container start
# will run 'chown -R' on /mnt/paperclip, which is slow on NFS volumes.
USER_UID=1000
USER_GID=1000


# -----------------------------------------------------------------------------
# AI Provider API Keys (OPTIONAL — needed for agent functionality in Phase 3)
# -----------------------------------------------------------------------------

# Anthropic API key — required for Claude Code adapter
# Get from: https://console.anthropic.com/settings/keys
ANTHROPIC_API_KEY=

# OpenAI API key — required for Codex adapter
# Get from: https://platform.openai.com/api-keys
OPENAI_API_KEY=

# GitHub personal access token — used by gh CLI inside agent containers
# Scopes needed: repo, read:org
# Get from: https://github.com/settings/tokens
GITHUB_TOKEN=
```

Important details about this file:
- Place it at `docker/.env.template` (not at project root — compose loads `.env` from same directory as compose file)
- Use blank values (not placeholder strings like `your-secret-here`) for secrets so operators get a clear error if they forget to fill in a value rather than accidentally using a placeholder
- `POSTGRES_USER` and `POSTGRES_DB` have sensible defaults (`paperclip`) already filled in since those are conventional
- `USER_UID=1000` and `USER_GID=1000` are filled in as defaults matching the Dockerfile ARG defaults, but operators MUST check actual values on docker-001
