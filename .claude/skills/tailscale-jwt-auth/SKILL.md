# Tailscale JWT Auth — Agent Authentication in Private Dev Mode

When Paperclip runs with `pnpm dev:tailscale` (or `pnpm dev --tailscale-auth`), the server switches from `local_trusted` to `authenticated` deployment mode. This means **every API request must carry valid credentials** — including requests from agents calling the Paperclip API via `pcurl` or direct `curl`.

## When to Use This Skill

- Agent runs fail with 401 or 403 errors when calling the Paperclip API
- `pcurl` returns authentication errors or empty responses
- Server logs show `local agent jwt secret missing or invalid; running without injected PAPERCLIP_API_KEY`
- Agent runs work with `pnpm dev` but break with `pnpm dev:tailscale`
- You need to understand or debug the auth chain for private-network dev
- `pcli comment` or other pcli write commands fail with HTTP 500 and the server log shows an FK constraint violation on `activity_log.run_id`
- Any `pcli` command returns 401 when running from a project directory that has `PAPERCLIP_AGENT_JWT_SECRET` configured

## The Auth Chain

There are four links in the chain. If any one breaks, agents cannot authenticate.

### Link 1: Deployment Mode

`pnpm dev:tailscale` sets these env vars via `scripts/dev-runner.mjs`:

```
PAPERCLIP_DEPLOYMENT_MODE=authenticated
PAPERCLIP_DEPLOYMENT_EXPOSURE=private
PAPERCLIP_AUTH_BASE_URL_MODE=auto
HOST=0.0.0.0
```

In `authenticated` mode, the auth middleware (`server/src/middleware/auth.ts`) no longer grants implicit board access. Every request must authenticate via:
- A session cookie (browser/UI), or
- A `Bearer` token in the `Authorization` header (agents)

**Contrast:** In `local_trusted` mode (plain `pnpm dev`), every request is implicitly treated as a board operator — no credentials needed.

### Link 2: JWT Secret

The server needs `PAPERCLIP_AGENT_JWT_SECRET` to mint and verify short-lived JWTs for agents. This secret is stored in the Paperclip env file at:

```
~/.paperclip/instances/<instance>/config/.env
```

The secret is a random 64-character hex string generated by the CLI during onboarding.

**How to check:**

```bash
pnpm paperclipai doctor
```

Look for the "Agent JWT secret" check. If it fails:

```bash
pnpm paperclipai doctor --repair
```

This creates the `.env` file with a fresh `PAPERCLIP_AGENT_JWT_SECRET` if missing.

**How it works internally:**
- `cli/src/config/env.ts` — `ensureAgentJwtSecret()` generates and persists the secret
- `cli/src/checks/agent-jwt-secret-check.ts` — doctor check that validates the secret exists
- `server/src/config.ts` — loads the `.env` file on startup (line 26-29)
- `server/src/agent-auth-jwt.ts` — `jwtConfig()` reads `PAPERCLIP_AGENT_JWT_SECRET` from `process.env`

### Link 3: JWT Minting and Injection

When the heartbeat service dispatches a run, it:

1. Checks if the adapter declares `supportsLocalAgentJwt: true` (in the server adapter registry)
2. Calls `createLocalAgentJwt(agentId, companyId, adapterType, runId)` to mint an HS256 JWT
3. Passes the JWT as `authToken` in `AdapterExecutionContext`
4. The adapter injects it as `PAPERCLIP_API_KEY` in the agent's environment

**Key code path:** `server/src/services/heartbeat.ts` (around line 2453):

```ts
const adapter = getServerAdapter(agent.adapterType);
const authToken = adapter.supportsLocalAgentJwt
  ? createLocalAgentJwt(agent.id, agent.companyId, agent.adapterType, run.id)
  : null;
```

The JWT contains:
- `sub` — agent ID
- `company_id` — company scope (enforced on verification)
- `adapter_type` — which adapter minted it
- `run_id` — the specific run
- `exp` — expiry (default 48 hours, configurable via `PAPERCLIP_AGENT_JWT_TTL_SECONDS`)

**If minting fails** (no secret configured), the server logs a warning and the agent runs without `PAPERCLIP_API_KEY`. This is the most common failure mode.

### Link 4: Token Usage by the Agent

The agent's environment receives `PAPERCLIP_API_KEY` containing the JWT. Two paths consume it:

**Direct curl/fetch:** The agent (or its tools) can use `Authorization: Bearer $PAPERCLIP_API_KEY` on any Paperclip API request.

**pcurl wrapper** (`scripts/context-mode/pcurl`): Automatically injects the auth header:

```bash
API_KEY="${PAPERCLIP_API_KEY:-pcli-local}"
AUTH_HEADERS=(-H "Authorization: Bearer $API_KEY")
```

In `local_trusted` mode, `pcurl` falls back to `pcli-local` as the token — which works because the server grants implicit access. In `authenticated` mode, this fallback fails with 401.

The auth middleware (`server/src/middleware/auth.ts`) verifies the token:
1. First tries to match the token hash against `agent_api_keys` table (static API keys)
2. If no match, tries `verifyLocalAgentJwt(token)` — validates signature, expiry, issuer, audience
3. If valid, loads the agent record and sets `req.actor` to `{ type: "agent", source: "agent_jwt" }`

## Diagnosis Flowchart

```
Agent API call fails with 401/403
│
├─ Is the server running with --tailscale-auth?
│  └─ No → This skill doesn't apply. In local_trusted mode, auth is implicit.
│
├─ Does `pnpm paperclipai doctor` pass the "Agent JWT secret" check?
│  └─ No → Run `pnpm paperclipai doctor --repair`, then restart the server.
│
├─ Does the server log show "local agent jwt secret missing"?
│  └─ Yes → The secret exists in the file but wasn't loaded into the server's
│           environment. Restart the server after running doctor --repair.
│
├─ Does the agent's env have PAPERCLIP_API_KEY set?
│  └─ No → Check adapter registry: does it set supportsLocalAgentJwt: true?
│          For oz_local, check server/src/adapters/registry.ts.
│
├─ Is the token a valid JWT (three dot-separated base64url segments)?
│  └─ No → The token might be "pcli-local" (the fallback). This means the
│          JWT was not injected. Check heartbeat logs for minting warnings.
│
└─ Is the JWT expired?
   └─ Yes → Default TTL is 48 hours. For long-running debug sessions,
            increase via PAPERCLIP_AGENT_JWT_TTL_SECONDS.
```

## Quick Fixes

### Fix 1: Missing JWT Secret (most common)

```bash
pnpm paperclipai doctor --repair
# Then restart the server:
# Kill the running pnpm dev:tailscale, then:
pnpm dev:tailscale
```

### Fix 2: Adapter Not Declaring JWT Support

In `server/src/adapters/registry.ts`, ensure your adapter entry includes:

```ts
supportsLocalAgentJwt: true,
```

Without this, the heartbeat service skips JWT minting entirely.

### Fix 3: Hostname Not Allowed

When accessing Paperclip from a Tailscale hostname, the server may reject requests with redirect errors if the hostname isn't registered:

```bash
pnpm paperclipai allowed-hostname <your-tailscale-hostname>
```

For example:

```bash
pnpm paperclipai allowed-hostname my-macbook.tailnet.ts.net
```

### Fix 4: pcurl Falling Back to pcli-local

If `pcurl` is using `pcli-local` instead of the JWT:
- Verify `PAPERCLIP_API_KEY` is in the agent's environment (check adapter `onMeta` logs)
- If the agent is oz-local, verify `execute.ts` injects `authToken` into both `PAPERCLIP_API_KEY` and `WARP_API_KEY`

## Environment Variables Reference

| Variable | Purpose | Set by |
|----------|---------|--------|
| `PAPERCLIP_DEPLOYMENT_MODE` | `local_trusted` or `authenticated` | dev-runner.mjs |
| `PAPERCLIP_DEPLOYMENT_EXPOSURE` | `private` or `public` | dev-runner.mjs |
| `PAPERCLIP_AGENT_JWT_SECRET` | HS256 signing secret for agent JWTs | CLI onboard/doctor |
| `PAPERCLIP_AGENT_JWT_TTL_SECONDS` | JWT lifetime (default: 172800 = 48h) | Manual override |
| `PAPERCLIP_AGENT_JWT_ISSUER` | JWT `iss` claim (default: `paperclip`) | Manual override |
| `PAPERCLIP_AGENT_JWT_AUDIENCE` | JWT `aud` claim (default: `paperclip-api`) | Manual override |
| `PAPERCLIP_API_KEY` | Bearer token injected into agent env | Heartbeat service |
| `PAPERCLIP_ALLOWED_HOSTNAMES` | Comma-separated allowed private hostnames | CLI or env |
| `HOST` | Server bind address (`0.0.0.0` for Tailscale) | dev-runner.mjs |
