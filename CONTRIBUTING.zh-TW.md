# 貢獻指南

[English](./CONTRIBUTING.md) | [简体中文](./CONTRIBUTING.zh-CN.md) | **繁體中文**

感謝您願意貢獻！

我們非常感謝小修復和深思熟慮的大改變。

## 让 Pull 请求被接受的两种途径

### 路徑 1：小而集中的變更（最快的合併方式）
- 選擇**一個**明確的事情來修復/改進
- 觸碰**盡可能少的檔案數量**
- 確保變更非常有針對性且易於審核
- 所有自動檢查均通過（包括 Greptile 註釋）
- 没有新的 lint/测试失败

當它們乾淨時，它們幾乎總是會很快合併。

### 路徑 2：更大或有影響力的改變
- **首先**在 Discord → #dev 頻道中談論它
  → 描述您想要解決的問題
  → 分享粗略的想法/方法
- 一旦達成粗略協議，就開始建造它
- 在您的公关中包括：
  - 之前/之後的螢幕截圖（如果使用者介面/行為發生變化，則為短影片）
  - 清晰描述内容和原因
  - 證明它有效（手動測試說明）
  - 所有測試均通過
  - 所有 Greptile + 其他PR評論均已解決

遵循這條路徑的 PR 被接受的可能性**大**，即使它們很大。

## 一般规则（两条路径）
- 编写清晰的提交消息
- 保持PR標題+描述有意義
- 一項 PR = 一項邏輯變更（除非是一個小的相關群體）
- 首先在本地運行測試
- 在討論中保持友善😄

有問題嗎？只需在#dev 中提問即可——我們很樂意提供協助。
