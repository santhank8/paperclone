## Paperclip 是什麼？

# 零人公司的開源編排

**如果 OpenClaw 是_僱員_，Paperclip 就是_公司_**

Paperclip 是 Node.js 伺服器和 React UI，用於協調 AI 智能體團隊來運作業務。自帶智能體、分配目標並透過一個控制台追蹤智能體的工作和成本。

它看起來像一個任務管理器，但在幕後它有組織架構圖、預算、治理、目標調整和智能體協調。

**管理業務目標，而不是拉取請求。 **

| |步驟|範例|
| ------ | --------------- | ------------------------------------------------------------------ |
| **01** |定義目標 | _“以 100 萬美元的 MRR 打造排名第一的人工智慧筆記應用程式。 ”_ |
| **02** |聘請團隊|執行長、技術長、工程師、設計師、行銷人員－任何機器人、任何提供者。 |
| **03** |批准並運行 |審查策略。設定預算。打吧。從控制台進行監控。  |

<br/>

> **即將推出：Clipmart** — 一鍵下載並運行整個公司。瀏覽預先建置的公司範本 - 完整的組織架構、智能體配置和技能 - 並在幾秒鐘內將它們匯入到您的 Paperclip 實例中。

## Paperclip 適合您，如果- ✅ 你想建立**自主人工智慧公司**
- ✅ 您**協調許多不同的智能體**（OpenClaw、Codex、Claude、Cursor）以實現共同目標
- ✅ 您有**20 個同時開啟的 Claude Code 終端機**並且無法追蹤每個人在做什麼
- ✅ 您希望智能體**24/7** 自主運行，但仍希望審核工作並在需要時介入
- ✅ 您想要**監控成本**並執行預算
- ✅ 您想要一個**感覺像是使用任務管理器**的智能體管理流程
- ✅ 您想要**透過手機**管理您的自主業務**

## 為什麼 Paperclip 很特別

Paperclip 正確處理硬編排細節。|                                   |                                                                                                               |
| --------------------------------- | ------------------------------------------------------------------------------------------------------------- |
| **原子執行。 ** |任務結帳和預算執行是原子的，因此不會重複工作，也不會出現失控的支出。                      |
| **持久智能體狀態。 ** |智能體透過心跳恢復相同的任務上下文，而不是從頭開始。                     |
| **運行時技能注入。 ** |智能體可以在運行時學習 Paperclip 工作流程和專案上下文，而無需重新培訓。                      |
| **回滾治理。 ** |強制執行批准門，修改配置更改，並且可以安全地回滾不良更改。        |
| **目標意識執行。 ** |任務具有完整的目標血統，因此智能體始終可以看到“原因”，而不僅僅是標題。                        |
| **便攜式公司模板。 ** |具有秘密清理和碰撞處理功能的匯出/匯入組織、智能體和技能。                          |
| **真正的多公司隔離。 ** |每個實體都是公司範圍的，因此一次部署可以運行許多具有單獨資料和審計追蹤的公司。 |

## Paperclip 不是什麼

|                              |                                                                                                                      |
| ---------------------------- | -------------------------------------------------------------------------------------------------------------------- |
| **不是聊天機器人。 ** |客服人員有工作，沒有聊天視窗。                                                                                  |
| **不是智能體框架。 ** |我們不會告訴您如何建立智能體。我們告訴您如何經營一家由他們組成的公司。                                |
| **不是工作流程建構器。 ** |沒有拖放管道。 Paperclip 透過組織架構圖、目標、預算和治理為公司建模。            |
| **不是提示管理員。 ** |智能體帶來自己的提示、模型和運行時。 Paperclip 管理他們工作的組織。
| **不是單一智能體工具。 ** |這是針對團隊的。如果您有智能體，則可能不需要 Paperclip。如果你有二十個──你肯定有。 |
| **不是程式碼審查工具。 ** | Paperclip 協調工作，而不是拉取請求。帶來您自己的審核流程。                                       |

## 快速入門

開源。自託管。無需 Paperclip 帳戶。

```bash
npx paperclipai onboard --yes
```

或手動：

```bash
git clone https://github.com/paperclipai/paperclip.git
cd paperclip
pnpm install
pnpm dev
```

這將啟動位於 `http://localhost:3100` 的 API 伺服器。嵌入式 PostgreSQL 資料庫會自動建立 - 無需設定。

> **要求：** Node.js 20+、pnpm 9.15+

## 常見問題解答

**典型的設定是什麼樣的？ **
在本地，單一 Node.js 進程管理嵌入式 Postgres 和本地檔案儲存。對於生產環境，將其指向您自己的 Postgres 並根據您的喜好進行部署。配置專案、智能體和目標－智能體負責剩下的事情。如果您是個人創業家，您可以使用 Tailscale 隨時隨地存取 Paperclip。然後您可以部署到例如Vercel 當你需要的時候。

**我可以經營多家公司嗎？ **
是的。一次部署可以運行無限數量的公司，並且資料完全隔離。

**Paperclip 與 OpenClaw 或 Claude Code 等智能體程式有何不同？ **
Paperclip_使用_那些智能體程式。它將他們編排成一家公司——有組織架構圖、預算、目標、治理和責任。

**為什麼我應該使用 Paperclip 而不是僅僅將我的 OpenClaw 指向 Asana 或 Trello？ **
智能體編排在如何協調誰簽出工作、如何維護會話、監控成本、建立治理方面具有微妙之處 - Paperclip 可以為您做到這一點。

（路線圖上有自備門票系統）

**智能體是否持續運作？ **
預設情況下，智能體程式根據計劃的偵測訊號和基於事件的觸發器（任務分配、@-提及）運行。您也可以掛接連續智能體，例如 OpenClaw。你帶上你的智能體和Paperclip座標。

## 發展

```bash
pnpm dev              # Full dev (API + UI, watch mode)
pnpm dev:once         # Full dev without file watching
pnpm dev:server       # Server only
pnpm build            # Build all
pnpm typecheck        # Type checking
pnpm test:run         # Run tests
pnpm db:generate      # Generate DB migration
pnpm db:migrate       # Apply migrations
```
