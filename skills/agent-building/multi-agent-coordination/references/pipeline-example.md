# Complete Example: 3-Agent Code Review Pipeline

## Architecture

```
Orchestrator (Sonnet)
  Reads PR diff → sets up coordination → spawns agents → aggregates results

  ├─► Scout (Haiku) — /tmp/agent-scout
  │     Input: list of changed files
  │     Output: coordination/result-scout.json (risk classifications)
  │
  ├─► Logic Reviewer (Sonnet) — /tmp/agent-logic
  │     Input: high-risk files from scout
  │     Output: coordination/result-logic.md
  │
  └─► Security Reviewer (Sonnet) — /tmp/agent-security
        Input: auth/api files from scout
        Output: coordination/result-security.md

Orchestrator reads all results → report.md
```

## Step 1: Orchestrator Setup

```bash
# Create coordination dir and worktrees
mkdir -p coordination
git worktree add /tmp/agent-scout -b agent/scout-$(date +%s)
git worktree add /tmp/agent-logic -b agent/logic-$(date +%s)
git worktree add /tmp/agent-security -b agent/security-$(date +%s)

# Get the diff
git diff main HEAD --name-only > coordination/changed-files.txt
```

## Step 2: Scout Agent Spawn Prompt

```markdown
You are running as claude-haiku-4-5-20251001. Your working directory is /tmp/agent-scout.

TASK: Classify changed files by review type.

Read: coordination/changed-files.txt

For each file, classify:
- "logic": business logic, data transformation, algorithms
- "security": auth, permissions, input validation, SQL queries, file handling
- "infra": config, CI, Dockerfile, dependencies
- "skip": tests, docs, generated files

RULES:
- Do NOT read file contents — classify by path and extension only
- Do NOT modify any files
- Return ONLY the JSON below and nothing else

Write to coordination/result-scout.json:
{
  "status": "done",
  "classifications": {
    "logic": ["src/api/users.ts", "src/lib/validation.ts"],
    "security": ["src/api/users.ts", "src/middleware/auth.ts"],
    "infra": [],
    "skip": ["tests/users.test.ts"]
  }
}
```

## Step 3: Logic Reviewer Spawn Prompt

```markdown
You are a code reviewer focused on correctness. Your working directory is /tmp/agent-logic.

TASK: Review the logic files listed in coordination/result-scout.json under "logic".

For each file:
1. Read the file
2. Identify bugs, N+1 queries, missing null checks, edge cases, logic errors
3. Do NOT review security issues (covered by security reviewer)
4. Do NOT modify files — this is a read-only review

RULES:
- Use LSP for symbol navigation (goToDefinition, findReferences)
- Do NOT modify any files
- Keep findings specific: include file path and line number

Write to coordination/result-logic.md:
## Logic Review

### Status: done

### Findings
- [file:line] description of issue

### Files Modified
none

### Blockers
none
```

## Step 4: Security Reviewer Spawn Prompt

```markdown
You are a security reviewer. Your working directory is /tmp/agent-security.

TASK: Review the security files listed in coordination/result-scout.json under "security".

Check for:
- Authentication bypass
- Authorization gaps (can user A access user B's data?)
- Input validation missing or insufficient
- SQL injection surfaces
- Path traversal in file operations
- Secrets or credentials in code

RULES:
- Use LSP for symbol navigation
- Do NOT modify any files — read-only review
- Reference OWASP Top 10 for classification

Write to coordination/result-security.md:
## Security Review

### Status: done

### Findings
- [OWASP-A01] [file:line] description

### Severity
- Critical: [count]
- High: [count]
- Medium: [count]

### Files Modified
none
```

## Step 5: Orchestrator — Spawn All Three

```python
# Pseudo-code for orchestrator

# Spawn scout first (need its output to scope reviewers)
Agent({
    prompt: SCOUT_PROMPT,
    subagent_type: "general-purpose",
    model: "haiku"
})

# Wait for scout
wait_for_status("coordination/status-scout.json", timeout=60)

# Spawn logic and security in parallel (background)
Agent({
    prompt: LOGIC_PROMPT,
    subagent_type: "general-purpose",
    run_in_background: True
})
Agent({
    prompt: SECURITY_PROMPT,
    subagent_type: "general-purpose",
    run_in_background: True
})

# Poll until both done
wait_for_all_done(["coordination/result-logic.md", "coordination/result-security.md"])
```

## Step 6: Aggregation

```python
def aggregate():
    scout = json.loads(Path("coordination/result-scout.json").read_text())
    logic = Path("coordination/result-logic.md").read_text()
    security = Path("coordination/result-security.md").read_text()

    report = f"""# Code Review Report

## Changed Files
{len(scout['classifications']['logic'])} logic, {len(scout['classifications']['security'])} security-sensitive

{logic}

---

{security}

---

## Summary
Generated by 3-agent pipeline: Haiku scout + Sonnet×2 reviewers
"""
    Path("report.md").write_text(report)
```

## Step 7: Cleanup

```bash
git worktree remove /tmp/agent-scout --force
git worktree remove /tmp/agent-logic --force
git worktree remove /tmp/agent-security --force
git branch -d agent/scout-* agent/logic-* agent/security-*
rm -rf coordination/
```

## Known Failure Modes

| Problem | Symptom | Fix |
|---------|---------|-----|
| Scout hasn't finished when reviewers spawn | Reviewers see empty result-scout.json | Always wait for scout before spawning reviewers |
| Reviewer writes to wrong worktree | Changes appear on wrong branch | Include worktree path explicitly in spawn prompt |
| Aggregation runs before all results exist | KeyError on missing result file | Check file existence before reading, not just status |
| Worktree branch already exists from prior run | `git worktree add` fails | Use timestamp suffix in branch names |
