# X Post Quality Gate

Review every draft before it ships. One bad post trains your audience to scroll past you.

## Review Checklist

Run each draft through these checks in order. Fix inline when possible, flag when judgment is needed.

### 1. Em Dash Ban

No em dashes. Period. Replace with colons, periods, commas, or restructure the sentence.

| Before | After |
|--------|-------|
| `Native > plugins — zero deps` | `Native > plugins. Zero deps.` |
| `SessionStart hook — fires on every launch` | `SessionStart hook: fires on every launch` |
| `Three tools — QMD, sync, recall` | `Three tools: QMD, sync, recall` |

### 2. AI Voice Detection

Kill these patterns. They scream "generated by AI":

- **Filler openers**: "Great question!", "I'd be happy to", "Absolutely!", "Here's the thing"
- **Hedge stacking**: "It's worth noting that", "It's important to understand"
- **Emoji spam**: More than 1 emoji per post
- **Buzzword soup**: "leverage", "utilize", "streamline", "empower", "unlock"
- **List formatting**: Numbered lists in tweets feel like LinkedIn (use line breaks or commas)
- **Exclamation inflation**: Max 1 per post, and only if genuinely earned

See `references/voice-rules.md` for the full pattern list and replacement suggestions.

### 3. Engagement Check

Every post needs at least one:

- **Hook**: First 10 words must make someone stop scrolling
- **Specificity**: Concrete numbers, tool names, or code snippets beat vague claims
- **Actionability**: Reader should be able to DO something after reading
- **Opinion**: Take a position. "It depends" gets zero engagement.

Weak: "Claude Code has some great features for memory management."
Strong: "Your CLAUDE.md is probably too long. Cut it to under 50 lines and move depth to references/."

See `references/engagement-patterns.md` for hook formulas and examples.

### 4. Character Count

- Single post: 280 max. If over, cut words, don't just truncate.
- Thread: First tweet hooks, each subsequent tweet stands alone. Number them (1/N).
- Replies: Can be longer (up to 280) but front-load the value. Don't bury the answer.

### 5. Promotion Filter

| Type | Rule |
|------|------|
| Value post (tip, insight, opinion) | No link required. This IS the content. |
| Reply to someone's problem | Link ONLY if we have a specific skill that solves it. Never generic site link. |
| Skill announcement | Link fine, but lead with what it does, not that it exists. |
| Thread | Link at most in the last tweet. Never in the first. |

If more than 1 in 5 posts contains a link, the ratio is wrong.

### 6. Tone Calibration

Read the draft out loud. Does it sound like a developer sharing what they learned at a meetup, or a brand account running a content calendar?

Target voice: knowledgeable peer, not authority figure. Casual but substantive. Would reply with "yeah, we hit that too" not "our solution addresses this need."

See `references/tone-examples.md` for before/after rewrites.

## Output Format

For each draft reviewed, output:

```
DRAFT: [original text]
STATUS: PASS | FAIL | NEEDS EDIT
ISSUES: [list of specific problems, if any]
FIXED: [rewritten version, if needed]
```

## Anti-Rationalization

| What you'll tell yourself | The truth |
|---|---|
| "It's close enough, nobody reads that carefully" | Your audience is developers. They read everything that carefully. |
| "One em dash won't matter" | It's a signal. One em dash means the whole post reads as AI-generated. |
| "Adding a link here is fine, it's relevant" | Is someone asking for help? If not, you're promoting, not helping. |
| "This opener is fine" | Read it without the first sentence. Better? Cut it. |
| "280 chars is too limiting for this point" | If you can't say it in 280, you don't understand it well enough. Compress. |
