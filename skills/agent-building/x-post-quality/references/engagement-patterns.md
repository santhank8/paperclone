# Engagement Patterns

## Hook Formulas That Work

### The Contrarian
State the opposite of conventional wisdom. Forces a reaction.
- "Your CLAUDE.md is probably too long."
- "Most Claude Code 'skills' are just prompts with a folder."
- "Stop mocking your database in tests."

### The Specific Number
Concrete beats abstract every time.
- "We cut our SKILL.md from 156 to 91 lines without losing any test score."
- "143K tokens before your first message. That's what an unaudited MCP stack costs."
- "After 34 sessions: zero memory saves. That's the real context problem."

### The "You're Doing It Wrong"
Identify a common mistake, state it bluntly, give the fix.
- "If you're using grep to find function definitions in Claude Code, stop. LSP goToDefinition is faster and has zero false positives."

### The Pattern Name
Give a recognizable name to something people experience but haven't labeled.
- "The write-batch anti-pattern: writing reference files one at a time wastes 9-29s per file."
- "Context amnesia: when your session knows less than your git history."

### The Before/After
Show the transformation in minimal words.
- "Before: 'any' type, works for now. After: unknown + type guard, works forever."

## Engagement Killers

- Starting with "I think" (weak, hedging)
- Starting with the tool name ("Claude Code can...")  - start with the problem
- Ending with "What do you think?" (lazy engagement bait)
- Tagging people who didn't ask to be tagged
- Quoting yourself
- "Thread incoming" without a hook

## Reply-Specific Patterns

When replying to someone's post:
1. Validate their point briefly (1 clause, not a paragraph)
2. Add something they didn't say
3. Be specific to THEIR context, not generic advice

Good: "Solid framework. Layer 5 is skills: repeatable workflows that survive across projects."
Bad: "Great post! Skills are really important for Claude Code workflows."
