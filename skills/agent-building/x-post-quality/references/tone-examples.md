# Tone Examples: Before and After

## Brand Account vs Developer Peer

### Example 1: Memory Feature

Brand: "Excited to announce our persistent memory skill! It helps developers maintain context across Claude Code sessions using native hooks and MEMORY.md files. Install today!"

Developer: "SessionStart hooks + MEMORY.md. That's it. Context survives /compact, /clear, and full session restarts. No plugins, no MCP overhead. Data stays in your repo."

### Example 2: Skill Announcement

Brand: "We've released 13 production-grade skills for Claude Code covering agent building, TDD, context management, and more. Check them out at aiskillslab.dev!"

Developer: "Built 13 Claude Code skills over the last month. The TDD one surprised me most: red-green-refactor in a PostToolUse hook loop. Claude fixes its own test failures without you touching the keyboard."

### Example 3: Tip Sharing

Brand: "Pro tip: Use anti-rationalization tables in your CLAUDE.md to prevent Claude from skipping important rules! This technique improves adherence significantly."

Developer: "Anti-rationalization tables in CLAUDE.md actually work. Column 1: what Claude tells itself ('this is too nuanced to delegate'). Column 2: the truth ('Sonnet handles 80% of implementation fine'). Catches the excuse before it happens."

### Example 4: Reply to a Question

Brand: "Great question! Our context-cost-management skill can help with that. It provides comprehensive strategies for managing your token budget efficiently."

Developer: "143K tokens before your first message. That's what an unaudited MCP stack costs. Run `claude --print-system-prompt | wc -c` to check yours. Most people are shocked."

## The Litmus Test

Read the draft out loud. Would you say this at a dev meetup while holding a beer? If not, rewrite.

Signs you nailed it:
- Someone would quote-tweet and add "this"
- It teaches one concrete thing
- You could defend it if challenged
- It sounds like something specific to YOUR experience, not generic advice anyone could generate

Signs you missed:
- You can imagine a different product name swapped in
- It uses "we" more than "you"
- The value is in the link, not the text
- It could appear on any brand's feed unchanged
