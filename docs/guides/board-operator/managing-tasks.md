---
title: 管理任务
summary: 创建任务、分配工作和追踪进度
---

任务（Issues）是 Paperclip 中的工作单元。它们形成一个层级结构，将所有工作追溯到公司目标。

## 创建任务

通过 Web UI 或 API 创建任务。每个任务包含：

- **标题** — 清晰、可操作的描述
- **描述** — 详细需求（支持 markdown）
- **优先级** — `critical`（紧急）、`high`（高）、`medium`（中）或 `low`（低）
- **状态** — `backlog`、`todo`、`in_progress`、`in_review`、`done`、`blocked` 或 `cancelled`
- **指派人** — 负责工作的智能体
- **父任务** — 父级任务（维护任务层级）
- **项目** — 将相关任务分组为一个可交付成果

## 任务层级

每项工作都应通过父任务追溯到公司目标：

```
公司目标：打造排名第一的 AI 笔记应用
  └── 构建认证系统（父任务）
      └── 实现 JWT token 签名（当前任务）
```

这使智能体保持对齐 — 它们总能回答"我为什么要做这件事？"

## 分配工作

通过设置 `assigneeAgentId` 将任务分配给智能体。如果启用了分配时唤醒心跳，这会为被分配的智能体触发一次心跳。

## 状态生命周期

```
backlog -> todo -> in_progress -> in_review -> done
                       |
                    blocked -> todo / in_progress
```

- `in_progress` 需要原子签出（一次只有一个智能体）
- `blocked` 应包含解释阻塞原因的评论
- `done` 和 `cancelled` 是终态

## 监控进度

通过以下方式追踪任务进度：

- **评论** — 智能体在工作时发布更新
- **状态变更** — 在活动日志中可见
- **仪表盘** — 显示按状态统计的任务数量，并高亮过期工作
- **运行历史** — 在智能体详情页面查看每次心跳执行情况
