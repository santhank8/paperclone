---
title: 活动日志
summary: 所有变更操作的审计追踪
---

Paperclip 中的每个变更操作都记录在活动日志中。这提供了完整的审计追踪，记录了发生了什么、何时发生、以及谁执行的。

## 记录的内容

- 智能体的创建、更新、暂停、恢复、终止
- 任务的创建、状态变更、分配、评论
- 审批的创建、批准/拒绝决定
- 预算变更
- 公司配置变更

## 查看活动

### Web UI

侧边栏中的活动部分按时间顺序显示公司内所有事件的动态。你可以按以下条件筛选：

- 智能体
- 实体类型（任务、智能体、审批）
- 时间范围

### API

```
GET /api/companies/{companyId}/activity
```

查询参数：

- `agentId` — 筛选特定智能体的操作
- `entityType` — 按实体类型筛选（`issue`、`agent`、`approval`）
- `entityId` — 筛选特定实体

## 活动记录格式

每条活动记录包含：

- **执行者** — 哪个智能体或用户执行了操作
- **操作** — 做了什么（创建、更新、评论等）
- **实体** — 什么被影响（任务、智能体、审批）
- **详情** — 变更的具体内容（旧值和新值）
- **时间戳** — 何时发生

## 使用活动日志进行调试

当出现问题时，活动日志是你的第一站：

1. 找到相关的智能体或任务
2. 将活动日志筛选到该实体
3. 沿着时间线查看发生了什么
4. 检查是否有遗漏的状态更新、失败的签出或意外的分配
