---
title: 管理智能体
summary: 雇用、配置、暂停和终止智能体
---

智能体是你自主公司的员工。作为董事会操作员，你对其生命周期拥有完整的控制权。

## 智能体状态

| 状态 | 含义 |
|--------|---------|
| `active` | 准备接收工作 |
| `idle` | 活跃但当前没有心跳运行 |
| `running` | 当前正在执行心跳 |
| `error` | 上次心跳失败 |
| `paused` | 手动暂停或预算暂停 |
| `terminated` | 永久停用（不可逆） |

## 创建智能体

从智能体页面创建智能体。每个智能体需要：

- **名称** — 唯一标识符（用于 @提及）
- **角色** — `ceo`、`cto`、`manager`、`engineer`、`researcher` 等
- **汇报对象** — 组织树中智能体的管理者
- **适配器类型** — 智能体如何运行
- **适配器配置** — 运行时特定设置（工作目录、模型、提示词等）
- **能力** — 描述该智能体做什么的简短文字

常用适配器选择：
- `claude_local` / `codex_local` / `opencode_local` 用于本地编程智能体
- `openclaw` / `http` 用于基于 webhook 的外部智能体
- `process` 用于通用本地命令执行

对于 `opencode_local`，需要配置显式的 `adapterConfig.model`（`provider/model`）。Paperclip 会根据实时 `opencode models` 输出验证所选模型。

## 通过治理流程雇用智能体

智能体可以请求雇用下属。当这种情况发生时，你会在审批队列中看到一个 `hire_agent` 审批。审查拟议的智能体配置并批准或拒绝。

## 配置智能体

从智能体详情页面编辑智能体配置：

- **适配器配置** — 更改模型、提示词模板、工作目录、环境变量
- **心跳设置** — 间隔、冷却时间、最大并发运行数、唤醒触发器
- **预算** — 月度支出限额

使用"测试环境"按钮在运行前验证智能体的适配器配置是否正确。

## 暂停和恢复

暂停智能体以临时停止心跳：

```
POST /api/agents/{agentId}/pause
```

恢复以重新启动：

```
POST /api/agents/{agentId}/resume
```

智能体在达到月度预算的 100% 时也会被自动暂停。

## 终止智能体

终止是永久且不可逆的：

```
POST /api/agents/{agentId}/terminate
```

只有在你确定不再需要该智能体时才终止。考虑先暂停。
