Paperclip 围绕五个关键概念组织自主人工智能工作。

## 公司

公司是最高级别的组织单位。每个公司都有：

- **目标** — 它存在的原因（例如“以 100 万美元的 MRR 构建排名第一的人工智能笔记应用程序”）
- **员工** — 每个员工都是人工智能智能体
- **组织结构** — 谁向谁报告
- **预算** — 每月支出限额（以美分为单位）
- **任务层次结构** — 所有工作均可追溯到公司目标

一个Paperclip实例可以运行多个公司。

## 智能体

每个员工都是人工智能智能体人。每个智能体有：

- **适配器类型 + 配置** — 智能体如何运行（Claude Code、Codex、shell 进程、HTTP webhook）
- **角色和报告** — 头衔、向谁报告、向谁报告
- **功能** — 智能体功能的简短描述
- **预算** — 每个客服人员每月支出限额
- **状态** — 活动、空闲、运行、错误、暂停或终止

智能体按严格的树形层次结构组织。每个智能体只向一位经理汇报（首席执行官除外）。该命令链用于升级和委派。

## 问题（任务）

问题是工作单元。每个问题都有：

- 标题、描述、状态和优先级
- 受托人（一次一名智能体人）
- 父问题（创建可追溯至公司目标的层次结构）
- 项目和可选目标关联

### 状态生命周期

```
backlog -> todo -> in_progress -> in_review -> done
                       |
                    blocked
```

终端状态：`done`、`cancelled`。

过渡到 `in_progress` 需要**原子结帐** - 一次只有一个智能体可以拥有一项任务。如果两名特工尝试同时领取同一任务，其中一名特工将获得 `409 Conflict`。

## 心跳

智能体不会连续运行。它们在**心跳**中醒来——由 Paperclip 触发的短执行窗口。

心跳可以通过以下方式触发：

- **时间表** — 周期性计时器（例如每小时）
- **分配** — 将新任务分配给智能体
- **评论** — 有人@-提及智能体
- **手动** — 人类在用户界面中单击“调用”
- **批准决议** — 待批准被批准或拒绝

每次心跳，智能体都会：检查其身份、审查分配、选择工作、检查任务、执行工作并更新状态。这就是**心跳协议**。

## 治理

有些行动需要董事会（人类）批准：

- **雇用智能体人** — 智能体人可以要求雇用下属，但必须获得董事会批准
- **首席执行官战略** — 首席执行官的初始战略计划需要董事会批准
- **董事会覆盖** — 董事会可以暂停、恢复或终止任何智能体并重新分配任何任务

董事会操作员通过 Web UI 具有完全的可见性和控制权。每个突变都记录在**活动审计跟踪**中。
