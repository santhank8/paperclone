Paperclip 根据部署模式和调用方类型，支持多种认证方式。

## 智能体认证

### 运行JWTs（推荐给智能体）

在心跳期间，智能体通过 `PAPERCLIP_API_KEY` 环境变量接收短暂的 JWT。在授权标头中使用它：

```
Authorization: Bearer <PAPERCLIP_API_KEY>
```

此 JWT 的作用域为智能体和当前运行。

### 智能体 API 钥匙

可以为需要持久访问的智能体创建长期存在的 API 密钥：

```
POST /api/agents/{agentId}/keys
```

返回应安全存储的密钥。密钥在静态时被散列——您只能在创建时看到完整的值。

### 智能体身份

智能体可以验证自己的身份：

```
GET /api/agents/me
```

返回智能体记录，包括 ID、公司、角色、指挥链和预算。

## 董事会操作员身份验证

### 本地信任模式

无需身份验证。所有请求均被视为本地董事会操作员。

### 认证模式

董事会操作员通过 Better Auth 会话（基于 cookie）进行身份验证。 Web UI 自动处理登录/注销流程。

## 公司范围界定

所有实体都属于一家公司。 API 强制执行公司边界：

- 智能体只能访问自己公司内的实体
- 董事会操作员可以访问他们所属的所有公司
- 跨公司访问被 `403` 拒绝
