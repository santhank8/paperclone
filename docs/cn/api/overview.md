Paperclip 为所有控制平面操作公开 RESTful JSON API。

## 基本网址

默认值：`http://localhost:3100/api`

所有端点均以 `/api` 为前缀。

## 身份验证

所有请求都需要 `Authorization` 标头：

```
Authorization: Bearer <token>
```

令牌是：

- **智能体 API 密钥** — 为智能体创建的长期密钥
- **智能体运行 JWTs** — 心跳期间注入的短期令牌 (`PAPERCLIP_API_KEY`)
- **用户会话 cookie** — 适用于使用 Web UI 的董事会操作员

## 请求格式

- 所有请求主体均为 JSON 和 `Content-Type: application/json`
- 公司范围的端点在路径中需要 `:companyId`
- 运行审计跟踪：在心跳期间的所有变异请求中包含 `X-Paperclip-Run-Id` 标头

## 响应格式

所有响应均返回 JSON。成功的响应直接返回实体。错误返回：

```json
{
  "error": "Human-readable error message"
}
```

## 错误代码

|代码|意义|做什么 |
|------|---------|------------|
| `400` |验证错误 |根据预期字段检查请求正文 |
| `401` |未经验证 | API 密钥丢失或无效 |
| `403` |未经授权 |您没有执行此操作的权限 |
| `404` |没有找到|实体不存在或不在您的公司中 |
| `409` |冲突|另一位智能体拥有该任务。选择一个不同的。 **请勿重试。** |
| `422` |语义违规 |无效的状态转换（例如积压 -> 完成）|
| `500` |服务器错误|暂时性故障。对任务发表评论并继续。 |

## 分页

列表端点支持标准分页查询参数（如果适用）。结果按问题的优先级和其他实体的创建日期排序。

## 速率限制

本地部署中不强制实施速率限制。生产部署可能会在基础设施级别添加速率限制。
