# 智能体配置和活动 UI

## 上下文

智能体人是Paperclip公司的员工。每个智能体都有一个适配器类型（`claude_local`、`codex_local`、`process`、`http`），用于确定其运行方式、组织架构图中的位置（向谁报告）、心跳策略（如何/何时唤醒）和预算。 `/agents` 的 UI 需要支持创建和配置智能体、查看其组织层次结构以及检查它们一直在做什么——它们的运行历史记录、实时日志和累积成本。

This spec covers three surfaces:

1. **智能体创建对话框**——“新智能体”流程
2. **智能体详细信息页面** -- 配置、活动和日志
3. **智能体列表页面** -- 对现有列表的改进

---

## 1. 智能体创建对话框

遵循现有的 `NewIssueDialog` / `NewProjectDialog` 模式：带有展开/最小化切换、公司徽章面包屑和 Cmd+Enter 提交的 `Dialog` 组件。

### 行为

- 提交时，调用 `agentsApi.create(companyId, data)`，其中 `data` 将顶层的身份字段和适配器特定字段打包到 `adapterConfig` 中，并将心跳/运行时打包到 `runtimeConfig` 中。
- 创建后，导航至新智能体的详细信息页面。
- 如果公司有零个智能体，则预填角色为 `ceo` 并禁用“报告至”。
- 当适配器类型更改时，适配器配置部分会更新其可见字段，保留任何共享字段值（cwd、promptTemplate 等）。

## 2. 智能体详细信息页面

重组现有的选项卡式布局。保留标题（名称、角色、头衔、状态徽章、操作按钮）并添加更丰富的选项卡。

### 标题

```
[StatusBadge]  Agent Name                    [Invoke] [Pause/Resume] [...]
               Role / Title
```

`[...]` 溢出菜单包含：终止、重置会话、创建 API 密钥。

### 标签

#### 概览选项卡

两栏布局：左栏是摘要卡，右栏是组织职位。

**摘要卡：**
- 适配器类型 + 型号（如果设置）
- 心跳间隔（例如“每 5 分钟”）或“禁用”
- 最后一次心跳时间（相对，例如“3 分钟前”）
- 会话状态：“活动（会话 abc123...）”或“无会话”
- 带进度条的当月支出/预算

**组织职位卡：**
- 报告对象：可点击的智能体名称（链接到其详细信息页面）
- 直接报告：向该智能体报告的智能体列表（可点击）

#### 配置选项卡

可编辑表单，具有与创建对话框相同的部分（适配器、运行时、心跳策略），但预先填充了当前值。使用内联编辑 - 单击要编辑的值，按 Enter 或模糊以通过 `agentsApi.update()` 保存。

部分：
- **身份**：姓名、头衔、角色、报告对象、能力
- **适配器配置**：当前适配器类型的所有适配器特定字段
- **心跳策略**：启用/禁用、间隔、唤醒触发、冷却
- **运行时**：上下文模式、预算、超时、宽限、环境变量、额外参数每个部分都是一张可折叠的卡片。保存是按字段进行的（模糊/输入时进行修补），而不是单个表单提交。验证错误显示为内联。

#### 运行选项卡

这是主要的活动/历史视图。显示心跳运行的分页列表，最新的在最前面。

**运行列表项：**
```
[StatusIcon] #run-id-short   source: timer     2 min ago     1.2k tokens   $0.03
             "Reviewed 3 PRs and filed 2 issues"
```

每行字段：
- 状态图标（绿色勾选 = 成功，红色 X = 失败，黄色旋转 = 正在运行，灰色时钟 = 已排队，橙色超时 = timed_out，斜线 = 已取消）
- 运行 ID（短，前 8 个字符）
- 调用源码芯片（定时器、分配、on_demand、自动化）
- 相对时间戳
- 代币使用总结（总输入+输出）
- 成本
- 结果摘要（结果或错误的第一行）

**单击运行**打开内联运行详细信息（手风琴展开）或滑动面板，显示：

- 带有时间戳的完整状态时间线（排队 -> 运行 -> 结果）
- 会议变更前/后
- 令牌细分：输入、输出、缓存输入
- 成本明细
- 错误消息和错误代码（如果失败）
- 退出代码和信号（如果适用）

**运行详细信息中的日志查看器**：
- 用于运行的流 `heartbeat_run_events`，由 `seq` 订购
- 每个事件呈现为带有时间戳、级别（颜色编码）和消息的日志行
- `stdout`/`stderr` 类型的事件以等宽字体显示
- 以独特的样式显示系统事件
- 对于正在运行的运行，通过 WebSocket 事件（`heartbeat.run.event`、`heartbeat.run.log`）实时自动滚动和附加
-“查看完整日志”链接从 `heartbeatsApi.log(runId)` 获取并显示在可滚动等宽容器中
- 截断：默认显示最后 200 个事件，“加载更多”按钮可获取较早的事件

#### 问题选项卡

保持原样：分配给该智能体的问题列表及其状态，可单击以导航到问题详细信息。

#### 成本选项卡

展开现有成本选项卡：

- **来自 `agent_runtime_state` 的累计总计**：总输入代币、总输出代币、总缓存代币、总成本
- **每月预算**进度条（当月支出与预算）
- **每次运行成本表**：日期、运行 ID、令牌输入/输出/缓存、成本 -- 可按日期或成本排序
- **图表**（拉伸）：过去 30 天内每日支出的简单条形图

### 属性面板（右侧边栏）

现有的 `AgentProperties` 面板继续显示快速浏览信息。添加：
- 会话 ID（已截断，带有复制按钮）
- 最后一个错误（如果有，以红色显示）
- 链接到“查看配置”（滚动到/切换到“配置”选项卡）

## 3. 智能体列表页面

### 当前状态

显示智能体的平面列表，其中包含状态徽章、姓名、角色、头衔和预算栏。

### 改进

**在标题中添加“新智能体”按钮**（加号图标 +“新智能体”），打开创建对话框。

**添加视图切换**：列表视图（当前）和组织架构图视图。**组织架构图视图：**
- 显示报告层次结构的树形布局
- 每个节点显示：座席姓名、角色、状态徽章
- 首席执行官在上，直接下属在下，等等。
- 使用已返回 `OrgNode[]` 的 `agentsApi.org(companyId)` 端点
- 单击节点导航至智能体详细信息

**列表视图改进：**
- 在每行添加适配器类型作为小芯片/标签
- 添加“最后活动”相对时间戳
- 如果智能体当前有运行心跳，则添加运行指示器（动画点）

**过滤：**
- 选项卡过滤器：全部、活动、暂停、错误（类似于问题页面模式）

## 5.API 表面

所有端点均已存在。 V1 不需要新的服务器工作。

|行动|端点|使用者 |
|--------|----------|---------|
|智能体列表 | `GET /companies/:id/agents` |列表页 |
|获取组织树 | `GET /companies/:id/org` |组织架构图视图 |
|创建智能体 | `POST /companies/:id/agents` |创建对话框 |
|更新智能体| `PATCH /agents/:id` |配置选项卡 |
|暂停/恢复/终止 | `POST /agents/:id/{action}` |标题操作 |
|重置会话 | `POST /agents/:id/runtime-state/reset-session` |溢出菜单|
|创建 API 密钥 | `POST /agents/:id/keys` |溢出菜单|
|获取运行时状态 | `GET /agents/:id/runtime-state` |概述选项卡、属性面板 |
|调用/唤醒| `POST /agents/:id/heartbeat/invoke` |标题调用按钮 |
|列表运行 | `GET /companies/:id/heartbeat-runs?agentId=X` |运行选项卡|
|取消运行 | `POST /heartbeat-runs/:id/cancel` |运行详情|
|举办活动| `GET /heartbeat-runs/:id/events` |日志查看器 |
|运行日志| `GET /heartbeat-runs/:id/log` |完整日志视图 |

2. **智能体列表改进** -- 添加新智能体按钮、选项卡过滤器、适配器芯片、运行指示器
3. **智能体详细信息：配置选项卡** -- 可编辑适配器/心跳/运行时配置
4. **智能体详细信息：运行选项卡** -- 包含状态、令牌、成本的运行历史记录列表
5. **智能体详细信息：运行详细信息 + 日志查看器** -- 可扩展的运行详细信息和流日志
6. **智能体详细信息：概览选项卡** -- 摘要卡、组织职位
7. **智能体详细信息：成本选项卡** -- 扩展成本细目
8. **组织架构图视图**——列表页树形可视化
9. **属性面板更新** -- 会话 ID、上次错误

步骤1-5是核心。步骤 6-9 是完善的。
