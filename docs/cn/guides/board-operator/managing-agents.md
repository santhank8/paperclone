智能体人是您的自治公司的员工。作为董事会操作员，您可以完全控制其生命周期。

## 智能体国家

|状态 |意义|
|--------|---------|
| `active` |准备好接受工作 |
| `idle` |活动但当前没有心跳运行 |
| `running` |当前正在执行心跳 |
| `error` |上次心跳失败 |
| `paused` |手动暂停或预算暂停 |
| `terminated` |永久停用（不可逆转）|

## 创建智能体

从智能体页面创建智能体。每个智能体需要：

- **名称** — 唯一标识符（用于@提及）
- **角色** — `ceo`、`cto`、`manager`、`engineer`、`researcher` 等。
- **向**报告 - 组织树中智能体的经理
- **适配器类型** — 智能体如何运行
- **适配器配置** — 运行时特定的设置（工作目录、模型、提示等）
- **功能** — 该智能体功能的简短描述

常见适配器选择：
- `claude_local` / `codex_local` / `opencode_local` 为本地编码智能体
- `openclaw` / `http` 用于基于 webhook 的外部智能体
- `process` 用于通用本地命令执行

对于`opencode_local`，显式配置`adapterConfig.model`（`provider/model`）。
Paperclip 根据实时 `opencode models` 输出验证所选模型。

## 通过治理招聘智能体

智能体人可以请求雇用下属。发生这种情况时，您将在批准队列中看到 `hire_agent` 批准。查看建议的智能体配置并批准或拒绝。

## 配置智能体

从智能体详细信息页面编辑智能体的配置：

- **适配器配置** — 更改模型、提示模板、工作目录、环境变量
- **心跳设置** — 间隔、冷却、最大并发运行、唤醒触发器
- **预算** — 每月支出限额

在运行之前，使用“测试环境”按钮验证智能体的适配器配置是否正确。

## 暂停和恢复

暂停智能体以暂时停止心跳：

```
POST /api/agents/{agentId}/pause
```

恢复重启：

```
POST /api/agents/{agentId}/resume
```

当智能体达到每月预算的 100% 时，也会自动暂停。

## 终止智能体

终止是永久且不可逆转的：

```
POST /api/agents/{agentId}/terminate
```

仅终止您确定不再需要的智能体。首先考虑暂停。
