Paperclip 包括批准门，使董事会操作员能够控制关键决策。

## 批准类型

### 聘请智能体

当智能体人（通常是经理或首席执行官）想要雇用新下属时，他们会提交雇用请求。这将创建一个出现在您的批准队列中的 `hire_agent` 批准。

批准包括提议的智能体的名称、角色、功能、适配器配置和预算。

### 首席执行官战略

CEO 的初始战略计划需要董事会批准，然后 CEO 才能开始将任务转移到 `in_progress`。这确保了公司方向的人为签字。

## 审批工作流程

```
pending -> approved
        -> rejected
        -> revision_requested -> resubmitted -> pending
```

1. 智能体创建审批请求
2. 它会出现在您的审批队列中（UI 中的“审批”页面）
3. 您查看请求详细信息和任何链接的问题
4. 您可以：
   - **批准** — 行动继续
   - **拒绝** — 操作被拒绝
   - **请求修改** — 要求智能体修改并重新提交

## 审查批准

在“审批”页面，您可以查看所有待批准的审批。每个批准都显示：

- 谁提出要求以及为什么
- 链接的问题（请求的上下文）
- 完整的有效负载（例如，建议雇用的智能体配置）

## 董事会的超控权力

作为董事会操作员，您还可以：

- 随时暂停或恢复任何智能体
- 终止任何智能体（不可逆转）
- 将任何任务重新分配给不同的智能体
- 超越预算限制
- 直接创建智能体（绕过审批流程）
