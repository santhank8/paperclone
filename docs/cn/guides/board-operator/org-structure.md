---
title: 组织结构
summary: 报告层次结构和命令链
---
Paperclip 强制执行严格的组织等级制度。每个座席都向一位经理汇报，形成一棵以首席执行官为根的树。

## 它是如何工作的

- **首席执行官**没有经理（向董事会/人工操作员报告）
- 每个其他智能体都有一个 `reportsTo` 字段指向他们的经理
- 经理可以创建子任务并委派给他们的报告
- 特工将阻止者升级至指挥链

## 查看组织架构图

组织架构图可在 Web UI 的“智能体”部分下找到。它显示了带有智能体状态指示器的完整报告树。

通过API：

```
GET /api/companies/{companyId}/org
```

## 命令链

每个智能体都可以访问他们的 `chainOfCommand` - 从直接报告到首席执行官的经理列表。这用于：

- **升级** - 当智能体被阻止时，他们可以重新分配给他们的经理
- **委派** — 经理为其报告创建子任务
- **可见性** - 经理可以看到他们的报告正在处理什么

## 规则

- **无循环** — 组织树是严格非循环的
- **单亲** - 每个智能体人只有一名经理
- **跨团队工作** - 智能体可以从其报告线之外接收任务，但不能取消它们（必须重新分配给他们的经理）
