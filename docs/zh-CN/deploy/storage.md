---
title: 块存储介质选项板 (Storage)
summary: 在本地小黑碟直接开落盘与全面正规化高攀挂上兼容 S3 级别大云存储之间的分水岭
---

就如何去存放由用户大爷在前台手动戳弄上传上去的众多花里胡哨文件垃圾（例如各种杂七杂八给打在问题 Issue 里跟做附件拖油瓶似的材料，各种画的大饼高清靓图等），Paperclip 特地为此开放了让底下用户去任选适配不同级别存储金主供应商 (storage provider) 的宽大回旋接口地带。

## 当穷折腾时直接甩入本地物理硬盘存落（此亦为出厂标配首选项）

不管一切三七二十一上传乱飞的数据，落红皆被无差别通通扫进并烂在如下这方小黑屋泥潭里头：

```
~/.paperclip/instances/default/data/storage
```

属于毫无大脑连想都不用去想就能跑通也绝不要你碰键盘劳神费心配来配去的。最配的上用来在你自家的狗窝里一个人把玩折腾当本地实验田练兵环境玩单机或是极其孤傲脱节的单体机小规模私服小打小闹。

## 真要干一场大事拥抱真正广交八方大容量可弹性横向延展与外部世界高度互联结通兼容 S3 级协议云大池 (S3-Compatible Storage)

要真是哪天业务起来被逼着要搞什么搞海量商用、企业生产投保或者大张旗鼓拉下脸面去弄那个横跨多台并集主机群联大集联挂 (multi-node deployments) 级别大仗势阵仗时，少啰嗦，二话不说快给我老老实实买那个老老实实遵守支持被各种第三方发烂支持遍的大路货工业标准 S3 级的各类对象云储服务包去（举个栗子比如：AWS S3 原厂，或者自己搞开源那挂用 MinIO 去内服，或者去蹭 Cloudflare R2 那家等）。

这块就得劳烦你大驾通过纯命令行控制端给上表叩配置入库登记入档才能见效了：

```sh
pnpm paperclipai configure --section storage
```

## 配置项配置指南指引明灯

| 用的是哪门子储物供应商流派 | 它们到底能扛下什么应用面场景的最优解 |
|----------|----------|
| `local_disk` | 大发慈悲给穷学生做本地过家家的试验田、小微局域的单兵级死撑硬磨勉强够用的机柜小应用 |
| `s3` | 用以震慑诸天神佛抗下亿万海量级的高清跨越各大节点无级跨度并发猛烈输出的正式超大生产用与云天边无垠之海 |

任何一次针对这一庞大脑库存储命根子大计配置的生杀大权的底层根数据都被悉数严格被锁并落在这套运行实例专为它老人家腾出的配置总台账老母本（config file）里面躺着封印记载定型了：

```
~/.paperclip/instances/default/config.json
```
