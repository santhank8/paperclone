为了捍卫资产静息安全（即休息静态下的存放安全）Paperclip 在执行落盘存储安全机制时，所有躺在磁盘底层里的各种敏感钥匙凭证全都强行被一把存镇在本地物理机器里的主控大母钥（master key）给上了锁了。那些真正挂载并关联在智能体配置项里的各类极度危险惹火的硬核配置（诸如重金开出来的 API 大客户 keys 通讯密钥或者是令牌 tokens 这些命根子）最终在这系统内部全都会无一例外地被以抽壳脱骨的形式置换阉割留成光秃秃的一条纯无意义不可溯的“加密后的引物指针引用 (encrypted secret references)”。

## 我们系统的御用常驻默认加密厂牌家：`local_encrypted` 协议区

凡是由它加密锁藏的资产机密，都会被拿一把名为本地主控制母钥的大锁结结实实地镇在如下的物理路境内：

```
~/.paperclip/instances/default/secrets/master.key
```

这把至高无上无所不能大开杀戒的关键物理钥匙串 (master key) 往往都是在您最初跑起系统跑配置初见开机上环境引导 (onboarding) 时就由这系统当您面上给一键盘子敲出来给造好的。而在此之后的一生里，它的尸首身躯将**绝不绝不 (never ever)** 会有半步能够踏出甚至游离过你这台开天辟地的原装宿主机！

## 高级定制配置

### CLI 操作面板

执行首次启动系统开机引导 (Onboarding) 向导就会直接顺手帮你铺设并生成一整套出厂自带配置好的缺省标准底盘 secrets 秘密配置文件出来：

```sh
pnpm paperclipai onboard
```

要是嫌不爽打算后期再去补敲换改重新定制这套加密门槛策略等级：

```sh
pnpm paperclipai configure --section secrets
```

觉得心里没底需要向系统底盘问诊以对证目前当下环境里这串机密配置设置组合这套家底有没有在不报错良好服役打转的话：

```sh
pnpm paperclipai doctor
```

### 硬塞超越一切的各种高级大后门环境变量指令 (Environment Overrides)

| 环境变量参数 | 描述说明 |
|----------|-------------|
| `PAPERCLIP_SECRETS_MASTER_KEY` | 32-byte 长明码大母钥匙本身 (支持用 base64, hex 甚至是干原味纯生 raw string 字符串生敲填塞进去) |
| `PAPERCLIP_SECRETS_MASTER_KEY_FILE` | 实在不想按剧本走，强行指定一处自定义且常人难觅其踪的极其隐蔽特工级变态存钥死角专属文件落盘指位路径 |
| `PAPERCLIP_SECRETS_STRICT_MODE` | 被你打上并狠狠设成为 `true` （真）时，那些想浑水摸鱼想把密令生填死配在明文里的臭毛病就将迎来血灾镇压全面禁止被扫除。 |

## 绝不留底退让的无条件全盘强打铁腕强抗模式 (Strict Mode)

试问要是系统在后台偷偷挂足马力并暗戳戳直接激活启用了这类强抗隔离的深渊级模式 (Strict mode)，那么任何试图想向环境变量那堆大筐里面再暗度陈仓妄图私藏些敏感配置字眼的话（但凡它的拼配名称匹配中了 `*_API_KEY`, `*_TOKEN`, `*_SECRET` 面相的敏感字眼的话），对不起，将**毫无通融余地直接被勒令只准以掩码打乱引用编号条目 (secret references) 的面貌出列**来乖乖站岗顶岗服役去；那种原始粗鄙地甚至直接大剌剌明目张胆地赤膊丢填塞其真实身份身价和裸奔敏感真字眼明值 (inline plain values) 上街的行为将被系统全数清零严词无情予以扫除封禁处刑了！

```sh
PAPERCLIP_SECRETS_STRICT_MODE=true
```

任何只要您胆敢将这系统抛头露面甚至半只脚跟从那老家土窝棚安分老实的本地受认证大环境 (`local trusted` mode) 给稍微往外网界稍微探足踏出伸半点去，我们就疯狂建议您**不容置疑立刻务必**顺带去开启这一绝对隔离防护伞。

## 升级迁移与抢救处理那些惨遭明文裸奔的老旧脏线源头 (Migrating Inline Secrets)

想当初你要是不幸手下还有大批之前早年不懂事胡搞乱建的老掉牙批次那些直接还在身上挂载穿配着全无掩饰大张旗鼓挂载满大街裸奔乱窜游荡着带裸明文写死 API key 传火的古旧呆板且极其落伍极不安全的弱智智能体们的话，快快使用下面这强行勒令的大刀指令在死线前把它们全部赶进加密安全黑箱庇护所去来个重洗上表：

```sh
pnpm secrets:migrate-inline-env         # (仅仅干打雷不落雨先来干跑验模拟一下走一遍给您展示展示)
pnpm secrets:migrate-inline-env --apply # (真正见红动刀强制一键上封加密处决覆写实装套换全盘大清洗的来真的动作)
```

## 在干员智能体的暗网资料档案表里所埋设的这些掩码暗号到底是个什么来头货色 (Secret References in Agent Config)

你大可翻阅抽查出随便一名干员他们本身所分发携携带背在背上拿去执行服役领命时被派发的运行环境变量清单，他们身上配置里的暗号引物配置 (secret references) 全长这副模样鬼样子：

```json
{
  "env": {
    "ANTHROPIC_API_KEY": {
      "type": "secret_ref",
      "secretId": "8f884973-c29b-44e4-8ea3-6413437f8081",
      "version": "latest"
    }
  }
}
```

只有当那个正在云端跳动大杀特杀的服务器主控系统眼巴巴看着这个被塞满暗号假表头可怜虫智能体这头都已经在前线端上桌子眼瞅着马上真要开始去执行那个要实打实耗银子的危险 API 调用时（at runtime 临时运行期），我们这个一直藏在这干员身后的大主控母系统幕后黑手此时在这个毫秒万分紧急死节骨眼上才会突然如同下凡般强行在暗中动用权限偷偷把那个真正的原始明文内容（被刚才那大母钥当场临时剥开大辟邪恶封壳的后）狠狠精准无误瞬填死塞入硬性猛灌输进这个可怜正在外延边界冲杀前线可怜的打工人那运行时的虚空孤立无援单独的进程独立物理环境变量容器内存里，给这孙子彻底真正打上最后一手满血鸡血药剂供弹！
