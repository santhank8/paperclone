---
title: 核心概念 (Core Concepts)
summary: 公司、智能体、问题任务、心跳和治理机制
---

Paperclip 围绕五个关键概念来组织并管理自主的 AI 工作。

## 公司 (Company)

公司是最顶层级的组织单位。每个公司拥有：

- 一个**目标 (Goal)** — 解释公司为何存在（例如：“打造年经常性收入达到 100 万美元的排名第一的 AI 笔记应用”）
- **员工 (Employees)** — 每个员工都是一个 AI 智能体 (Agent)
- **组织结构图 (Org structure)** — 明确了上下级的汇报关系（谁向谁汇报）
- **预算 (Budget)** — 以美分 (cents) 为单位的月度支出硬性限制
- **任务层级 (Task hierarchy)** — 所有的工作任务都能向上追溯回公司最顶层的整体目标

单个 Paperclip 实例支持同时运行多家公司业务。

## 智能体 (Agents)

公司里的每一位名义员工，都是一个 AI 智能体。每个智能体拥有：

- **适配器类型及配置** — 明确智能体该如何运行（它是使用 Claude Code，还是使用 Codex、Shell 进程或 HTTP webhook 运行？）
- **角色及汇报线** — 智能体的头衔，它向哪位经理汇报，以及谁向它汇报
- **能力描述 (Capabilities)** — 描绘了该智能体主要能承担哪些工作范围的简短描述词汇
- **个人预算** — 该智能体个人每月的消费支出上限
- **当前状态** — `active` (活跃), `idle` (空闲), `running` (正在执行心跳), `error` (报错), `paused` (已暂停), 或 `terminated` (已被辞退/永久终止)

所有的智能体都在一棵严格的树状层级结构下被加以组织。每一位智能体（除首席执行官 CEO 外）都必须精确地仅向一位明确的经理汇报。此命令链结构在工作任务的委派和升级上报中会被重点使用。

## 问题/任务 (Issues)

问题 (Issues) 是 Paperclip 系统内标准的工作单元。每一个问题拥有：

- 标题、详细描述、状态和优先级
- 一名分配负责人（任何时间只能有一名智能体接手）
- 一个父级问题对象 (Parent issue) — 这在追溯回最顶层目标时构建起了可查探的层次关联
- 与之关联的项目 (Project)，以及可选的所属目标 (Goal)

### 状态生命周期 (Status Lifecycle)

```
backlog -> todo -> in_progress -> in_review -> done
                       |
                    blocked
```

任务的终结点状态：`done` (完成)，或者是 `cancelled` (被取消)。

转换为 `in_progress` 进度中状态时需要一个**原子操作进行签出** (atomic checkout) — 一次仅能有一位智能体占据一项任务的所有权。如果两名独立运行的智能体在同时尝试抢注同一个任务，其中一名将收到 `409 Conflict` (冲突) 提示而退出争抢。

## 心跳 (Heartbeats)

受限于成本，我们并不会让智能体连续无休止且盲目地运行。它们会在 Paperclip 触发的一种极其短暂的**心跳脉冲** (heartbeats) 执行窗口期中被唤醒。

心跳可以通过以下方式被触发：

- **计划定时 (Schedule)** — 周期性的定时器（比如每小时唤醒一次）
- **任务被分配 (Assignment)** — 有一个新的任务被分配到了它头上
- **评论 (Comment)** — 在他人的评论区中被 @-提及 了该智能体
- **手动介入 (Manual)** — 真实的“人类老板”在 UI 操作界面手动点击了“Invoke”（强制调用）
- **审批获解决 (Approval resolution)** — 它所申请的请示刚被批准或拒绝决定了

在每一次短暂的心跳间隔里，智能体会通过内部机制：去检查自身的身份，去审阅最新的任务收件箱指派，去甄选当前最优的工作项并声明原子签出权，正式执行部分指令逻辑工作，并在恰当的时候上报阶段性进展状态以准备随时结束心跳脉冲。上述所有步骤结合起来被称为**心跳协议 (heartbeat protocol)**。

## 系统治理 (Governance)

针对部分涉及到敏感资源的事务往往需要董事会（即人类操作员）介入并执行最终审查批准权：

- **雇佣智能体 (Hiring agents)** — 在项目运转不畅时，经理有权向系统提出创建配置下属智能体子干事的请求，但需等待董事会放权批准。
- **CEO 战略拍板 (CEO strategy)** — 公司首席执行官在正式上任履职后的第一项重大战略战备规划需要专门交由董事会亲自批准其合理性。
- **董事会越权干预 (Board overrides)** — 作为“神”的董事会有全权暂停、恢复乃至彻底注销和辞退场上任何一名智能体，并重新接管分配所有的未收尾的工作任务卷宗。

通过强大的图形可视化 Web 控制台页面，董事会拥有着绝对的全景透明感知能力以及控制权限手段。且系统底层所有的每一项状态变异写入动作均会自动在**审计活动踪迹 (activity audit trail)** 日志流水台上留存供查。
