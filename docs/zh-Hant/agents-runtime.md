# 智能體運行時指南

狀態：面向使用者的指南
最後更新: 2026-02-17
受眾：在 Paperclip 中設定和運行智能體的營運商

## 1. 這個系統的作用

Paperclip 中的智能體程式不會連續運作。
它們以**心跳**的方式運作：由喚醒觸發的短執行視窗。

每一次心跳：

1. 啟動設定的智能體適配器（例如，Claude CLI 或 Codex CLI）
2. 給予當前提示/上下文
3. 讓它一直工作，直到退出、超時或被取消
4. 儲存結果（狀態、令牌使用、錯誤、日誌）
5. 即時更新UI

## 2. 當智能體醒來時

可以透過四種方式喚醒智能體：

- `timer`：預定間隔（例如每5分鐘）
- `assignment`：當工作被指派/簽出給該智能體時
- `on_demand`：手動喚醒（按鈕/API）
- `automation`：系統觸發的喚醒以實現未來的自動化

如果智能體程式已在運行，則會合併新的喚醒，而不是啟動重複的運行。

## 3. 每個智能體要配置什麼

## 3.1 適配器選擇

常見選擇：

- `claude_local`：運行本地`claude` CLI
- `codex_local`：運行本地`codex` CLI
- `process`：通用 shell 指令適配器
- `http`：呼叫外部HTTP端點

對於 `claude_local` 和 `codex_local`，Paperclip 假定 CLI 已在主機上安裝並經過驗證。

## 3.2 運行時行為

在智能體程式運行時設定中，配置心跳策略：

- `enabled`：允許預定的心跳
- `intervalSec`：計時器間隔（0 = 停用）
- `wakeOnAssignment`：分配工作時喚醒
- `wakeOnOnDemand`：允許 ping 式按需喚醒
- `wakeOnAutomation`：允許系統自動喚醒

## 3.3 工作目錄與執行限制

對於本地適配器，設定：

- `cwd`（工作目錄）
- `timeoutSec`（每個心跳的最大運行時間）
- `graceSec`（超時/取消後強行殺死前的時間）
- 可選的環境變數和額外的 CLI 參數
- 在儲存之前在智能體程式配置中使用**測試環境**來執行適配器特定的診斷

## 3.4 提示模板

您可以設定：

- `promptTemplate`：用於每次運行（首次運行和恢復會話）

模板支援 `{{agent.id}}`、`{{agent.name}}` 等變數和運行上下文值。

## 4. 會話恢復行為

Paperclip 儲存可恢復適配器的會話 ID。

- 下一個心跳會自動重複使用已儲存的會話。
- 這提供了心跳的連續性。
- 如果上下文變得陳舊或混亂，您可以重置會話。

在以下情況下使用會話重設：

- 你顯著改變了提示策略
- 智能體陷入了惡性循環
- 你想要一個乾淨的重啟

## 5. 日誌、狀態和運行歷史記錄對於每次心跳運行，您都會得到：

- 運作狀態（`queued`、`running`、`succeeded`、`failed`、`timed_out`、`cancelled`）
- 錯誤文字和 stderr/stdout 摘錄
- 適配器可用時的令牌使用/成本
- 完整日誌（儲存在核心運行行之外，針對大輸出進行了最佳化）

在本機/開發設定中，完整日誌儲存在磁碟上配置的運行日誌路徑下。

## 6. UI 中的即時更新

Paperclip 將運行時/活動更新即時推送到瀏覽器。

您應該會看到以下內容的即時變更：

- 智能體狀態
- 心跳運行狀態
- 智能體工作引起的任務/活動更新
- 相關的控制台/成本/活動面板

如果連線斷開，UI 會自動重新連線。

## 7. 常見操作模式

## 7.1 簡單的自治循環

1. 啟用定時器喚醒（例如每300秒）
2. 保持分配喚醒開啟
3. 使用有重點的提示模板
4. 觀察運行日誌並隨時間調整提示/配置

## 7.2 事件驅動循環（較少持續輪詢）

1. 停用定時器或設定較長的間隔
2. 保持分配喚醒啟用
3. 使用按需喚醒進行手動微移

## 7.3 安全第一循環

1、超時時間短
2. 保守提示
3. 監控錯誤並在需要時快速取消
4. 出現漂移時重置會話

## 8. 故障排除

如果運行多次失敗：

1. 檢查適配器指令可用性（`claude`/`codex` 安裝並登入）。
2. 驗證 `cwd` 是否存在且可存取。
3. 檢查運行錯誤 + stderr 摘錄，然後檢查完整日誌。
4. 確認超時時間是否太低。
5. 重置會話並重試。
6. 如果智能體程式導致重複出現錯誤更新，請暫停智能體程式。

典型故障原因：

- CLI 未安裝/驗證
- 錯誤的工作目錄
- 適配器參數/env 格式錯誤
- 提示過於寬泛或缺少約束
- 處理超時

Claude具體說明：

- 如果在適配器環境或主機環境中設定了 `ANTHROPIC_API_KEY`，則 Claude 使用 API-key 驗證而不是訂閱登入。 Paperclip 在環境測試中將其視為警告，而不是硬錯誤。

## 9. 安全與風險說明

本機 CLI 轉接器在主機上以非沙盒方式運作。

這意味著：

- 提示指示很重要
- 配置的憑證/環境變數是敏感的
- 工作目錄權限很重要

盡可能從最低權限開始，並避免在廣泛的可重複使用提示中暴露秘密，除非故意需要。

## 10. 最小設定清單

1. 選擇適配器（`claude_local` 或 `codex_local`）。
2. 將`cwd` 設定為目標工作空間。
3. 新增bootstrap+普通提示範本。
4. 配置心跳策略（計時器和/或分配喚醒）。
5. 觸發手動喚醒。
6. 確認運行成功並記錄會話/令牌使用情況。
7. 觀看即時更新並迭代提示/配置。
