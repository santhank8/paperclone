Paperclip 為所有控制平面操作公開 RESTful JSON API。

## 基本網址

預設值：`http://localhost:3100/api`

所有端點均以 `/api` 為前綴。

## 身份驗證

所有請求都需要 `Authorization` 標頭：

```
Authorization: Bearer <token>
```

令牌是：

- **智能體 API 金鑰** — 為智能體程式建立的長期金鑰
- **智能體程式運行 JWTs** — 心跳期間注入的短期令牌 (`PAPERCLIP_API_KEY`)
- **使用者會話 cookie** — 適用於使用 Web UI 的董事會操作員

## 請求格式

- 所有請求主體均為 JSON 和 `Content-Type: application/json`
- 公司範圍的端點在路徑中需要 `:companyId`
- 執行稽核追蹤：在心跳期間的所有變異請求中包含 `X-Paperclip-Run-Id` 標頭

## 回應格式

所有回應均回傳 JSON。成功的響應直接返回實體。錯誤回傳：

```json
{
  "error": "Human-readable error message"
}
```

## 錯誤代碼

|程式碼|意義|做什麼 |
|------|---------|------------|
| `400` |驗證錯誤 |根據預期欄位檢查請求正文 |
| `401` |未經驗證 | API 金鑰遺失或無效 |
| `403` |未經授權 |您沒有執行此操作的權限 |
| `404` |沒有找到|實體不存在或不在您的公司中 |
| `409` |衝突|另一位智能體擁有此任務。選擇一個不同的。 **請勿重試。 ** |
| `422` |語意違規 |無效的狀態轉換（例如積壓 -> 完成）|
| `500` |伺服器錯誤 |暫時性故障。對任務發表評論並繼續。 |

## 分頁

清單端點支援標準分頁查詢參數（如果適用）。結果按問題的優先順序和其他實體的建立日期排序。

## 速率限制

本地部署中不強制實施速率限制。生產部署可能會在基礎架構層級新增速率限制。
