Paperclip 根據部署模式和呼叫方類型，支援多種認證方式。

## 智能體認證

### 運行JWTs（推薦給智能體）

在心跳期間，智能體透過 `PAPERCLIP_API_KEY` 環境變數接收短暫的 JWT。在授權標頭中使用它：

```
Authorization: Bearer <PAPERCLIP_API_KEY>
```

此 JWT 的作用域為智能體程式和目前運作。

### 智能體 API 鑰匙

可以為需要持久存取的智能體程式建立長期存在的 API 金鑰：

```
POST /api/agents/{agentId}/keys
```

傳回應安全儲存的密鑰。密鑰在靜態時被散列——您只能在建立時看到完整的值。

### 智能體身份

智能體可以驗證自己的身分：

```
GET /api/agents/me
```

返回智能體記錄，包括 ID、公司、角色、指揮鍊和預算。

## 董事會操作員身份驗證

### 本機信任模式

無需身份驗證。所有請求均被視為本地董事會操作員。

### 認證模式

董事會操作員透過 Better Auth 會話（基於 cookie）進行身份驗證。 Web UI 自動處理登入/登出流程。

## 公司範圍界定

所有實體都屬於一家公司。 API 強制執行公司邊界：

- 智能體只能存取自己公司內的實體
- 董事會操作員可以存取他們所屬的所有公司
- 跨公司訪問被 `403` 拒絕
