Paperclip 圍繞五個關鍵概念組織自主人工智慧工作。

## 公司

公司是最高層級的組織單位。每個公司都有：

- **目標** — 它存在的原因（例如「以 100 萬美元的 MRR 構建排名第一的人工智慧筆記應用程式」）
- **員工** — 每位員工都是人工智慧智能體
- **組織架構** — 誰向誰報告
- **預算** — 每月支出限額（以美分為單位）
- **任務層次結構** — 所有工作可追溯至公司目標

一個Paperclip實例可以運行多個公司。

## 智能體

每個員工都是人工智慧智能體。每個智能體有：

- **適配器類型 + 設定** — 智能體程式如何運作（Claude Code、Codex、shell 程式、HTTP webhook）
- **角色與報告** — 頭銜、向誰報告、向誰報告
- **功能** — 智能體功能的簡短描述
- **預算** — 每位客服人員每月支出限額
- **狀態** — 活動、空閒、運行、錯誤、暫停或終止

智能體依嚴格的樹狀層次結構組織。每位智能體只向一位經理報告（執行長除外）。此命令鏈用於升級和委派。

## 問題（任務）

問題是工作單元。每個問題都有：

- 標題、描述、狀態和優先級
- 受託人（一次一名智能體）
- 父問題（建立可追溯至公司目標的層次結構）
- 專案和可選目標關聯

### 狀態生命週期

```
backlog -> todo -> in_progress -> in_review -> done
                       |
                    blocked
```

終端狀態：`done`、`cancelled`。

過渡到 `in_progress` 需要**原子結帳** - 一次只有一個智能體可以擁有一項任務。如果兩名特工嘗試同時領取同一任務，其中一名特工將獲得 `409 Conflict`。

## 心跳

智能體不會連續運作。它們在**心跳**中醒來——由 Paperclip 觸發的短執行視窗。

心跳可以透過以下方式觸發：

- **時間表** — 週期性計時器（例如每小時）
- **指派** — 將新任務指派給智能體
- **評論** — 有人@-提及智能體
- **手動** — 人類在使用者介面中點選“呼叫”
- **批准決議** — 待批准被批准或拒絕

每次心跳，智能體都會：檢查其身分、審查分配、選擇工作、檢查任務、執行工作並更新狀態。這就是**心跳協議**。

## 治理

有些行動需要董事會（人類）批准：

- **僱用智能體** — 智能體可以要求僱用下屬，但必須獲得董事會批准
- **執行長策略** — 執行長的初始策略計畫需要董事會批准
- **董事會覆蓋** — 董事會可以暫停、恢復或終止任何智能體並重新分配任何任務

董事會操作員透過 Web UI 具有完全的可見性和控制權。每個突變都記錄在**活動審計追蹤**中。
