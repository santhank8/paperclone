# 智能體程式設定和活動 UI

## 上下文

智能體是Paperclip公司的員工。每個智能體程式都有一個適配器類型（`claude_local`、`codex_local`、`process`、`http`），用於確定其運作方式、組織架構圖中的位置（向誰報告）、心跳策略（如何/喚醒）和預算策略（如何/喚醒）。 `/agents` 的 UI 需要支援建立和配置智能體程式、查看其組織層次結構以及檢查它們一直在做什麼——它們的運行歷史記錄、即時日誌和累積成本。

此規格涵蓋三個表面：

1. **智能體程式建立對話框**—「新智能體」流程
2. **智能體詳細資料頁面** -- 配置、活動和日誌
3. **智能體清單頁** -- 對現有清單的改進

---

### 行為

- 提交时，调用 `agentsApi.create(companyId, data)`，其中 `data` 将顶层的身份字段和适配器特定字段打包到 `adapterConfig` 中，并将心跳/运行时打包到 `runtimeConfig` 中。
- 建立後，導覽至新智能體的詳細資訊頁面。
- 如果公司有零個智能體，則預填角色為 `ceo` 並停用「報告至」。
- 當適配器類型變更時，適配器配置部分會更新其可見字段，保留任何共用字段值（cwd、promptTemplate 等）。

## 2. 智能體詳細資料頁面

重組現有的選項卡式佈局。保留標題（名稱、角色、頭銜、狀態徽章、操作按鈕）並添加更豐富的選項卡。

### 標題

```
[StatusBadge]  Agent Name                    [Invoke] [Pause/Resume] [...]
               Role / Title
```

`[...]` 溢位選單包含：終止、重設會話、建立 API 金鑰。

### 標籤

#### 概覽標籤

兩欄佈局：左欄是摘要卡，右欄是組織職位。

**摘要卡：**
- 適配器類型 + 型號（如果設定）
- 心跳間隔（例如“每 5 分鐘”）或“停用”
- 最後一次心跳時間（相對，例如「3 分鐘前」）
- 會話狀態：“活動（會話 abc123...）”或“無會話”
- 帶進度條的當月支出/預算

**組織職位卡：**
- 報告對象：可點選的智能體名稱（連結至其詳細資料頁面）
- 直接報告：向該智能體報告的智能體清單（可點擊）

#### 設定選項卡

可編輯表單，具有與建立對話方塊相同的部分（適配器、執行時間、心跳策略），但預先填入了目前值。使用內聯編輯 - 按一下要編輯的值，按 Enter 或模糊以透過 `agentsApi.update()` 儲存。

部分：
- **身分**：姓名、頭銜、角色、報告對象、能力
- **適配器配置**：當前適配器類型的所有適配器特定字段
- **心跳策略**：啟用/停用、間隔、喚醒觸發、冷卻
- **運行時**：情境模式、預算、逾時、寬限、環境變數、額外參數每個部分都是一張可折疊的卡片。保存是按欄位進行的（模糊/輸入時進行修補），而不是單一表單提交。驗證錯誤顯示為內聯。

#### 運行選項卡

這是主要的活動/歷史視圖。顯示心跳運行的分頁列表，最新的在最前面。

**運行清單項目：**
```
[StatusIcon] #run-id-short   source: timer     2 min ago     1.2k tokens   $0.03
             "Reviewed 3 PRs and filed 2 issues"
```

每行欄位：
- 狀態圖示（綠色勾選 = 成功，紅色 X = 失敗，黃色旋轉 = 正在運行，灰色時鐘 = 已排隊，橘色超時 = timed_out，斜線 = 已取消）
- 運行 ID（短，前 8 個字元）
- 呼叫原始碼晶片（計時器、分配、on_demand、自動化）
- 相對時間戳
- 代幣使用總結（總輸入+輸出）
- 成本
- 結果摘要（結果或錯誤的第一行）

**按一下執行**開啟內嵌運作詳細資訊（手風琴展開）或滑動面板，顯示：

- 帶有時間戳的完整狀態時間軸（排隊 -> 運行 -> 結果）
- 會議之前/之後
- 令牌細分：輸入、輸出、快取輸入
- 成本明細
- 錯誤訊息和錯誤代碼（如果失敗）
- 退出代碼和訊號（如果適用）

**運行詳細資訊中的日誌檢視器**：
- 用於運行的流 `heartbeat_run_events`，由 `seq` 訂購
- 每個事件呈現為帶有時間戳記、等級（顏色編碼）和訊息的日誌行
- `stdout`/`stderr` 類型的事件以等寬字體顯示
- 以獨特的樣式顯示系統事件
- 對於正在運行的運行，透過 WebSocket 事件（`heartbeat.run.event`、`heartbeat.run.log`）即時自動捲動和附加
-「查看完整日誌」連結從 `heartbeatsApi.log(runId)` 取得並顯示在可捲動等寬容器中
- 截斷：預設顯示最後 200 個事件，「載入更多」按鈕可取得較早的事件

#### 問題選項卡

保持原樣：指派給該智能體程式的問題清單及其狀態，可按一下以導覽至問題詳細資料。

#### 成本選項卡

展開現有成本標籤：

- **來自 `agent_runtime_state` 的累計總計**：總輸入代幣、總輸出代幣、總緩存代幣、總成本
- **每月預算**進度條（當月支出與預算）
- **每次運行成本表**：日期、運行 ID、令牌輸入/輸出/快取、成本 -- 可依日期或成本排序
- **圖表**（拉伸）：過去 30 天內每日支出的簡單條形圖

### 屬性面板（右側邊欄）

現有的 `AgentProperties` 面板繼續顯示快速瀏覽資訊。添加：
- 會話 ID（已截斷，複製按鈕）
- 最後一個錯誤（如果有，以紅色顯示）
- 連結至「檢視配置」（捲動至/切換至「設定」標籤）

## 3. 智能體清單頁面

### 目前狀態

顯示智能體程式的平面列表，其中包含狀態徽章、姓名、角色、頭銜和預算欄。

### 改進

**在標題中新增「新智能體」按鈕**（加號圖示 +「新智能體」），開啟建立對話方塊。

**新增視圖切換**：清單視圖（目前）和組織架構圖視圖。**組織架構圖視圖：**
- 顯示報表層次結構的樹狀佈局
- 每個節點顯示：座席姓名、角色、狀態徽章
- 執行長在上，直接下屬在下，等等。
- 使用已傳回 `OrgNode[]` 的 `agentsApi.org(companyId)` 端點
- 點擊節點導航至智能體詳細信息

**清單視圖改進：**
- 在每行新增適配器類型作為小晶片/標籤
- 新增「最後活動」相對時間戳
- 如果智能體目前有運行心跳，則新增運行指示器（動畫點）

**過濾：**
- 選項卡過濾器：全部、活動、暫停、錯誤（類似問題頁面模式）

## 5.API 表面

所有端點均已存在。 V1 不需要新的伺服器工作。

|行動|端點|使用者 |
|--------|----------|---------|
|智能體列表 | `GET /companies/:id/agents` |列表頁 |
|取得組織樹 | `GET /companies/:id/org` |組織架構圖視圖 |
|建立智能體 | `POST /companies/:id/agents` |建立對話方塊 |
|更新智能體程式| `PATCH /agents/:id` |設定選項卡 |
|暫停/恢復/終止 | `POST /agents/:id/{action}` |標題操作 |
|重置會話 | `POST /agents/:id/runtime-state/reset-session` |溢位選單|
|建立 API 金鑰 | `POST /agents/:id/keys` |溢位選單|
|取得運行時狀態 | `GET /agents/:id/runtime-state` |概述標籤、屬性面板 |
|呼叫/喚醒| `POST /agents/:id/heartbeat/invoke` |標題呼叫按鈕 |
|清單運行 | `GET /companies/:id/heartbeat-runs?agentId=X` |運行選項卡|
|取消運行 | `POST /heartbeat-runs/:id/cancel` |運行詳情|
|舉辦活動| `GET /heartbeat-runs/:id/events` |日誌檢視器 |
| 運行日誌| `GET /heartbeat-runs/:id/log` |完整日誌視圖 |
