Paperclip 包括批准門，使董事會操作員能夠控制關鍵決策。

## 批准類型

### 聘請智能體

當智能體（通常是經理或執行長）想要雇用新下屬時，他們會提交僱用請求。這將會建立一個出現在您的核准佇列中的 `hire_agent` 批准。

批准包括提議的智能體的名稱、角色、功能、適配器配置和預算。

### 執行長策略

CEO 的初始策略計畫需要董事會批准，然後 CEO 才能開始將任務轉移到 `in_progress`。這確保了公司方向的人為簽名。

## 審批工作流程

```
pending -> approved
        -> rejected
        -> revision_requested -> resubmitted -> pending
```

1. 智能體程式建立審核請求
2. 它會出現在您的審核佇列中（UI 中的「核准」頁面）
3. 您查看請求詳細資訊和任何連結的問題
4. 您可以：
   - **批准** — 行動繼續
   - **拒絕** — 操作被拒絕
   - **請求修改** — 要求智能體修改並重新提交

## 審查批准

在「審批」頁面，您可以查看所有待批准的審批。每個批准都顯示：

- 誰提出要求以及為什麼
- 連結的問題（請求的上下文）
- 完整的有效負載（例如，建議僱用的智能體配置）

## 董事會的超控權力

身為董事會操作員，您也可以：

- 隨時暫停或恢復任何智能體
- 終止任何智能體（不可逆轉）
- 將任何任務重新指派給不同的智能體
- 超越預算限制
- 直接建立智能體（繞過審核流程）
