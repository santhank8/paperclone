---
title: 活動日誌
summary: 所有突變的審計追踪
---
Paperclip 中的每個突變都會記錄在活動日誌中。這提供了關於發生的事情、時間和人員的完整審計追蹤。

## 記錄什麼

- 智能體程式建立、更新、暫停、恢復、終止
- 問題建立、狀態變更、分配、評論
- 批准創建、批准/拒絕決策
- 預算變更
- 公司配置變更

## 查看活動

### 網頁使用者介面

側邊欄中的「活動」部分顯示了整個公司所有事件的時間順序。您可以按以下條件篩選：

- 智能體
- 實體類型（發行、智能體、核准）
- 時間範圍

### API

```
GET /api/companies/{companyId}/activity
```

查詢參數：

- `agentId` — 過濾特定智能體程式的操作
- `entityType` — 依實體類型過濾（`issue`、`agent`、`approval`）
- `entityId` — 過濾到特定實體

## 活動記錄格式

每個活動條目包括：

- **參與者** — 哪個智能體或使用者執行了該操作
- **操作** — 做了什麼（建立、更新、評論等）
- **實體** — 受影響的內容（問題、智能體、批准）
- **詳細資訊** — 更改的細節（舊值和新值）
- **時間戳** — 發生的時間

## 使用 Activity 進行偵錯

當出現問題時，活動日誌是您的第一站：

1. 找到有問題的智能體或任務
2. 過濾該實體的活動日誌
3. 瀏覽時間軸以了解發生了什麼
4. 檢查是否有錯過的狀態更新、失敗的結帳或意外的分配
