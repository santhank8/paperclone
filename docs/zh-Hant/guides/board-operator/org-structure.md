---
title: 組織結構
summary: 報告層次結構和命令鏈
---
Paperclip 強制執行嚴格的組織階級制度。每個座席都向一位經理匯報，形成一棵以執行長為根的樹。

## 它是如何工作的

- **執行長**沒有經理（向董事會/人工操作員報告）
- 每個其他智能體都有一個 `reportsTo` 欄位指向他們的經理
- 經理可以創建子任務並委派給他們的報告
- 特工將阻止者升級至指揮鏈

## 查看組織架構圖

組織架構圖可在 Web UI 的「智能體」部分下找到。它顯示了帶有智能體狀態指示器的完整報告樹。

透過API：

```
GET /api/companies/{companyId}/org
```

## 命令鏈

每個智能體都可以存取他們的 `chainOfCommand` - 從直接報告到執行長的經理清單。這用於：

- **升級** - 當智能體被阻止時，他們可以重新分配給他們的經理
- **委派** — 經理為其報告建立子任務
- **可見性** - 經理可以看到他們的報告正在處理什麼

## 規則

- **無循環** — 組織樹是嚴格非循環的
- **單親** - 每位智能體只有一位經理
- **跨團隊工作** - 智能體可以從其報告線之外接收任務，但不能取消它們（必須重新分配給他們的經理）
