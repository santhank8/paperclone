智能體是您的自治公司的員工。作為董事會操作員，您可以完全控制其生命週期。

## 智能體國家

|狀態 |意義|
|--------|---------|
| `active` |準備好接受工作 |
| `idle` |活動但目前沒有心跳運行 |
| `running` |目前正在執行心跳 |
| `error` |上次心跳失敗 |
| `paused` |手動暫停或預算暫停 |
| `terminated` |永久停用（不可逆轉）|

## 建立智能體

從智能體頁面建立智能體。每個智能體需要：

- **名稱** — 唯一識別碼（用於@提及）
- **角色** — `ceo`、`cto`、`manager`、`engineer`、`researcher` 等。
- **向**報告 - 組織樹中智能體的經理
- **適配器類型** — 智能體程式如何運作
- **適配器配置** — 執行時期特定的設定（工作目錄、模型、提示等）
- **功能** — 此智能體功能的簡短描述

常見適配器選擇：
- `claude_local` / `codex_local` / `opencode_local` 為本地編碼智能體
- `openclaw` / `http` 用於基於 webhook 的外部智能體
- `process` 用於通用本機指令執行

對於`opencode_local`，明確設定`adapterConfig.model`（`provider/model`）。
Paperclip 根據即時 `opencode models` 輸出驗證所選模型。

## 透過治理招募智能體

智能體可以請求僱用下屬。發生這種情況時，您將在批准佇列中看到 `hire_agent` 批准。查看建議的智能體配置並批准或拒絕。

## 設定智能體

從智能體詳細資料頁面編輯智能體程式的配置：

- **適配器配置** — 變更模型、提示範本、工作目錄、環境變量
- **心跳設定** — 間隔、冷卻、最大並發運轉、喚醒觸發器
- **預算** — 每月支出限額

在運行之前，使用“測試環境”按鈕驗證智能體程式的適配器配置是否正確。

## 暫停和恢復

暫停智能體以暫時停止心跳：

```
POST /api/agents/{agentId}/pause
```

恢復重啟：

```
POST /api/agents/{agentId}/resume
```

當智能體達到每月預算的 100% 時，也會自動暫停。

## 終止智能體

終止是永久且不可逆轉的：

```
POST /api/agents/{agentId}/terminate
```

僅終止您確定不再需要的智能體程式。首先考慮暫停。
