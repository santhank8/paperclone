## Story

**As a** CTO ou founder com dúvidas antes de se cadastrar,
**I want** encontrar respostas claras às minhas principais objeções numa seção de FAQ,
**so that** eu remova as últimas barreiras à conversão sem precisar contatar o time de vendas.

## Acceptance Criteria

1. Seção `#faq` implementada com mínimo de 6 perguntas e respostas
2. Perguntas focam nas objeções do ICP não-técnico: segurança, custo, curva de aprendizado, controle, lock-in
3. Componente accordion (expandir/colapsar) com transição CSS suave — sem JavaScript framework
4. Estado acessível: `<details>/<summary>` HTML nativo ou ARIA `expanded`/`collapsed` correto
5. Primeira pergunta aberta por padrão ou todas fechadas — implementar conforme melhor UX
6. Design: perguntas em `--text-primary`, respostas em `--text-secondary`, separadores `--border`
7. Heading da seção: "Perguntas frequentes" ou "Dúvidas mais comuns"
8. Link no final: "Tem mais dúvidas? Fale com a gente" → email ou chat

## Dev Notes

### Fonte: `docs/lp2/project-brief.md`

**Objetivo do FAQ** (do project brief): "responder objeções de líderes não-técnicos (segurança, custo, curva de aprendizado)" — estas são as principais barreiras de conversão para o ICP da LP2.

**Implementação recomendada — HTML nativo `<details>/<summary>`:**
```html
<details>
  <summary>Preciso saber programar para usar?</summary>
  <p>Não. O Paperclip tem uma interface visual intuitiva para configurar agentes,
  acompanhar o progresso e revisar resultados. Seu time de dev configura os agentes
  técnicos — você acompanha o resultado.</p>
</details>
```

**Vantagem:** `<details>/<summary>` é HTML nativo sem necessidade de JavaScript. Acessível por padrão (keyboard navigable). Apenas adicionar CSS para estilizar.

**CSS de transição suave para accordion:**
```css
details > summary {
  list-style: none;
  cursor: pointer;
  display: flex;
  justify-content: space-between;
  align-items: center;
}
details > summary::after {
  content: '+';
  font-size: 1.5rem;
  color: var(--bb-lime);
  transition: transform 0.2s ease;
}
details[open] > summary::after {
  content: '−';
}
```

**Nota:** Não usar `transition` em `height: auto` (não funciona). Usar opacity + max-height se quiser transição na resposta — ou aceitar sem transição de conteúdo (apenas o ícone).

### Testing
