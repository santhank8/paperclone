## Story

**As a** desenvolvedor implementando a Landing Page v2,
**I want** uma estrutura HTML/CSS base em `landing-v2/index.html` com o design system configurado,
**so that** todas as stories de seção possam ser implementadas consistentemente sobre uma base sólida com tokens, fontes e layout grid definidos.

## Acceptance Criteria

1. Arquivo `landing-v2/index.html` criado com estrutura HTML5 semântica válida
2. Design tokens CSS configurados (cores, tipografia, espaçamentos) conforme `docs/lp2/project-brief.md` — manter Dark Cockpit palette: `--bb-lime: #D1FF00`, `--bb-dark`, `--bb-cream`, etc.
3. Fontes importadas: `Geist`, `Geist Mono` via Google Fonts (consistência com LP1)
4. Layout responsivo base: desktop-first, mobile-responsive — breakpoints: `md: 768px`, `lg: 1024px`, `xl: 1280px`
5. Meta tags SEO diferenciadas de LP1 (title, description, og:tags) para não canibalizar
6. `landing-v2/.gitignore` criado se necessário
7. Performance baseline: estrutura sem bloquear LCP — inlining crítico de CSS above-the-fold
8. `<nav>` de navegação interna com links âncora para as seções principais

## Dev Notes

### Fonte: `docs/lp2/project-brief.md`

**Repositório:** mesma estrutura que LP1 (`landing/`), mas em subpasta `landing-v2/` para não quebrar LP1.

**Design System:** Manter Dark Cockpit tokens — LP2 não é "LP1 reformatada", mas usa a mesma paleta.

**Referência de tokens** (extraído do `docs/frontend/landing-visual-spec.md`):
```css
--bb-lime: oklch(93.4% .2264 121.95); /* #D1FF00 */
--bb-dark: oklch(11.49% 0 0);
--bb-cream: oklch(96.39% .0158 106.69);
--bb-muted: oklch(79.52% 0 0);
--bg-base: #0a0a0a;
--bg-surface: oklch(16.93% .0041 285.95);
--text-primary: oklch(99.52% .0235 106.82);
--text-secondary: oklch(69.27% 0 0);
```

**SEO diferenciado:** URL da LP2 ainda não definida (Open Question no project brief). Usar `og:url` provisório ou deixar vazio.

**Performance targets** (do project brief): LCP < 2.5s, CLS < 0.1, TBT < 200ms.

**Não usar:** backend custom, frameworks JS (LP estática como LP1).

### Testing

- Validar HTML com W3C Validator (sem erros críticos)
- Abrir no browser e verificar que a estrutura base está correta
- Verificar responsividade em 375px (mobile), 768px (tablet), 1280px (desktop)
- Checar que nenhuma fonte está bloqueando renderização
