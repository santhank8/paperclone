## Story

**As a** CTO ou founder visitando a LP2,
**I want** ver depoimentos reais ou logos de early adopters com contexto de resultado,
**so that** eu tenha credibilidade social suficiente para avançar no funil sem sentir que é uma ferramenta experimental sem usuários reais.

## Acceptance Criteria

1. Seção `#social-proof` implementada com 3-5 cards de depoimento
2. Cada card de depoimento contém: quote, nome/role, empresa (ou placeholder coerente para MVP)
3. Quotes focam em outcomes de negócio — não em features técnicas ("Entregamos o roadmap do Q2 sem contratar ninguém" vs "Os agentes são incríveis")
4. Layout: grid responsivo — 3 colunas desktop, 1 coluna mobile
5. Estilo visual: cards com `--bg-surface`, borda sutil `--border`, texto `--text-secondary`
6. Badge/label opcional "Early Adopters" ou "O que dizem sobre nós" acima dos cards
7. Se logos de empresas: seção de logos em escala de cinza com hover colorido
8. Dados podem ser mockados para MVP — mas devem ser plausíveis para o ICP (CTOs, founders, eng leads)

## Dev Notes

### Fonte: `docs/lp2/project-brief.md`

**Risco identificado:** "Sem social proof, seção fica vazia ou mockada demais" — o project brief reconhece este risco. Para MVP, usar depoimentos ilustrativos coerentes com o ICP.

**Conteúdo mockado sugerido (plausível):**

```

— Ana R., Founder · E-commerce B2B

"A taxa de bugs em produção caiu 60% no primeiro mês. O QA automatizado é transformador."

```

**Estrutura HTML sugerida para cada card:**
```html
<article class="testimonial-card">
  <blockquote>"[quote]"</blockquote>
  <footer>
    <strong>[Nome]</strong>, [Role] · [Empresa]
  </footer>
</article>
```

**Nota:** o project brief levanta a questão de obter permissão para usar logos/nomes reais. Para MVP, usar nomes fictícios mas plausíveis. Remover nota de "ilustrativo" quando dados reais forem captados.
