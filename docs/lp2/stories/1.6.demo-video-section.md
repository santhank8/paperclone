## Story

**As a** CTO ou founder visitando a LP2,
**I want** ver uma demonstração visual (video ou screenshots) do produto em ação mostrando o ciclo completo,
**so that** eu entenda como funciona na prática sem precisar ler documentação técnica — e tome a decisão de me cadastrar.

## Acceptance Criteria

1. Seção `#demo` implementada com conteúdo visual do produto
2. **Opção A (Video):** Player de vídeo embed (YouTube ou equivalente) com controles — `loading="lazy"`, sem autoplay sem som
3. **Opção B (Screenshots):** Carrossel ou grid de 3-5 screenshots/mockups do dashboard focados em resultado (não em terminal)
4. Título da seção: "Veja como funciona" ou equivalente — sem jargão técnico
5. Descrição breve de 1-2 frases contextualizando o que está sendo mostrado (foco em outcome, não em feature)
6. Se video: thumbnail customizado, lazy loading, não bloquear LCP
7. Performance: elemento visual não deve degradar tempo de carregamento — usar `loading="lazy"` e dimensões explícitas
8. Fallback elegante caso o vídeo não carregue (mensagem ou screenshot estático)
9. CTA após a demo: "Experimente você mesmo" → link para signup

## Dev Notes

### Fonte: `docs/lp2/project-brief.md`

**Especificação do project brief:** "screen recording de 60s mostrando um ciclo completo (não apenas terminal)" — o vídeo deve mostrar o produto do ponto de vista do usuário (CTO/founder), não do dev.

**Conteúdo ideal para o vídeo/screenshots:**
- Painel de issues/tarefas (do Paperclip) com agentes atribuídos
- Ciclo de desenvolvimento: story → implementação → QA → deploy (sem mostrar código em detalhes)
- Run trail auditável (valor para CTOs)
- Dashboard com métricas de velocidade

**Nota:** O vídeo pode não estar disponível no momento da implementação. Neste caso, implementar a estrutura HTML com placeholder e comentário `<!-- TODO: substituir por video real quando disponível -->`. Usar screenshots do dashboard como fallback.

**Performance crítico:** Este é o elemento que mais impacta LCP/TBT. Priorizar `loading="lazy"` e evitar carregamento síncrono de scripts externos.

**Não usar:** Terminal CLI como visual principal. A audiência de LP2 (CTOs/founders) tem baixa afinidade com interfaces de linha de comando.
