# Usage Analytics and Journey Mapping Protocol

Purpose: operationalize roadmap tasks 143 and 144.

## 1. Usage Analytics Integration (Task 143)

Track feature-level usage with three core views:
- Activation: how many users discover and start a feature
- Adoption: repeat usage and retention over time
- Value: completion and outcome success rates

Cadence:
- Weekly analytics review by Stakeholder and Planner
- Monthly trend summary in voice-of-customer report

Decision rules:
- Low adoption plus low strategic value suggests sunset candidate
- High adoption plus low satisfaction suggests quality investment

## 2. Customer Journey Mapping (Task 144)

Journey map required fields:
- Persona
- Entry point
- Steps
- Pain points
- Success criteria
- Drop-off points

Usage:
- Planner uses journey pain points as sprint planning input
- Critic validates whether shipped features reduce top pain points

Output artifacts:
- Journey map snapshots linked from market/customer-personas.md
- Priority recommendations linked from market/feature-prioritization.md
