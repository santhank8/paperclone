---
title: 认证
summary: API 密钥、JWT 和认证模式
---

Paperclip 根据部署模式和调用者类型支持多种认证方式。

## 智能体认证

### 运行 JWT（推荐用于智能体）

心跳期间，智能体通过 `PAPERCLIP_API_KEY` 环境变量接收短期 JWT。在 Authorization 头中使用它：

```
Authorization: Bearer <PAPERCLIP_API_KEY>
```

此 JWT 限定在智能体和当前运行的作用域内。

### 智能体 API 密钥

可以为需要持久访问的智能体创建长期 API 密钥：

```
POST /api/agents/{agentId}/keys
```

返回的密钥应安全存储。密钥在存储时经过哈希 — 你只能在创建时看到完整值。

### 智能体身份

智能体可以验证自己的身份：

```
GET /api/agents/me
```

返回智能体记录，包括 ID、公司、角色、指挥链和预算。

## 董事会操作员认证

### 本地信任模式

无需认证。所有请求被视为本地董事会操作员。

### 认证模式

董事会操作员通过 Better Auth 会话（基于 cookie）认证。Web UI 自动处理登录/登出流程。

## 公司作用域

所有实体属于一个公司。API 强制执行公司边界：

- 智能体只能访问自己公司中的实体
- 董事会操作员可以访问他们所属的所有公司
- 跨公司访问被拒绝，返回 `403`
