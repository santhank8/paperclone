# v2026.401.0

> Released: 2026-04-01

## Highlights

- **Penclip release pipeline hardening** — GitHub release automation now treats `penclipai/paperclip-cn` as the canonical publish source, requires an explicit canonical remote for local manual release commands, and prevents accidental tag or GitHub Release pushes to upstream remotes.
- **Automated browser smoke after publish** — The release workflow now chains the reusable Docker + Playwright smoke suite after both canary and stable publishes, validating the real public install path before a release is considered complete.
- **Public CLI command cleanup** — User-facing onboarding and recovery guidance now consistently points to `penclip` / `npx penclip` instead of stale `paperclipai` command names.

## Improvements

- **Release summaries in GitHub Actions** — Canary, stable, and stable dry-run workflows now write explicit version, tag, smoke status, and rollback guidance to the Actions job summary.
- **UI package publishing docs** — Maintainer docs now reference the real public package name `@penclipai/ui` and the `penclipai` npm organization.
- **Worktree release gate stability** — The linked-worktree git hooks test now has a release-safe timeout budget on slower machines so `pnpm test:run` can serve as a stable pre-release gate.

## Upgrade Guide

This release changes the release process, not the runtime data model.

- No new database migrations are introduced.
- Local manual release dry-runs and publishes now require `RELEASE_REMOTE=<paperclip-cn-remote>` or `PUBLISH_REMOTE=<paperclip-cn-remote>`.
- The post-publish smoke path is now part of the normal canary and stable workflow, so a red smoke result should be treated as a release-blocking failure.
