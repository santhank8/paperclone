# v0.3.0

> 發佈時間：2026-03-09

## 亮點

- **新適配器：Cursor、OpenCode 和 Pi** — Paperclip 現在支援三個額外的本地編碼智能體。 Cursor 和 OpenCode 作為一流的適配器集成，具有模型發現、運行日誌流和技能注入功能。 Pi 新增了具有成本追蹤功能的本地 RPC 模式。所有三個都與 Claude Code 和 Codex 一起出現在入門嚮導中。 ([#62](https://github.com/paperclipai/paperclip/pull/62)、[#141](https://github.com/paperclipai/paperclip/pull/141)、[#240](https://github.com/paperclipai/paperclip/pull/240)、[#183](ZXQQQ00023QQQZ)、@aaaaa
- **OpenClaw 閘道轉接器** — 新的僅閘道 OpenClaw 串流取代了舊轉接器。它使用嚴格的 SSE 流，支援設備密鑰配對，並透過加入令牌驗證處理基於邀請的加入。 ([#270](https://github.com/paperclipai/paperclip/pull/270))
- **收件匣和未讀語意** - 問題現在追蹤每個使用者的閱讀狀態。未讀取指示器出現在收件匣、控制台和瀏覽器標籤（藍點）。收件匣徽章包括加入請求和批准，收件匣排序以警報為重點。 （[#196]（https://github.com/paperclipai/paperclip/pull/196），@hougangdev）
- **PWA 支援** — UI 作為可安裝的漸進式 Web 應用程式提供，帶有服務工作人員和增強的清單。 Service Worker 使用網路優先策略來防止內容過時。
- **智能體程式建立精靈** — 新的選擇模式和全頁設定流程使新增智能體變得更加容易。側邊欄 AGENTS 標題現在有一個快速新增按鈕。

## 改進- **Markdown 中的美人魚圖** — 圍欄 `mermaid` 區塊在問題評論和描述中呈現為圖表。
- **即時運行輸出** — 透過 WebSocket 即時運行詳細資訊頁面串流輸出，並合併增量和重複資料刪除的提要項目。
- **將評論複製為 Markdown** — 每個評論標題都有一個一鍵複製為 Markdown 按鈕。
- **重試失敗的運行** — 失敗和逾時的運行現在會在運行詳細資訊頁面上顯示「重試」按鈕。
- **可按一下專案狀態** — 專案屬性窗格中的狀態晶片現在可按一下以進行快速更新。
- **捲動至底部按鈕** — 向上捲動時，問題詳細資料和執行頁面會顯示浮動捲動到底部按鈕。
- **資料庫備份 CLI** — `paperclipai db:backup` 可讓您按需快照資料庫，並可選擇自動排程。
- **停用註冊** — 新的 `auth.disableSignUp` 設定選項（和 `AUTH_DISABLE_SIGNUP` 環境變數）可讓操作員鎖定註冊。 （[#279]（https://github.com/paperclipai/paperclip/pull/279），@JasonOA888）
- **刪除重複的短名稱** — 智能體和項目短名稱現在在建立和更新時自動刪除重複項，而不是拒絕重複項。 （[#264]（https://github.com/paperclipai/paperclip/pull/264），@mvanhorn）
- **人類可讀的角色標籤** — 智能體程式清單和屬性窗格顯示友善的角色名稱。 （[#263]（https://github.com/paperclipai/paperclip/pull/263），@mvanhorn）
- **受讓人選擇器排序** — 首先顯示最近的選擇，然後按字母順序顯示。
- **行動佈局最佳化** — 跨問題、收件匣和控制台統一 GitHub 樣式的問題行。改進了行動裝置上的彈出視窗滾動、命令面板居中和屬性切換。 （[#118]（https://github.com/paperclipai/paperclip/pull/118），@MumuTW）
- **邀請使用者體驗改進** — 邀請連結自動複製到剪貼簿、設定中僅片段流程、10 分鐘邀請 TTL 以及更清晰的網站主機指導。
- **評論上的永久連結錨點** — 每個評論都有一個穩定的錨鏈接和一個 GET-by-ID API 端點。
- **Docker 部署強化** — 預設經過驗證的部署模式、命名資料磁碟區、在撰寫檔案中公開的 `PAPERCLIP_PUBLIC_URL` 和 `PAPERCLIP_ALLOWED_HOSTNAMES`、執行狀況檢查資料庫等待和 Node 24 基礎映像。 ([#400](https://github.com/paperclipai/paperclip/pull/400)、[#283](https://github.com/paperclipai/paperclip/pull/283)、[#284](https://github.com/paperclipai/paperclip/pull/284)、@AiMagic5000、@mingfang)
- **更新了模型清單** — 將 `claude-sonnet-4-6`、`claude-haiku-4-6` 和 `gpt-5.4` 加入到適配器模型常數中。 （[#293]（https://github.com/paperclipai/paperclip/pull/293），[#110]（https://github.com/paperclipai/paperclip/pull/110），@cpfarhood，@artokun）
- **Playwright e2e 測試** — 新的端對端測試套件涵蓋了入門精靈流程。

## 修復- **運行日誌中的秘密編輯** — 來自秘密的環境變數現在按來源進行編輯，並具有一致的 `secretKeys` 追蹤。 （[#261]（https://github.com/paperclipai/paperclip/pull/261），@mvanhorn）
- **SPA catch-all 500s** — 伺服器在 catch-all 路由中提供快取的 `index.html`，並在 `sendFile` 中使用 `root`，防止點檔案路徑和 SPA 刷新上出現 500 刷新錯誤。 （[#269]（https://github.com/paperclipai/paperclip/pull/269），[#78]（https://github.com/paperclipai/paperclip/pull/78），@mvanhorn，@MumuTW）
- **不匹配的 API 路由返回 404 JSON** — 之前落入 SPA 處理程序。 （[#269]（https://github.com/paperclipai/paperclip/pull/269），@mvanhorn）
- **智能體喚醒邏輯** — 當問題移出積壓工作時，智能體會喚醒，跳過自己評論的自我喚醒，並跳過積壓狀態更改的喚醒。待批准的智能體不包括在心跳計時器中。 ([#159](https://github.com/paperclipai/paperclip/pull/159)、[#154](https://github.com/paperclipai/paperclip/pull/154)、[#267](https://github.com/paperclipai/paperclip/pull/267)、[#72](ZXQQQ00042QQQXZ)、@Logesh-c​​at
- **執行日誌 fd 洩漏** — 修正了日誌附加中導致 `spawn EBADF` 錯誤的檔案描述符洩漏。 （[#266]（https://github.com/paperclipai/paperclip/pull/266），@mvanhorn）
- **500 錯誤日誌記錄** — 錯誤日誌現在包括實際的錯誤訊息和請求上下文，而不是通用的 pino-http 輸出。
- **布林環境解析** - `parseBooleanFromEnv` 不再默默地將常見的 true 值視為 false。 （[#91]（https://github.com/paperclipai/paperclip/pull/91），@zvictor）
- **載入環境預設值** - `onboard` 現在可以正確地從環境變數中匯出機密，並報告 `local_trusted` 模式下忽略的暴露設定。 （[#91]（https://github.com/paperclipai/paperclip/pull/91），@zvictor）
- **Windows 路徑相容性** — 遷移路徑使用 `fileURLToPath` 進行 Windows 安全性解析。 （[#265]（https://github.com/paperclipai/paperclip/pull/265），[#413]（https://github.com/paperclipai/paperclip/pull/413），@mvanhorn，@online5880）
- **HTTP 上的安全性 cookie** — 停用普通 HTTP 部署的安全性 cookie 標誌，以防止身分驗證失敗。 （[#376]（https://github.com/paperclipai/paperclip/pull/376），@dalestubblefield）
- **URL 編碼** — `buildUrl` 分割路徑和查詢以防止 `%3F` 編碼問題。 （[#260]（https://github.com/paperclipai/paperclip/pull/260），@mvanhorn）
- **驗證可信任來源** — 有效的可信任來源和允許的主機名稱現在可以在公共模式下正確應用。 （[#99]（https://github.com/paperclipai/paperclip/pull/99），@zvictor）
- **UI 穩定性** — 修正了提示範本清空時的空白畫面、搜尋 URL 同步導致重新渲染、收件匣中問題標題溢位以及側邊欄徽章計數（包括核准）。 ([#262](https://github.com/paperclipai/paperclip/pull/262)、[#196](https://github.com/paperclipai/paperclip/pull/196)、[#423](https://github.com/paperclipai/paperclip/pull/423)、@mvanhorn、@hougangdev、@RememberV)
