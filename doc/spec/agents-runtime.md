# 智能体运行时指南

状态：面向用户的指南
最后更新：2026-02-17
受众：在 Paperclip 中设置和运行智能体的操作员

## 1. 该系统做什么

Paperclip 中的智能体不会持续运行。
它们以**心跳**方式运行：由唤醒触发的短执行窗口。

每次心跳：

1. 启动配置的智能体适配器（例如 Claude CLI 或 Codex CLI）
2. 给它当前的提示/上下文
3. 让它工作直到退出、超时或被取消
4. 存储结果（状态、token 使用量、错误、日志）
5. 实时更新 UI

## 2. 智能体何时被唤醒

智能体可以通过四种方式被唤醒：

- `timer`：定时间隔（例如每 5 分钟）
- `assignment`：当工作被分配/检出给该智能体时
- `on_demand`：手动唤醒（按钮/API）
- `automation`：系统触发的唤醒，用于未来的自动化

如果智能体已在运行，新的唤醒会被合并（合并）而非启动重复运行。

## 3. 每个智能体需要配置什么

## 3.1 适配器选择

常见选择：

- `claude_local`：运行你的本地 `claude` CLI
- `codex_local`：运行你的本地 `codex` CLI
- `process`：通用 shell 命令适配器
- `http`：调用外部 HTTP 端点

对于 `claude_local` 和 `codex_local`，Paperclip 假设 CLI 已在主机上安装并完成认证。

## 3.2 运行时行为

在智能体运行时设置中，配置心跳策略：

- `enabled`：允许定时心跳
- `intervalSec`：定时器间隔（0 = 禁用）
- `wakeOnAssignment`：分配工作时唤醒
- `wakeOnOnDemand`：允许 ping 风格的按需唤醒
- `wakeOnAutomation`：允许系统自动化唤醒

## 3.3 工作目录和执行限制

对于本地适配器，设置：

- `cwd`（工作目录）
- `timeoutSec`（每次心跳的最大运行时间）
- `graceSec`（超时/取消后强制终止前的时间）
- 可选的环境变量和额外 CLI 参数

## 3.4 提示模板

你可以设置：

- `promptTemplate`：用于每次运行（首次运行和恢复的会话）

模板支持变量如 `{{agent.id}}`、`{{agent.name}}`，以及运行上下文值。

## 4. 会话恢复行为

Paperclip 按 `(agent, taskKey, adapterType)` 存储可恢复的会话状态。
`taskKey` 从唤醒上下文（`taskKey`、`taskId` 或 `issueId`）派生。

- 相同任务键的心跳复用该任务的前一个会话。
- 同一智能体的不同任务键保持独立的会话状态。
- 如果恢复失败，适配器应使用全新会话重试一次并继续。
- 你可以重置智能体的所有会话或按任务键重置一个任务会话。

在以下情况使用会话重置：

- 你显著更改了提示策略
- 智能体陷入了坏循环
- 你想要干净的重新开始

## 5. 日志、状态和运行历史

每次心跳运行你会获得：

- 运行状态（`queued`、`running`、`succeeded`、`failed`、`timed_out`、`cancelled`）
- 错误文本和 stderr/stdout 摘录
- 适配器提供时的 token 使用量/成本
- 完整日志（存储在核心运行行之外，针对大输出优化）

在本地/开发设置中，完整日志存储在配置的运行日志路径下的磁盘上。

## 6. UI 中的实时更新

Paperclip 实时将运行时/活动更新推送到浏览器。

你应该看到以下内容的实时变化：

- 智能体状态
- 心跳运行状态
- 智能体工作导致的任务/活动更新
- 仪表盘/成本/活动面板（如相关）

如果连接中断，UI 会自动重新连接。

## 7. 常见运营模式

## 7.1 简单自主循环

1. 启用定时器唤醒（例如每 300 秒）
2. 保持分配唤醒开启
3. 使用聚焦的提示模板
4. 观察运行日志并随时间调整提示/配置

## 7.2 事件驱动循环（减少持续轮询）

1. 禁用定时器或设置长间隔
2. 保持分配唤醒启用
3. 使用按需唤醒进行手动推动

## 7.3 安全优先循环

1. 短超时
2. 保守的提示
3. 监控错误 + 需要时快速取消
4. 出现漂移时重置会话

## 8. 故障排除

如果运行反复失败：

1. 检查适配器命令可用性（`claude`/`codex` 已安装并登录）。
2. 验证 `cwd` 存在且可访问。
3. 检查运行错误 + stderr 摘录，然后查看完整日志。
4. 确认超时不会太短。
5. 重置会话并重试。
6. 如果智能体导致反复的错误更新，暂停该智能体。

典型的失败原因：

- CLI 未安装/未认证
- 错误的工作目录
- 格式错误的适配器参数/环境变量
- 提示太宽泛或缺少约束
- 进程超时

## 9. 安全和风险说明

本地 CLI 适配器在主机上无沙箱运行。

这意味着：

- 提示指令很重要
- 配置的凭据/环境变量是敏感的
- 工作目录权限很重要

尽可能从最小权限开始，避免在广泛的可复用提示中暴露密钥，除非确实有意需要。

## 10. 最小设置检查清单

1. 选择适配器（`claude_local` 或 `codex_local`）。
2. 将 `cwd` 设置为目标工作区。
3. 添加引导 + 正常提示模板。
4. 配置心跳策略（定时器和/或分配唤醒）。
5. 触发手动唤醒。
6. 确认运行成功且会话/token 使用量已记录。
7. 观察实时更新并迭代提示/配置。
