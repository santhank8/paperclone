# 2026-03-16 Daily Log - Activity Drilldown

## Repo context inspected today

### 1. Product and workflows

- Paperclip remains a company-scoped control plane, not a general chat surface.
- Core work stays anchored to goals, projects, issues, comments, approvals, runs, and cost visibility.
- The board default view should answer what the company is doing, who is doing it, and what needs intervention.

### 2. Actors, auth, and permissions

- Board access is mode-dependent through `actorMiddleware`, with `local_trusted` implicit board access and `authenticated` session-backed board access.
- Company boundaries are enforced through `assertCompanyAccess`, and authenticated board mutations are protected by the trusted-origin guard.
- Agent credentials remain company-scoped through API keys or local agent JWTs.

### 3. APIs, routes, and orchestration

- The server mounts company-scoped routes under `/api` from `server/src/app.ts`.
- Activity is already a first-class route group with company and issue views, backed by activity-log writes across issue, agent, approval, project, company, and cost mutations.
- The existing activity service already supported company, agent, entity-type, and entity-id filters, which made it a safe place to deepen board visibility.

### 4. Types, interfaces, validators, and schema

- Shared API/domain contracts live in `packages/shared`, and the activity surface previously exported only `ActivityEvent`.
- Activity data is backed by `activity_log`, keyed by `company_id`, actor metadata, entity metadata, run linkage, and optional JSON details.
- No schema changes or migrations were required for this improvement.

### 5. UI workflows and persistence

- `ui/src/pages/IssueDetail.tsx` already uses visibility-aware polling for comments, activity, linked runs, and approvals.
- The company Activity page did not yet use the same visibility-aware polling and only offered a local entity-type filter.
- The current UI pattern supports additive query-based drilldown without expanding the product beyond the existing board visibility model.
