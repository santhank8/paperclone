# ClipHub — 公司註冊處

**下載公司。 **

ClipHub 是人們分享、發現和下載 Paperclip 公司配置的公開註冊表。公司範本是一種便攜式工件，包含整個組織（智能體程式、報表結構、適配器配置、角色定義、種子任務），準備透過一個指令啟動。

---

## 它是什麼

ClipHub 之於 Paperclip 就像是包註冊表之於程式語言。 Paperclip 已經支援可匯出的組織配置（請參閱 [SPEC.md](SPEC.md) §2）。 ClipHub 是這些匯出所在的公共目錄。

用戶在 Paperclip 中建立了一家工作公司——一家開發商店、一家行銷機構、一家研究實驗室、一家內容工作室——導出模板，並將其發佈到 ClipHub。任何人都可以在自己的 Paperclip 實例上瀏覽、搜尋、下載和啟動該公司。

標語：**您可以真正下載一家公司。 **

---

## 發佈內容

ClipHub 套件是**公司範本匯出** — Paperclip 規格中定義的便攜式工件格式。它包含：

|組件|描述 |
|---|---|
| **公司元資料** |名稱、描述、預期用例、類別 |
| **組織架構圖** |完整的報告層次結構－誰向誰報告|
| **智能體定義** |每個智能體：姓名、角色、頭銜、能力描述 |
| **適配器配置** |每個智能體適配器類型和配置（SOUL.md、HEARTBEAT.md、CLAUDE.md、處理命令、Webhook URL — 無論適配器需要什麼）|
| **種子任務** |用於引導公司首次運行的可選啟動任務和舉措 |
| **預算預設值** |每個智能體和每個公司的建議代幣/成本預算 |

模板是**結構，而不是狀態。 **沒有正在進行的任務，沒有歷史成本數據，沒有運行時工件。只是藍圖。

### 子包

並非每個用例都需要整個公司。 ClipHub 也支援發佈單一元件：

- **智能體模板** — 單一智能體配置（例如“高級 TypeScript 工程師”、“SEO 內容編寫者”、“DevOps 智能體”）
- **團隊範本** — 組織架構圖的子樹（例如「行銷團隊：CMO + 3 份報告」、「工程 Pod：技術主管 + 4 名工程師」）
- **適配器配置** — 獨立於任何特定智能體角色的可重複使用適配器配置

這些可以混入現有公司。下載智能體，將其放入您的組織，分配經理，然後開始。

## 核心特性

### 瀏覽和發現

主頁從多個維度展示公司：

- **精選** — 編輯策劃的高品質模板
- **流行** — 按下載量、星級和分叉排名
- **最近** — 最新發布或更新
- **類別** — 可依使用案例瀏覽（請參閱下方的類別）每個清單顯示：名稱、簡短描述、組織規模（智能體數量）、類別、使用的適配器類型、星數、下載數量和迷你組織圖表預覽。

### 搜尋

搜尋是**語義性的，而不是僅限關鍵字的。 ** 由向量嵌入提供支持，因此您可以按意圖進行搜尋：

- “運行 Facebook 廣告的營銷機構” → 找到相關的公司模板，即使標題中沒有這些確切的詞
- “建造 APIs 的小型開發團隊”→ 找到精實工程組織
- “與作家和編輯的內容管道”→ 尋找內容工作室模板

也支援依下列條件篩選：類別、智能體程式計數範圍、適配器類型、星級計數、新近度。

### 公司詳細資料頁面

點選進入公司範本會顯示：

- **完整描述** — 這家公司從事什麼業務、如何運作、有何期望
- **互動式組織架構圖** — 每個智能體的視覺化樹，包含角色、頭銜和能力
- **智能體清單** — 每個智能體程式可擴展的詳細資訊（適配器類型、配置摘要、角色描述）
- **種子任務** — 包括啟動計劃和任務
- **預算概覽** — 建議的成本結構
- **安裝指令** — 一行CLI指令下載並建立
- **版本歷史記錄** — 變更日誌、semver、先前可用的版本
- **社群** — 星星、評論、分叉計數

### 安裝和分叉

使用模板的兩種方法：

**安裝（全新開始）：**
```
paperclip install cliphub:<publisher>/<company-slug>
```
下載範本並在本機 Paperclip 實例中建立新公司。您新增自己的 API 金鑰、設定預算、自訂智能體，然後點擊開始。

**叉子：**
Forking 在您自己的 ClipHub 帳戶下建立範本的副本。您可以修改它，將其重新發佈為您自己的變體，並追蹤分叉譜系。這使得漸進式改進成為可能——有人發布了一個行銷機構，你分叉它，添加一個社交媒體團隊，然後重新發布。

### 星星和評論

- **星星** — 書籤和信號品質。星星數是主要的排名訊號。
- **評論** — 對每個清單進行主題討論。提出問題、分享結果、提出改進建議。

### 下載計數和訊號

每次安裝都會被計算在內。註冊表追蹤：

- 總下載量（所有時間）
- 每個版本的下載量
- 貨叉計數
- 星星計數

這些訊號會影響搜尋排名和發現。

## 出版

### 誰可以發布

任何擁有 GitHub 帳戶的人都可以發佈到 ClipHub。透過 GitHub OAuth 進行身份驗證。

### 如何發布

從 Paperclip 中，將您的公司匯出為模板，然後發布：

```
paperclip export --template my-company
paperclip publish cliphub my-company
```

或使用 Web UI 直接上傳模板匯出。

### 你提供什麼

發佈時，您指定：|領域 |必填 |說明 |
|---|---|---|
| `slug` |是的 | URL 安全標識符（例如 `lean-dev-shop`）|
| `name` |是的 |顯示名稱 |
| `description` |是的 |這家公司是做什麼的以及它的服務對象 |
| `category` |是的 |主要類別（見下文）|
| `tags` |沒有|用於發現的附加標籤 |
| `version` |是的 | Semver（例如 `1.0.0`）|
| `changelog` |沒有|此版本有何變化 |
|`readme` |沒有|擴充文件（降價）|
| `license` |沒有|使用條款|

### 版本控制

模板使用語意版本控制。每次發布都會創建一個不可變的版本。用戶可以安裝任何版本或預設為`latest`。版本歷史記錄和變更日誌在詳細資料頁面上可見。

### `sync` 指令

對於維護多個範本的進階使用者：

```
paperclip cliphub sync
```

掃描本機匯出的範本並發布任何新的或更新的範本。對於從單一儲存庫維護公司範本組合非常有用。

## 類別

公司範本按用例組織：

|類別 |範例 |
|---|---|
| **軟體開發** |全端開發商店，API 開發團隊，行動應用工作室 |
| **行銷與成長** |績效行銷智能體、內容行銷團隊、SEO 商店 |
| **內容與媒體** |內容工作室、Podcast製作、電子報營運 |
| **研究與分析** |市場研究公司、競爭情報、資料分析團隊|
| **營運** |客戶支援組織、內部營運團隊、品質保證/測試中心 |
| **銷售** |外向銷售團隊、潛在客戶開發、客戶管理 |
| **財務與法律** |簿記服務、合規監控、財務分析 |
| **創意** |設計機構、文案工作室、品牌發展|
| **通用** |入門範本、最小組織、單一智能體設定 |

類別不是排他性的－範本可以有一個主要類別以及用於橫切關注點的標籤。

## 節制與信任

### 已驗證的發布商

滿足特定門檻（帳戶年齡、已發布的具有良好訊號的模板）的發布商將獲得經過驗證的徽章。經過驗證的模板在搜尋中排名更高。

### 安全審查

公司範本包含適配器配置，其中可能包括可執行命令（進程適配器）或 Webhook URL（HTTP 適配器）。審核系統：

1. **自動掃描** — 檢查適配器配置是否存在可疑模式（任意程式碼執行、滲透 URL、憑證收集）
2. **社群報告** — 任何登入使用者都可以標記範本。多個報告等待審核後自動隱藏。
3. **手動審核** — 版主可以批准、拒絕或要求更改

### 帳戶門控新帳戶在發布之前有一段等待期。這可以防止路過的垃圾郵件。

## 架構

ClipHub 是一項獨立於 Paperclip 本身的**獨立服務**。 Paperclip是自架的； ClipHub 是 Paperclip 執行個體與之通訊的託管註冊表。

### 整合點

|層|角色 |
|---|---|
| **ClipHub 網路** |瀏覽、搜尋、發現、評論、加星標——網站 |
| **ClipHub API** |登錄 API 用於以程式設計方式發佈、下載、搜尋 |
| **Paperclip CLI** | `paperclipai install`、`paperclipai publish`、`paperclipai cliphub sync` — 內建於 Paperclip |
| **Paperclip UI** | Paperclip Web UI 中的「瀏覽 ClipHub」面板，無需離開應用程式即可發現範本 |

### 技術堆疊

|層|技術 |
|---|---|
|前端 | React + Vite（與Paperclip一致）|
|後端| TypeScript + Hono（與Paperclip一致）|
|資料庫| PostgreSQL |
|搜尋 |用於語意搜尋的向量嵌入 |
|授權 | GitHub OAuth |
|儲存|儲存在物件儲存（S3 或同等儲存）中的範本 zip |

### 資料模型（草圖）

```
Publisher
  id, github_id, username, display_name, verified, created_at

Template
  id, publisher_id, slug, name, description, category,
  tags[], readme, license, created_at, updated_at,
  star_count, download_count, fork_count,
  forked_from_id (nullable)

Version
  id, template_id, version (semver), changelog,
  artifact_url (zip), agent_count, adapter_types[],
  created_at

Star
  id, publisher_id, template_id, created_at

Comment
  id, publisher_id, template_id, body, parent_id (nullable),
  created_at, updated_at

Report
  id, reporter_id, template_id, reason, created_at
```

## 使用者流程

### “我想開一家公司”

1. 開啟ClipHub，按類別瀏覽或搜尋“建立SaaS的開發商店”
2. 找到適合的模板－“精益SaaS開發商店（CEO + CTO + 3名工程師）”
3. 閱讀描述，檢查組織架構圖，檢查註釋
4. 運行`paperclipai install cliphub:acme/lean-saas-shop`
5. Paperclip 在本地創建公司並預先配置所有智能體
6. 設定你的API鍵，調整預算，加入你的初始任務
7. 點擊開始

### “我創造了一些很棒的東西並想分享它”

1. 在 Paperclip 中建置並迭代一家公司，直到運作良好
2、出口：`paperclipai export --template my-agency`
3. 發布：`paperclipai publish cliphub my-agency`
4. 在Web UI上填寫描述、類別、標籤
5. 範本已上線 — 其他人可以找到並安裝它

### “我想改善別人的公司”

1. 在 ClipHub 上尋找最接近您需要的模板
2. 將其分叉到您的帳戶
3. 在本機安裝您的分叉，修改組織（新增智能體、更改配置、重組團隊）
4. 匯出並重新發佈為自己的變體
5. 分叉血統在原始版本和您的版本上均可見

### “我只需要一位優秀的智能體，而不是整個公司”

1. 在ClipHub中搜尋智能體模板：“高級Python工程師”
2. 找到一個加星號的智能體配置
3. 僅安裝此智能體：`paperclipai install cliphub:acme/senior-python-eng --agent`
4. 將其分配給現有公司的經理
5. 完成

## 與 Paperclip 的關係

使用 Paperclip **不需要** ClipHub。您可以完全從頭開始建立公司，而無需接觸 ClipHub。但 ClipHub 大大降低了進入門檻：- **新用戶** 在幾分鐘而不是幾小時內找到一家正在運營的公司
- **有經驗的使用者**與社群分享經過驗證的配置
- **生態系**複合－每個好的模板都能讓下一家公司更容易建立

ClipHub 之於 Paperclip 就像是包註冊表之於語言執行時期：可選，但具有變革性。

- [ ] 範本詳細資料頁面（描述、組織架構圖、智能體程式清單、安裝指令）

- [ ] 星星
- [ ] 下載計數
- [ ] 版本控制（semver、版本歷史記錄）
- [ ] 基本審核（社群報告、自動隱藏）

### V2

- [ ] 評論/線索討論
- [ ] 帶有血統追蹤的分叉
- [ ] 智能體與團隊子包

- 付費/高級模板（一切都是免費和公開的，至少在最初是這樣）
- 私有註冊表（可能是未來的企業功能）
- 在ClipHub上運行公司（它是一個註冊表，而不是一個運行時－與Paperclip自己的理念一致）
