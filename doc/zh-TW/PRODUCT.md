# Paperclip — 產品定義

## 它是什麼

Paperclip是自主AI公司的控制平面。 Paperclip 的一個實例可以運行多個公司。 **公司**是一階物件。

## 核心概念

### 公司

某公司有：

- **目標** — 它存在的原因（“創建排名第一的人工智能筆記應用程序，在 3 個月內實現 100 萬美元的 MRR”）
- **員工** — 每位員工都是人工智慧智能體
- **組織架構** — 誰向誰報告
- **收入和支出** — 在公司層面進行跟踪
- **任務層次結構** — 所有工作可追溯至公司目標

### 員工和智能體

每個員工都是智能體。當你創建一家公司時，你首先要定義首席執行官，然後從那裡開始發展。

每個員工都有：

- **適配器類型 + 配置** — 該智能體程式如何運作以及定義其身分/行為的內容。這是特定於適配器的（例如，OpenClaw 智能體程式可能使用 SOUL.md 和 HEARTBEAT.md 檔案；Claude Code 智能體程式可能使用 CLAUDE.md；裸腳本可能使用 CLI 參數）。 Paperclip 沒有規定格式－適配器規定了。
- **角色和報告** — 他們的頭銜、向誰報告、誰向他們報告
- **功能描述** - 簡短的段落，介紹該智能體的作用以及它們何時相關（幫助其他智能體髮現誰可以提供幫助）

例如：CEO 智能體的適配器配置告訴它在每次心跳時「審查您的高階主管正在做什麼，檢查公司指標，根據需要重新確定優先級，並分配新的策略計畫」。工程師的配置告訴它「檢查分配的任務，選擇最高優先級，然後執行它」。

然後，您定義誰向 CEO 報告：CTO 管理程式設計師，CMO 管理行銷團隊，等等。樹中的每個智能體程式都有自己的適配器配置。

### 智能體執行

運行智能體的心跳有兩種基本模式：

1. **執行指令** — Paperclip 啟動一個進程（shell 指令、Python 腳本等）並追蹤它。心跳是“執行這個並監視它”。
2. **觸發並忽略請求** — Paperclip 向外部運行的智能體程式發送 webhook/API 呼叫。心跳就是「通知這個智能體醒來」。 （OpenClaw 鉤子就是這樣運作的。）

我們提供合理的預設設定 - 預設智能體可以根據您的配置呼叫 Claude Code 或 Codex，記住會話 ID，運行基本腳本。但你可以插入任何東西。

### 任務管理

任務管理是分層的。任何時候，每項工作都必須透過父級任務鏈回溯到公司的頂層目標：

```
I am researching the Facebook ads Granola uses (current task)
  because → I need to create Facebook ads for our software (parent)
    because → I need to grow new signups by 100 users (parent)
      because → I need to get revenue to $2,000 this week (parent)
        because → ...
          because → We're building the #1 AI note-taking app to $1M MRR in 3 months
```任務有血統。每項任務都服務於父任務，直至實現公司目標。這就是讓自主智能體保持一致的原因——他們總是可以回答“我為什麼要這樣做？”

更詳細的任務結構待定。

## 原則

1. **對如何運行智能體沒有意見。 ** 您的智能體程式可以是 OpenClaw 機器人、Python 腳本、節點腳本、Claude Code 會話、Codex 實例 - 我們不在乎。 Paperclip 定義了通訊的控制平面並提供了心跳的實用基礎設施。它不強制要求智能體運行時。

2. **公司是組織單位。 **一切都在公司之下。一個Paperclip實例，多家公司。

3. **適配器配置定義智能體程式。 ** 每個智能體程式都有一個適配器類型和配置來控制其身分和行為。最低合約只是「可贖回」。

4. **一切工作都追隨目標。 ** 分層任務管理意味著沒有任何事情是孤立存在的。如果你無法解釋為什麼某項任務對公司目標很重要，那麼它就不應該存在。

5. **控制平面，而非執行平面。 ** Paperclip 進行編排。特工們跑到哪裡就跑到哪裡，打電話回家。

## 使用者流程（夢想場景）

1. 開啟Paperclip，創建一個新公司
2. 定義公司的目標：“創建排名第一的人工智慧筆記應用程序，3 個月內 MRR 100 萬美元”
3. 創建首席執行官
   - 選擇轉接器（例如，Claude Code 的製程轉接器、OpenClaw 的 HTTP 轉接器）
   - 設定適配器（智能體身份、循環行為、執行設定）
   - CEO提出策略分解→董事會批准
4. 定義CEO的報告：CTO、CMO、CFO等。
   - 每個都有自己的適配器配置和角色定義
5. 定義他們的報告：CTO 下的工程師、CMO 下的行銷人員等。
6. 設定預算，定義初始策略任務
7. 點擊執行－智能體開始心跳，公司開始運作

## 指南

Paperclip 必須支援兩種運作模式：

- `local_trusted`（預設）：單一用戶本地可信任部署，無登入摩擦
- `authenticated`：需要登入模式，同時支援私網和公網部署暴露策略

規範模式設計和指令期望位於 `doc/DEPLOYMENT-MODES.md` 中。
