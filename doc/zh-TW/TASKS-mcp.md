# 任务管理 MCP 界面

Paperclip任務管理系統的函數契約。定義
智能體程式（和外部工具）可透過 MCP 進行操作。參考
[TASKS.md](TASKS.md) 底层数据模型。

所有操作返回JSON。 ID 是 UUID。時間戳為 ISO 8601。
任何出現問題 `id` 的地方都接受問題識別碼（例如 `ENG-123`）
預計。

---

## 問題

### `list_issues`

列出并过滤工作区中的问题。

|参数|类型 |必填|笔记|
| ----------------- | -------- | -------- | ----------------------------------------------------------------------------------------------- |
| `query` |字串|沒有|跨標題和描述的自由文字搜尋 |
| `teamId` |字串|沒有|依團隊篩選 |
| `status` |字串|沒有|依特定工作流程狀態過濾 |
| `stateType` |字串|沒有|依州類別過濾：`triage`、`backlog`、`unstarted`、`started`、ZXQQQ00011QQ、Q
| `assigneeId` |字串|沒有|按受讓人（智能體 ID）過濾 |
| `projectId` |字串|沒有|依項目篩選 |
| `parentId` |字串|沒有|按父問題過濾（返回子問題）|
| `labelIds` |字串[] |沒有|過濾所有這些標籤的問題 |
| `priority` |數量 |沒有|依優先權過濾 (0-4) |
| `includeArchived` |布林 |沒有|包含存檔的問題。預設值： false |
| `orderBy` |字串|沒有| `created`、`updated`、`priority`、`due_date`。預設：`created` |
| `limit` |数量 |没有|最大结果。預設值：50 |
| `after` |字串|沒有| Cursor 正向分頁 |
| `before` |字串|沒有| Cursor 向後分頁 |

**返回：** `{ issues: Issue[], pageInfo: { hasNextPage, endCursor, hasPreviousPage, startCursor } }`

---### `get_issue`

透過 ID 或識別碼檢索單一問題，並擴展所有關係。

|參數|型別 |必填 |筆記|
| --------- | ------ | -------- | -------------------------------------------------- |
| `id` |字串|是的 | UUID 或人類可讀的識別碼（例如 `ENG-123`）|

**返回：** 完整的 `Issue` 對象，包括：

- `state`（擴展的工作流程狀態）
- `assignee`（擴充智能體，如果設定）
- `labels`（擴充標籤[]）
- `relations`（IssueRelation[] 以及擴充的相關問題）
- `children`（子問題摘要：id、識別碼、標題、狀態、受讓人）
- `parent`（摘要，如果這是一個子問題）
- `comments`（評論[]，最新的在前）

### `create_issue`

建立一個新問題。

|參數|型別 |必填 |筆記|
| ------------- | -------- | -------- | --------------------------------------------- |
| `title` |字串|是的 | |
| `teamId` |字串|是的 |問題所屬團隊 |
| `description` |字串|沒有|降價|
| `status` |字串|沒有|工作流程狀態。預設：團隊的預設狀態|
| `priority` |數量 |沒有| 0-4。預設值：0（無）|
| `estimate` |數量 |沒有|點估計|
| `dueDate` |字串|沒有| ISO 日期 |
| `assigneeId` |字串|沒有|智能體分配|
| `projectId` |字串|沒有|關聯項目|
| `milestoneId` |字串|沒有|專案內的里程碑|
| `parentId` |字串|沒有|父問題（使其成為子問題）|
| `goalId` |字串|沒有|連結的目標/目的 |
| `labelIds` |字串[] |沒有|要套用的標籤 |
| `sortOrder` |數量 |沒有|在視圖內訂購|

**傳回：** 使用計算欄位建立 `Issue` 物件（`identifier`、`createdAt` 等）

**副作用：**

- 如果設定了 `parentId`，則從父級繼承 `projectId`（除非明確提供）
- `identifier`是根據團隊金鑰+下一個序號自動產生的

### `update_issue`

更新現有問題。|參數|型別 |必填 |筆記|
| ------------- | -------- | -------- | -------------------------------------------- |
| `id` |字串|是的 | UUID 或識別碼 |
| `title` |字串|沒有| |
| `description` |字串|沒有| |
| `status` |字串|沒有|過渡到新的工作流程狀態 |
| `priority` |數量 |無| 0-4 |
| `estimate` |數量 |無| |
| `dueDate` |字串|沒有| ISO 日期，或 `null` 來清除 |
| `assigneeId` |字串|沒有|智能體 ID，或 `null` 取消分配 |
| `projectId` |字串|沒有|要從項目中刪除的項目 ID 或 `null` |
| `milestoneId` |字串|沒有|里程碑 id，或 `null` 清除 |
| `parentId` |字串|沒有| Reparent，或 `null` 升級為獨立版 |
| `goalId` |字串|沒有|目標 ID，或 `null` 取消連結 |
| `labelIds` |字串[] |沒有| **替換**所有標籤（非附加）|
| `teamId` |字串|沒有|轉移到不同的團隊|
| `sortOrder` |數量 |沒有|在視圖內訂購 |

**傳回：** 更新的 `Issue` 物件。

**副作用：**

- 將 `status` 變更為類別為 `started` 的狀態會設定 `startedAt`（如果尚未設定）
- 將 `status` 變更為 `completed` 設定 `completedAt`
- 將 `status` 變更為 `cancelled` 設定 `cancelledAt`
- 移至 `completed`/`cancelled` 並啟用子問題自動關閉以完成開啟的子問題
- 變更 `teamId` 會重新指派識別碼（例如 `ENG-42` → `DES-18`）；舊識別碼保留在 `previousIdentifiers` 中

## 問題關係

### `list_issue_relations`

列出问题的所有关系。

|參數|型別 |必填 |
| --------- | ------ | -------- |
| `issueId` |字串|是的 |

**返回：** `{ relations: IssueRelation[] }` -- 每個都有擴展的 `relatedIssue` 摘要（id、標識符、標題、狀態）。

---

### `create_issue_relation`

創建兩個問題之間的關係。

|參數|型別 |必填 |筆記|
| ---------------- | ------ | -------- | ---------------------------------------------- |
| `issueId` |字串|是的 |來源問題 |
| `relatedIssueId` |字符串|是的 |目标问题|
| `type` |字串|是的 | `related`、`blocks`、`blocked_by`、`duplicate` |

**返回：** 创建了 `IssueRelation` 对象。

**副作用：**

- `duplicate` 自動將來源問題轉換為已取消狀態
- 從 A->B 建立 `blocks` 隱式表示 B 是 `blocked_by` A（兩者
  查询任一问题时方向可见）

---
