# 發布工作樹支持

狀態：實驗性的，僅限運行時，尚未作為面向使用者的功能發布。

此分支包含問題範圍的工作樹所需的執行時間和播種工作：

- 專案執行工作空間政策支持
- 問題級執行工作區設置
- 用於隔離問題執行的 git worktree 實現
- 可選的基於命令的工作樹配置
- 種子工作樹修復了秘密金鑰相容性
- 種子專案工作區重新綁定到目前的 git 工作樹

我們特意不為此提供 UI。運行時程式碼仍然保留，但主要 UI 入口點目前已硬關閉。

## 今天什麼有效

- 專案可以在後端攜帶執行工作區策略
- 問題可以在後端攜帶執行工作區設置
- 心跳執行可以實現隔離的git工作樹
- 執行時期可以在衍生工作樹內執行專案定義的設定指令
- 種子工作樹實例可以保持本地加密的秘密工作
- 種子工作樹實例可以將相同的儲存庫專案工作空間路徑重新綁定到目前的 git 工作樹上

## 隱藏的 UI 入口點

這些是該功能目前面向使用者的 UI 介面，現已被有意停用：

- 項目設定：
  - `ui/src/components/ProjectProperties.tsx`
  - 執行工作區策略控制
  - git worktree 基礎引用/分支範本/父目錄
  - 提供/拆卸命令輸入

- 問題創建：
  - `ui/src/components/NewIssueDialog.tsx`
  - 孤立的問題結帳切換
  - 從專案策略預設問題執行工作區設置

- 問題編輯：
  - `ui/src/components/IssueProperties.tsx`
  - 問題等級工作區模式切換
  - 專案變更時預設問題執行工作區設置

- 智能體/運行時設定：
  - `ui/src/adapters/runtime-json-fields.tsx`
  - 運行時服務 JSON 字段，它是更廣泛的工作區運行時支援面的一部分

## 為什麼 UI 被隱藏

- 運行時行為仍在驗證中
- 工作流程和操作員人體工學不是最終的
- 我們不想在問題、項目或設定中公開部分不成熟的面向用戶的功能
