# 发布工作树支持

状态：实验性的，仅限运行时，尚未作为面向用户的功能发布。

该分支包含问题范围的工作树所需的运行时和播种工作：

- 项目执行工作空间政策支持
- 问题级执行工作区设置
- 用于隔离问题执行的 git worktree 实现
- 可选的基于命令的工作树配置
- 种子工作树修复了秘密密钥兼容性
- 种子项目工作区重新绑定到当前的 git 工作树

我们特意不为此提供 UI。运行时代码仍然保留，但主要 UI 入口点目前已硬关闭。

## 今天什么有效

- 项目可以在后端携带执行工作区策略
- 问题可以在后端携带执行工作区设置
- 心跳执行可以实现隔离的git工作树
- 运行时可以在派生工作树内运行项目定义的配置命令
- 种子工作树实例可以保持本地加密的秘密工作
- 种子工作树实例可以将相同的存储库项目工作空间路径重新绑定到当前的 git 工作树上

## 隐藏的 UI 入口点

这些是该功能当前面向用户的 UI 界面，现已被有意禁用：

- 项目设置：
  - `ui/src/components/ProjectProperties.tsx`
  - 执行工作区策略控制
  - git worktree 基础引用/分支模板/父目录
  - 提供/拆卸命令输入

- 问题创建：
  - `ui/src/components/NewIssueDialog.tsx`
  - 孤立的问题结帐切换
  - 从项目策略中默认问题执行工作区设置

- 问题编辑：
  - `ui/src/components/IssueProperties.tsx`
  - 问题级工作区模式切换
  - 项目更改时默认问题执行工作区设置

- 智能体/运行时设置：
  - `ui/src/adapters/runtime-json-fields.tsx`
  - 运行时服务 JSON 字段，它是更广泛的工作区运行时支持面的一部分

## 为什么 UI 被隐藏

- 运行时行为仍在验证中
- 工作流程和操作员人体工程学不是最终的
- 我们不想在问题、项目或设置中公开部分不成熟的面向用户的功能
