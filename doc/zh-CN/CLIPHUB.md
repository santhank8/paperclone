# ClipHub — 公司注册处

**下载公司。**

ClipHub 是人们共享、发现和下载 Paperclip 公司配置的公共注册表。公司模板是一种便携式工件，包含整个组织（智能体、报告结构、适配器配置、角色定义、种子任务），准备通过一个命令启动。

---

## 它是什么

ClipHub 之于 Paperclip 就像包注册表之于编程语言。 Paperclip 已经支持可导出的组织配置（请参阅 [SPEC.md](SPEC.md) §2）。 ClipHub 是这些导出所在的公共目录。

用户在 Paperclip 中建立了一家工作公司——一家开发商店、一家营销机构、一家研究实验室、一家内容工作室——导出模板，并将其发布到 ClipHub。任何人都可以在自己的 Paperclip 实例上浏览、搜索、下载和启动该公司。

标语：**您可以真正下载一家公司。**

---

## 发布内容

ClipHub 包是**公司模板导出** — Paperclip 规范中定义的便携式工件格式。它包含：

|组件|描述 |
|---|---|
| **公司元数据** |名称、描述、预期用例、类别 |
| **组织架构图** |完整的报告层次结构——谁向谁报告|
| **智能体定义** |每个智能体：姓名、角色、头衔、能力描述 |
| **适配器配置** |每个智能体适配器类型和配置（SOUL.md、HEARTBEAT.md、CLAUDE.md、处理命令、Webhook URL — 无论适配器需要什么）|
| **种子任务** |用于引导公司首次运行的可选启动任务和举措 |
| **预算默认值** |每个智能体和每个公司的建议代币/成本预算 |

模板是**结构，而不是状态。**没有正在进行的任务，没有历史成本数据，没有运行时工件。只是蓝图。

### 子包

并非每个用例都需要整个公司。 ClipHub 还支持发布单个组件：

- **智能体模板** — 单个智能体配置（例如“高级 TypeScript 工程师”、“SEO 内容编写者”、“DevOps 智能体”）
- **团队模板** — 组织架构图的子树（例如“营销团队：CMO + 3 份报告”、“工程 Pod：技术主管 + 4 名工程师”）
- **适配器配置** — 独立于任何特定智能体角色的可重用适配器配置

这些可以混入现有公司。下载智能体，将其放入您的组织，分配经理，然后开始。

## 核心特性

### 浏览和发现

主页从多个维度展示公司：

- **精选** — 编辑策划的高质量模板
- **流行** — 按下载量、星级和分叉排名
- **最近** — 最新发布或更新
- **类别** — 可按用例浏览（请参阅下面的类别）每个列表显示：名称、简短描述、组织规模（智能体数量）、类别、使用的适配器类型、星数、下载数量和迷你组织图表预览。

### 搜索

搜索是**语义性的，而不是仅限关键字的。** 由矢量嵌入提供支持，因此您可以按意图进行搜索：

- “运行 Facebook 广告的营销机构” → 找到相关的公司模板，即使标题中没有这些确切的词
- “构建 APIs 的小型开发团队”→ 找到精益工程组织
- “与作家和编辑的内容管道”→ 查找内容工作室模板

还支持按以下条件过滤：类别、智能体计数范围、适配器类型、星级计数、新近度。

### 公司详细信息页面

单击进入公司模板会显示：

- **完整描述** — 这家公司从事什么业务、如何运营、有何期望
- **交互式组织架构图** — 每个智能体的可视化树，包含角色、头衔和能力
- **智能体列表** — 每个智能体的可扩展详细信息（适配器类型、配置摘要、角色描述）
- **种子任务** — 包括启动计划和任务
- **预算概览** — 建议的成本结构
- **安装命令** — 一行CLI命令下载并创建
- **版本历史记录** — 变更日志、semver、以前可用的版本
- **社区** — 星星、评论、分叉计数

### 安装和分叉

使用模板的两种方法：

**安装（全新开始）：**
```
paperclip install cliphub:<publisher>/<company-slug>
```
下载模板并在本地 Paperclip 实例中创建一个新公司。您添加自己的 API 密钥、设置预算、自定义智能体，然后点击开始。

**叉子：**
Forking 在您自己的 ClipHub 帐户下创建模板的副本。您可以修改它，将其重新发布为您自己的变体，并跟踪分叉谱系。这使得渐进式改进成为可能——有人发布了一个营销机构，你分叉它，添加一个社交媒体团队，然后重新发布。

### 星星和评论

- **星星** — 书签和信号质量。星星数是主要的排名信号。
- **评论** — 对每个列表进行主题讨论。提出问题、分享结果、提出改进建议。

### 下载计数和信号

每次安装都会被计算在内。注册表跟踪：

- 总下载量（所有时间）
- 每个版本的下载量
- 货叉计数
- 星星计数

这些信号会影响搜索排名和发现。

## 出版

### 谁可以发布

任何拥有 GitHub 帐户的人都可以发布到 ClipHub。通过 GitHub OAuth 进行身份验证。

### 如何发布

从 Paperclip 中，将您的公司导出为模板，然后发布：

```
paperclip export --template my-company
paperclip publish cliphub my-company
```

或者使用 Web UI 直接上传模板导出。

### 你提供什么

发布时，您指定：|领域 |必填 |描述 |
|---|---|---|
| `slug` |是的 | URL 安全标识符（例如 `lean-dev-shop`）|
| `name` |是的 |显示名称 |
| `description` |是的 |这家公司是做什么的以及它的服务对象 |
| `category` |是的 |主要类别（见下文）|
| `tags` |没有|用于发现的附加标签 |
| `version` |是的 | Semver（例如 `1.0.0`）|
| `changelog` |没有|此版本有何变化 |
| `readme` |没有|扩展文档（降价）|
| `license` |没有|使用条款|

### 版本控制

模板使用语义版本控制。每次发布都会创建一个不可变的版本。用户可以安装任何版本或默认为`latest`。版本历史记录和变更日志在详细信息页面上可见。

### `sync` 命令

对于维护多个模板的高级用户：

```
paperclip cliphub sync
```

扫描本地导出的模板并发布任何新的或更新的模板。对于从单个存储库维护公司模板组合非常有用。

## 类别

公司模板按用例组织：

|类别 |示例 |
|---|---|
| **软件开发** |全栈开发商店，API 开发团队，移动应用工作室 |
| **营销与增长** |绩效营销机构、内容营销团队、SEO 店 |
| **内容与媒体** |内容工作室、播客制作、时事通讯运营 |
| **研究与分析** |市场研究公司、竞争情报、数据分析团队|
| **运营** |客户支持组织、内部运营团队、质量保证/测试中心 |
| **销售** |外向销售团队、潜在客户开发、客户管理 |
| **财务与法律** |簿记服务、合规监控、财务分析 |
| **创意** |设计机构、文案工作室、品牌开发|
| **通用** |入门模板、最小组织、单智能体设置 |

类别不是排他性的——模板可以有一个主要类别以及用于横切关注点的标签。

## 节制与信任

### 已验证的发布商

满足特定阈值（帐户年龄、已发布的具有良好信号的模板）的发布商将获得经过验证的徽章。经过验证的模板在搜索中排名更高。

### 安全审查

公司模板包含适配器配置，其中可能包括可执行命令（进程适配器）或 Webhook URL（HTTP 适配器）。审核系统：

1. **自动扫描** — 检查适配器配置是否存在可疑模式（任意代码执行、渗透 URL、凭据收集）
2. **社区报告** — 任何登录用户都可以标记模板。多个报告等待审核后自动隐藏。
3. **手动审核** — 版主可以批准、拒绝或请求更改

### 账户门控新帐户在发布之前有一段等待期。这可以防止路过的垃圾邮件。

## 架构

ClipHub 是一项独立于 Paperclip 本身的**独立服务**。 Paperclip是自托管的； ClipHub 是 Paperclip 实例与之通信的托管注册表。

### 整合点

|层 |角色 |
|---|---|
| **ClipHub 网络** |浏览、搜索、发现、评论、加星标——网站 |
| **ClipHub API** |注册表 API 用于以编程方式发布、下载、搜索 |
| **Paperclip CLI** | `paperclipai install`、`paperclipai publish`、`paperclipai cliphub sync` — 内置于 Paperclip |
| **Paperclip UI** | Paperclip Web UI 中的“浏览 ClipHub”面板，无需离开应用程序即可发现模板 |

### 技术堆栈

|层 |技术 |
|---|---|
|前端 | React + Vite（与Paperclip一致）|
|后端 | TypeScript + Hono（与Paperclip一致）|
|数据库| PostgreSQL |
|搜索 |用于语义搜索的向量嵌入 |
|授权 | GitHub OAuth |
|存储|存储在对象存储（S3 或同等存储）中的模板 zip |

### 数据模型（草图）

```
Publisher
  id, github_id, username, display_name, verified, created_at

Template
  id, publisher_id, slug, name, description, category,
  tags[], readme, license, created_at, updated_at,
  star_count, download_count, fork_count,
  forked_from_id (nullable)

Version
  id, template_id, version (semver), changelog,
  artifact_url (zip), agent_count, adapter_types[],
  created_at

Star
  id, publisher_id, template_id, created_at

Comment
  id, publisher_id, template_id, body, parent_id (nullable),
  created_at, updated_at

Report
  id, reporter_id, template_id, reason, created_at
```

## 用户流程

### “我想开一家公司”

1. 打开ClipHub，按类别浏览或搜索“构建SaaS的开发商店”
2. 找到适合的模板——“精益SaaS开发商店（CEO + CTO + 3名工程师）”
3. 阅读描述，检查组织架构图，检查注释
4. 运行`paperclipai install cliphub:acme/lean-saas-shop`
5. Paperclip 在本地创建公司并预配置所有智能体
6. 设置你的API键，调整预算，添加你的初始任务
7. 点击开始

### “我创造了一些很棒的东西并想分享它”

1. 在Paperclip中构建并迭代一家公司，直到运行良好
2、出口：`paperclipai export --template my-agency`
3. 发布：`paperclipai publish cliphub my-agency`
4. 在Web UI上填写描述、类别、标签
5. 模板已上线 — 其他人可以找到并安装它

### “我想改善别人的公司”

1. 在 ClipHub 上查找最接近您需要的模板
2. 将其分叉到您的帐户
3. 在本地安装您的分叉，修改组织（添加智能体、更改配置、重组团队）
4. 导出并重新发布为您自己的变体
5. 分叉血统在原始版本和您的版本上均可见

### “我只需要一位优秀的智能体人，而不是整个公司”

1. 在ClipHub中搜索智能体模板：“高级Python工程师”
2. 找到一个加星标的智能体配置
3. 仅安装该智能体：`paperclipai install cliphub:acme/senior-python-eng --agent`
4. 将其分配给现有公司的经理
5. 完成

## 与 Paperclip 的关系

使用 Paperclip **不需要** ClipHub。您可以完全从头开始创建公司，而无需接触 ClipHub。但 ClipHub 极大地降低了进入门槛：- **新用户** 在几分钟而不是几小时内即可找到一家正在运营的公司
- **有经验的用户**与社区分享经过验证的配置
- **生态系统**复合——每一个好的模板都能让下一家公司更容易建立

ClipHub 之于 Paperclip 就像包注册表之于语言运行时：可选，但具有变革性。

- [ ] 模板发布（通过CLI或网页上传）
- [ ] 模板浏览（列表、按类别过滤）
- [ ] 模板详细信息页面（描述、组织架构图、智能体列表、安装命令）
- [ ] 语义搜索（向量嵌入）

- [ ] GitHub OAuth认证
- [ ] 星星
- [ ] 下载计数
- [ ] 版本控制（semver、版本历史记录）
- [ ] 基本审核（社区报告、自动隐藏）

### V2

- [ ] 评论/线索讨论
- [ ] 带有血统跟踪的分叉
- [ ] 智能体和团队子包
- [ ] 已验证的发布商徽章

- [ ] `paperclipai cliphub sync` 用于批量发布
- [ ] 发行商简介和作品集

### 不在范围内

- 付费/高级模板（一切都是免费和公开的，至少在最初是这样）
- 私有注册表（可能是未来的企业功能）
- 在ClipHub上运行公司（它是一个注册表，而不是一个运行时——与Paperclip自己的理念一致）
