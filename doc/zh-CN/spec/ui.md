# Paperclip UI 规范

状态：草案
日期：2026-02-17

## 1. 设计理念

Paperclip的UI是一个专业级的控制平面，而不是一个玩具控制台。它应该感觉就像你整天使用的那种工具——快速、键盘驱动、信息密集而不混乱、默认为黑暗主题。每个像素都应该赢得自己的位置。

设计原则：

- **密集但可扫描。** 显示最大信息，无需点击即可显示。使用空格来分隔，而不是填充。
- **键盘优先。** 用于搜索 (Cmd+K)、新问题 (C)、导航的全局快捷键。高级用户应该很少修改鼠标。
- **上下文，非模态。** 通过对话框进行内联编辑。页面导航上的下拉菜单。用户的心理环境不应该被不必要地破坏。
- **默认深色主题。**中性灰色，而不是纯黑色。强调颜色很少用于表示状态和优先级。文本是主要的视觉元素。

### 颜色系统

- **背景：** `hsl(220, 13%, 10%)`（深木炭，不是纯黑色）
- **表面/卡：** `hsl(220, 13%, 13%)`
- **边框：** `hsl(220, 10%, 18%)`
- **主要短信：** `hsl(220, 10%, 90%)`
- **短信辅助：** `hsl(220, 10%, 55%)`
- **强调（互动）：** `hsl(220, 80%, 60%)`（柔和的蓝色）

状态颜色（所有实体一致）：
- **积压：** 灰色 `hsl(220, 10%, 45%)`
- **Todo：**灰蓝色 `hsl(220, 20%, 55%)`
- **进行中：** 黄色 `hsl(45, 90%, 55%)`
- **评论中：** 紫罗兰 `hsl(270, 60%, 60%)`
- **完成：** 绿色 `hsl(140, 60%, 50%)`
- **取消：** 灰色 `hsl(220, 10%, 40%)`
- **已阻止：** 琥珀色 `hsl(25, 90%, 55%)`

优先指标：
- **关键：** 红色圆圈，已填充
- **高：** 橙色圆圈，半填充
- **中：** 黄色圆圈，轮廓
- **低：** 灰色圆圈、轮廓、虚线

### 版式

- **字体：**系统字体堆栈（如果加载则为Inter，否则为`-apple-system, BlinkMacSystemFont, 'Segoe UI'`）
- **正文：** 13px / 1.5 行高
- **标签/元数据：** 11px / 大写跟踪
- **标题：** 14-18px / 半粗体，切勿全部大写

### 图标

始终使用 `lucide-react`。每个侧边栏项目、每个状态指示器、每个操作按钮都应该有一个图标。图标在导航中为 16 像素，在行中为 14 像素。

## 2. 应用程序外壳

该应用程序采用三区域布局：

```
┌──────────┬────────────────────────────────────────────────┐
│          │  Breadcrumb bar                                │
│ Sidebar  ├──────────────────────────┬─────────────────────┤
│ (240px)  │  Main content            │  Properties panel   │
│          │  (flex-1)                │  (320px, optional)  │
│          │                          │                     │
└──────────┴──────────────────────────┴─────────────────────┘
```

- **侧边栏：** 固定在左侧，240 像素。通过切换或键盘快捷键可折叠为仅图标（48px）。
- **面包屑栏：** 跨越主+属性上方的整个宽度。显示导航路径、实体操作和视图控件。
- **主要内容：** 可滚动。包含主要视图（列表、详细信息、图表等）。
- **属性面板：** 右侧，320 像素。显示在详细视图上（问题详细信息、项目详细信息、智能体详细信息）。隐藏在列表视图和控制台上。可调整大小。

当您单击详细视图时，属性面板会滑入；当您返回列表时，属性面板会滑出。它不是侧边栏——它与所选实体相关。

## 3. 侧边栏侧边栏是主要导航。它被分组为具有可折叠标题的逻辑部分。

### 3.1 公司标题

侧边栏顶部。始终可见。

```
┌─────────────────────────┐
│ [icon] Acme Corp      ▼ │  ← Company switcher dropdown
├─────────────────────────┤
│ [🔍]  [✏️]              │  ← Search + New Issue
└─────────────────────────┘
```

**公司切换器**是一个下拉按钮，占据侧边栏标题的整个宽度。它显示：
- 公司图标（带有公司颜色的首字母头像，或上传的图标）
- 公司名称（如果过长则用省略号截断）
- V 形向下图标

单击打开一个下拉菜单：
- 所有公司列表（带有状态点：绿色=活动，黄色=暂停，灰色=存档）
- 下拉列表顶部的搜索字段（适用于拥有许多公司的用户）
- 分频器
- `+ Create company` 底部操作

公司名称下方，有一排图标按钮：
- **搜索**（放大镜图标）- 打开 Cmd+K 搜索模式
- **新问题**（铅笔/方笔图标）- 在当前公司环境中打开新问题模式

### 3.2 个人部分

没有部分标题 - 这些部分始终位于顶部，公司标题下方。

```
  Inbox                    3
  My Issues
```

- **收件箱** — 需要董事会操作员注意的项目。徽章计数在右侧。包括：待批准、预算警报、失败的检测信号。该数字是未读/未解决的总数。
- **我的问题** — 由董事会操作员创建或分配给董事会操作员的问题。

### 3.3 工作部分

节标题：**工作**（可折叠，带有 V 形开关）

```
  Work                     ▼
    Issues
    Projects
    Goals
    Views
```

- **问题** — 所选公司的主要任务列表。这是主力视图。
- **项目** — 项目列表。项目对问题进行分组并与目标联系起来。
- **目标** — 公司目标层次结构。
- **视图** — 保存的过滤/排序配置（例如，“关键错误”、“未分配的任务”、“CEO 的任务”）。用户可以在此处创建、命名和固定自定义视图。

### 3.4 公司部分

节标题：**公司**（可折叠）

```
  Company                  ▼
    Dashboard
    Org Chart
    Agents
    Costs
    Activity
```

- **控制台** — 公司健康状况概览：智能体状态、任务速度、成本消耗、待审批计数。
- **组织架构图** — 智能体报告层次结构的交互式树形可视化。
- **智能体** — 所有智能体的简单列表，包括状态、角色、最后一次心跳、支出。
- **成本** — 成本控制台，按智能体、项目、模型、时间进行细分。
- **活动** — 所有系统事件的审核日志。

注意：批准没有顶级侧边栏条目。它们通过**收件箱**（主要交互点）、**控制台**（待处理计数指标）和**实体页面上的内联**（例如，智能体详细信息页面显示授权其雇用的批准）来显示。 `/approvals` 路由仍然存在，并且可以通过收件箱和控制台中的“查看所有批准”链接访问，但它不在侧边栏导航中。

### 3.5 部分行为- 每个部分标题均可单击以折叠/展开其子项。
- 折叠状态保留在 localStorage 中。
- 活动导航项以左边框强调和背景色调突出显示。
- 将鼠标悬停在导航项上会显示微妙的背景突出显示。
- 徽章计数右对齐，呈现为小药丸（例如，圆角矩形中的 `3`）。
- 图标为 16 像素，左对齐，与标签文本有 8 像素间隙。

### 3.6 侧边栏图标

每个导航项都有一个独特的图标（lucide-react）：

|项目 |图标|
|------|------|
|收件箱 | `Inbox` |
|我的问题 | `CircleUser` |

|项目 | `Hexagon` |

## 4. 面包屑导航栏

面包屑栏位于主要内容和属性面板上方。它既充当导航又充当上下文指示器。

### 4.1 结构

```
┌─────────────────────────────────────────────────────────────────────┐
│ Projects › Paperclip › Issues › CLIP-42  [⭐] [···]     [🔔] [⬜] │
└─────────────────────────────────────────────────────────────────────┘
```

**左侧：**
- 面包屑片段，由 `›` V 形分隔。
- 每个部分都可以单击以导航到该级别。
- 当前片段是不可点击的、稍微粗体的文本。
- 星形图标可收藏/固定当前实体。
- 用于实体操作的三点菜单（删除、存档、复制、复制链接等）

**右侧：**
- 通知铃声（如果在详细视图中 - 订阅该实体的更改）
- 面板切换（显示/隐藏右侧属性面板）

### 4.2 特定于视图的选项卡

在某些详细信息页面上，面包屑栏还包含面包屑下方的选项卡行：

**项目详情：**
```
  Overview    Updates    Issues    Settings
```

**智能体详细信息：**
```
  Overview    Heartbeats    Issues    Costs
```

选项卡呈现为药丸状按钮。活动选项卡具有微妙的背景填充。

## 5. 问题（任务管理）

问题是核心工作单元。本节详细介绍了完整的问题体验。

### 5.1 问题列表视图

单击侧边栏中的“问题”时，问题列表是默认视图。

**布局：**
```
┌─────────────────────────────────────────────────────────────────┐
│ [All Issues] [Active] [Backlog]  [⚙ Settings]    [≡ Filter]  [Display ▼] │
├─────────────────────────────────────────────────────────────────┤
│ ▼ Todo                                                3    [+] │
│ ☐ --- CLIP-5  ○ Implement user auth          @CTO    Feb 16  │
│ ☐ --- CLIP-3  ○ Set up CI pipeline           @DevOps Feb 16  │
│ ☐ --- CLIP-8  ○ Write API documentation      @Writer Feb 17  │
│                                                                 │
│ ▼ In Progress                                         2    [+] │
│ ☐ !!! CLIP-1  ◐ Build landing page           @FE     Feb 15  │
│ ☐ --- CLIP-4  ◐ Database schema design       @CTO    Feb 14  │
│                                                                 │
│ ▼ Backlog                                             5    [+] │
│ ☐ --- CLIP-9  ◌ Research competitors                  Feb 17  │
│ ...                                                             │
└─────────────────────────────────────────────────────────────────┘
```

**顶部工具栏：**
- **状态选项卡：** `All Issues`、`Active`（待办事项 + 进行中 + 审核中 + 已阻止）、`Backlog`。每个选项卡显示一个状态图标和计数。活动选项卡已填充，其他选项卡已概述。
- **设置齿轮：** 配置问题显示默认值、自定义字段。
- **过滤器按钮：** 打开工具栏下方的过滤器栏。
- **显示下拉菜单：** 在分组模式（按状态、按优先级、按受让人、按项目、无）和布局模式（列表、看板/看板）之间切换。

**分组：**
- 默认情况下，问题按状态分组（与参考屏幕截图匹配）。
- 每个组标题显示：折叠 V 形、状态图标、状态名称、计数以及用于在该状态下创建新问题的 `+` 按钮。
- 组是可折叠的。折叠的组仅显示带有计数的标题。**问题行：**
每行从左到右包含：
1. **复选框** — 用于批量选择。默认隐藏，悬停时显示（优先级左侧）。
2. **优先级指示器** — 代表严重/高/中/低的图标（参见上面的颜色系统）。始终可见。
3. **问题密钥** — 例如，`CLIP-5`。等宽、柔和的颜色。前缀源自项目（如果没有项目则源自公司）。
4. **状态圈** — 可单击以打开状态更改下拉列表（与参考屏幕截图相同）。圆圈的填充/颜色反映当前状态。
5. **标题** — 主要文本，如果太长则用省略号截断。
6. **受让人** — 头像（智能体图标）+ 智能体名称，右对齐。如果未分配，则显示虚线圆圈占位符。
7. **日期** — 创建日期或目标日期，静音文本，最右侧。

**行交互：**
- 单击行 → 导航至问题详细信息视图。
- 单击状态圆圈 → 打开内联状态下拉菜单（待办事项、待办事项、进行中、审核中、完成、已取消），并使用键盘数字作为快捷键 (1-6)。
- 单击复选框 → 选择批量操作。选中任何复选框后，列表底部会出现一个批量操作栏。
- 悬停 → 显示复选框，行会获得微妙的背景突出显示。
- 右键单击​​ → 上下文菜单（与三点菜单相同的操作）。

**批量操作栏：**
当选择一个或多个问题时，底部会出现一个浮动栏：
```
┌─────────────────────────────────────────────────────────┐
│  3 selected    [Status ▼] [Priority ▼] [Assignee ▼] [Project ▼]  [🗑 Delete]  [✕ Cancel] │
└─────────────────────────────────────────────────────────┘
```

### 5.2 问题过滤栏

单击“过滤器”会在工具栏下方显示一个过滤器栏：

```
┌─────────────────────────────────────────────────────────┐
│ [+ Add filter]  Status is Todo, In Progress  [×]        │
│                 Priority is Critical, High    [×]        │
│                 Assignee is CTO-Agent         [×]        │
└─────────────────────────────────────────────────────────┘
```

- 每个过滤器都是一个显示 `field operator value` 的芯片。
- 单击一个芯片即可对其进行编辑。
- `×` 删除过滤器。
- `+ Add filter` 打开可用字段的下拉列表：状态、优先级、受让人、项目、目标、创建日期、标签、创建者。
- 过滤器由 AND 组成。
- 活动过滤器保​​留在 URL 查询字符串中，因此可以共享/可添加书签。

#### 中间窗格（主要内容）

**标题区域：**
- 问题标题，大（18px，半粗体），点击可编辑（内联编辑）。
- 字幕：以静音文本发出密钥 `CLIP-42`。
- 标题下方：内联属性栏将关键属性显示为可点击的芯片（与参考屏幕截图相同的模式）：`[○ In Progress] [!!! High] [👤 CTO] [📅 Target date] [📁 Auth] [···]`。每个芯片都可以单击来更改内联属性。

**描述：**
- Markdown 渲染的描述。
- 单击编辑 — 就地打开 Markdown 编辑器。
- 支持标题、列表、代码块、链接、图像。

**子任务（如果有）：**
- 下面列出了作为可折叠部分的描述。
- 每个子任务都是一个小问题行（状态圈+标题+受让人）。
- `+ Add subtask` 按钮位于底部。**评论：**
- 按时间顺序排列的评论列表。
- 每条评论显示：作者头像/图标、作者姓名、时间戳、正文（降价渲染）。
- 底部的评论输入 — 一个支持 Markdown 的文本区域和一个“评论”按钮。
- 智能体的评论显示机器人图标；董事会的评论显示一个用户图标。

**标题：**“属性”标签，带有 `+` 按钮，用于添加自定义字段。

|标签|多选标签输入 |

|开始日期 |日期选择器 |

**活动部分：**
- 带有“查看全部”链接的“活动”标题。
- 最近事件的紧凑时间表：状态更改、分配更改、评论等。
- 每个条目：图标+描述+相对时间戳。

### 5.4 新问题模态

由侧边栏铅笔图标、键盘快捷键 `C` 或问题列表中的 `+` 按钮触发。

```
┌─────────────────────────────────────────────────────────┐
│ [📁 CLIP] › New issue               [Save as draft] [↗] [×] │
├─────────────────────────────────────────────────────────┤
│                                                         │
│ Issue title                                             │
│ ___________________________________________________     │
│                                                         │
│ Add a description...                                    │
│                                                         │
│                                                         │
│                                                         │
│                                                         │
├─────────────────────────────────────────────────────────┤
│ [○ Todo] [--- Priority] [👤 Assignee] [📁 Project]     │
│ [🏷 Labels] [···]                                       │
├─────────────────────────────────────────────────────────┤
│ [📎]                    [◻ Create more] [Create issue]  │
└─────────────────────────────────────────────────────────┘
```

**顶栏：**
- 显示上下文的面包屑：项目密钥（或公司密钥）`›`“新问题”。
- “另存为草稿”按钮。
- 展开图标（作为整页而不是模态打开）。
- 关闭`×`。

**身体：**
- 标题字段：大输入，占位符“问题标题”。自动对焦打开。
- 描述：下面的 Markdown 编辑器，占位符“添加描述...”。可扩展。

**财产筹码（底栏）：**
- 紧凑的属性按钮行。每个都会打开一个下拉菜单来设置该属性。
- 显示的默认选项：状态（默认为待办事项）、优先级、受让人、项目、标签。
- `···` 更多按钮显示：目标、开始日期、目标日期、帐单代码、父问题。

**页脚：**
- 附件按钮（Paperclip图标）。
- “创建更多”切换 — 打开后，创建问题会清除表单并保持打开状态以供快速输入。
- “创建问题”主按钮。

**行为：**
- `Cmd+Enter` 提交表格。
- 如果从项目上下文中打开，则该项目已预先填充。
- 如果从特定状态组的 `+` 按钮打开，则该状态已预先填充。
- slug/key 是根据项目前缀 + 递增数字（显示在面包屑中）自动生成的。

### 5.5 问题板视图（看板）

可通过“显示”下拉菜单→“板布局”进行访问。

列代表状态：待办事项 |都都 |进行中 |回顾中 |完毕每张卡片显示：
- 发出键（静音）
- 标题（主要文本）

## 11. 批准（根据上下文，而不是独立的）

批准是治理之门——董事会必须做出的决定（聘请智能体人、批准首席执行官战略）。它们不是工作项目。他们的数据模型与问题（不同的状态机、副作用触发器、非结构化有效负载）保持分离。但他们不需要自己的顶级导航条目。

### 11.1 批准的体现

**1.收件箱（主要）。** 待批准是最高优先级的收件箱项目。董事会操作员可以通过内联批准/拒绝操作看到它们的前面和中间（参见第 14 节）。

**2.控制台指标卡。**“待批准”卡显示计数以及完整批准列表的链接。

**3.内嵌在实体页面上。** 通过批准创建实体时，详细信息页面会显示上下文横幅：
- 智能体详细信息页面：`"Hired via approval — requested by CEO on Feb 15"`，带有审批记录链接。
- 处于 `pending` 状态（尚未创建）的智能体可能会显示：`"Pending approval — requested by CEO"`，并内联批准/拒绝操作。**4.活动日志。** 批准事件（创建、批准、拒绝）与任何其他事件一样显示在活动时间线中。

### 11.2 审批列表页面（`/approvals`）

### 14.2 收件箱类别

项目按类别分组，最可操作的项目排在前面：

**待批准**（最高优先级）。每个审批项目显示：
- 盾牌图标+批准类型+标题
- 请求者+相对时间戳
- 关键负载摘要（1 行 — 智能体名称/招聘角色、策略计划标题）
- 内联 **[批准]** 和 **[拒绝]** 按钮，用于简单批准 (hire_agent)。单击“批准/拒绝”会显示一个简短的确认信息，并带有可选的决策注释字段。
- **[查看详细信息 →]** 复杂批准 (approve_ceo_strategy) 的链接，需要在做出决定之前进行全面审查。
- 类别标题中的“查看所有批准 →”链接导航至 `/approvals`。

**警报。** 智能体错误（心跳失败、错误状态）和预算警报（智能体或公司接近 80% 或 100% 限制）。每个链接到相关智能体或成本页面。**过时的工作。** `in_progress` 或 `todo` 中没有活动（没有评论，没有状态更改）超出可配置阈值（默认值：24 小时）的任务。每个都显示问题键、标题以及自上次活动以来的时间。单击导航到问题。

### 14.3 收件箱行为

- 未读项目的左侧有一个实心蓝点指示器。
- 单击一个项目将其标记为已读。
- 一旦批准/拒绝，批准将从收件箱中消失（它们转为已解决状态）。
- 当根本情况得到解决（智能体恢复、预算增加）时，警报就会消失。
- 侧边栏徽章计数反映了未解决的收件箱项目总数。
- 收件箱是根据实时数据（待批准查询+警报条件）计算的，而不是单独的通知表。这使得 V1 变得简单。

| `S` |切换所选问题的状态 |
| `X` |切换复选框选择|
| `Cmd+A` |全选（在列表上下文中）|

## 17. 响应行为

- **>1400px：** 完整的三窗格布局（侧边栏+主+属性）。
- **1024-1400px：** 侧边栏折叠为图标。通过切换可用的属性面板。
- **<1024px：** 侧边栏隐藏（汉堡菜单）。属性面板隐藏（切换或选项卡）。

属性面板始终是可关闭的——它永远不应该阻挡主要内容。

## 18. 空状态

每个列表视图都应该有一个深思熟虑的空状态：

- **没有问题：**“还没有问题。创建您的第一期以开始跟踪工作。”带有 `[Create issue]` 按钮。
- **无智能体：** “本公司没有智能体。创建一个智能体来开始组建您的团队。”带有 `[Create agent]` 按钮。
- **未选择公司：**“选择一家公司开始。”使用公司切换器或 `[Create company]` 按钮。

空状态应该使用柔和的插图（简单的线条艺术，而不是卡通）和单一的号召性用语。

## 19. 加载和错误状态- **加载：** 骨架占位符与预期内容的布局相匹配（不是旋转器）。骷髅块带有微妙的微光。
- **错误：** 带有重试按钮的内联错误消息。除非应用程序本身损坏，否则永远不会出现整页错误。
- **冲突 (409)：** Toast 通知：“此问题已由其他用户更新。刷新即可查看更改。”通过[刷新]操作。
- **乐观更新：** 状态更改和属性编辑应立即在 UI 中更新，并在失败时回滚。
