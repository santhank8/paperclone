# Paperclip — 产品定义

## 它是什么

Paperclip是自主AI公司的控制平面。 Paperclip 的一个实例可以运行多个公司。 **公司**是一阶对象。

## 核心概念

### 公司

某公司有：

- **目标** — 它存在的原因（“创建排名第一的人工智能笔记应用程序，在 3 个月内实现 100 万美元的 MRR”）
- **员工** — 每个员工都是人工智能智能体
- **组织结构** — 谁向谁报告
- **收入和支出** — 在公司层面进行跟踪
- **任务层次结构** — 所有工作均可追溯到公司目标

### 员工和智能体人

每个员工都是智能体人。当你创建一家公司时，你首先要定义首席执行官，然后从那里开始发展。

每个员工都有：

- **适配器类型 + 配置** — 该智能体如何运行以及定义其身份/行为的内容。这是特定于适配器的（例如，OpenClaw 智能体可能使用 SOUL.md 和 HEARTBEAT.md 文件；Claude Code 智能体可能使用 CLAUDE.md；裸脚本可能使用 CLI 参数）。 Paperclip 没有规定格式——适配器规定了。
- **角色和报告** — 他们的头衔、向谁汇报、谁向他们汇报
- **功能描述** - 简短的段落，介绍该智能体的作用以及它们何时相关（帮助其他智能体发现谁可以提供帮助）

示例：CEO 智能体的适配器配置告诉它在每次心跳时“审查您的高管正在做什么，检查公司指标，根据需要重新确定优先级，分配新的战略计划”。工程师的配置告诉它“检查分配的任务，选择最高优先级，然后执行它”。

然后，您定义谁向 CEO 汇报：CTO 管理程序员，CMO 管理营销团队，等等。树中的每个智能体都有自己的适配器配置。

### 智能体执行

运行智能体的心跳有两种基本模式：

1. **运行命令** — Paperclip 启动一个进程（shell 命令、Python 脚本等）并跟踪它。心跳是“执行这个并监视它”。
2. **触发并忽略请求** — Paperclip 向外部运行的智能体发送 webhook/API 调用。心跳就是“通知这个智能体醒来”。 （OpenClaw 钩子就是这样工作的。）

我们提供合理的默认设置 - 默认智能体可以根据您的配置调用 Claude Code 或 Codex，记住会话 ID，运行基本脚本。但你可以插入任何东西。

### 任务管理

任务管理是分层的。任何时候，每一项工作都必须通过父级任务链回溯到公司的顶层目标：

```
I am researching the Facebook ads Granola uses (current task)
  because → I need to create Facebook ads for our software (parent)
    because → I need to grow new signups by 100 users (parent)
      because → I need to get revenue to $2,000 this week (parent)
        because → ...
          because → We're building the #1 AI note-taking app to $1M MRR in 3 months
```任务有血统。每项任务都服务于父任务，直至实现公司目标。这就是让自主智能体保持一致的原因——他们总是可以回答“我为什么要这样做？”

## 原则

1. **对如何运行智能体没有意见。** 您的智能体可以是 OpenClaw 机器人、Python 脚本、节点脚本、Claude Code 会话、Codex 实例 - 我们不在乎。 Paperclip 定义了通信的控制平面并提供了心跳的实用基础设施。它不强制要求智能体运行时。

2. **公司是组织单位。**一切都在公司之下。一个Paperclip实例，多家公司。

3. **适配器配置定义智能体。** 每个智能体都有一个适配器类型和配置来控制其身份和行为。最低合同只是“可赎回”。

4. **一切工作都追随目标。** 分层任务管理意味着没有任何事情是孤立存在的。如果你无法解释为什么某项任务对公司目标很重要，那么它就不应该存在。

5. **控制平面，而非执行平面。** Paperclip 进行编排。特工们跑到哪里就跑到哪里，并打电话回家。

## 用户流程（梦想场景）

1. 打开Paperclip，创建一个新公司
2. 定义公司的目标：“创建排名第一的人工智能笔记应用程序，3 个月内 MRR 100 万美元”
3. 创建首席执行官
   - 选择适配器（例如，Claude Code 的过程适配器、OpenClaw 的 HTTP 适配器）
   - 配置适配器（智能体身份、循环行为、执行设置）
   - CEO提出战略分解→董事会批准
4. 定义CEO的报告：CTO、CMO、CFO等。
   - 每个都有自己的适配器配置和角色定义
5. 定义他们的报告：CTO 下的工程师、CMO 下的营销人员等。
6. 设定预算，定义初始战略任务
7. 点击运行——智能体开始心跳，公司开始运转

## 指南

Paperclip 必须支持两种运行模式：

- `local_trusted`（默认）：单用户本地可信部署，无登录摩擦
- `authenticated`：需要登录模式，同时支持私网和公网部署暴露策略

规范模式设计和命令期望位于 `doc/DEPLOYMENT-MODES.md` 中。
