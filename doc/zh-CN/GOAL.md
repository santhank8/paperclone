# Paperclip

**Paperclip 是自主经济的支柱。** 我们正在构建自主 AI 公司运行的基础设施。我们的目标是让由 Paperclip 驱动的公司共同创造出能与世界最大国家 GDP 媲美的经济产出。我们所做的每一个决定都应该服务于此：使自主公司更有能力、更易于治理、更具可扩展性，并且更加真实。

## 愿景

自主公司——具有真实结构、治理和问责制的 AI 工作者组织——将成为全球经济的主要力量。不是一家公司，而是成千上万、数以百万计的。一个完全通过 Paperclip 协调，运行在 AI 劳动力之上的全新经济层。

Paperclip 本身不是这些公司。Paperclip 是使这些公司成为可能的基础设施。我们是控制平面 (control plane)、神经系统和操作层。每一个自主公司都需要结构、任务管理、成本控制、目标对齐以及人类的治理。这就是我们提供的。对于自主公司而言，我们就像是人类公司的企业操作系统——只不过这一次，操作系统是真正的软件，而不仅仅是个比喻。

衡量我们成功的标准不是某一家公司是否能运作，而是 Paperclip 是否能成为自主公司构建时的默认基础，以及这些公司共同形成的力量能否成为足以媲美国家产出的强大经济体。

## 待解决的问题

传统的任务管理软件远远不够。当你的整个劳动力都是 AI 代理时，你需要的不仅仅是一个待办事项列表——你需要的是整个公司的**控制平面 (control plane)**。

## 这到底是什么

Paperclip 是一个 AI 代理公司的指挥、通信和控制平面。它是你唯一需要的地方，用于：

- **将代理作为员工进行管理** —— 雇佣、组织并追踪谁在做什么
- **定义组织架构** —— 代理们自己在其中运作的组织结构图 (org charts)
- **实时追踪工作** —— 随时查看每个代理正在从事的工作
- **控制成本** —— 每个代理的 Token 薪酬预算、支出跟踪和燃烧率 (burn rate)
- **目标对齐** —— 代理能看到他们的工作如何服务于更大的使命
- **存储公司知识** —— 组织的共享大脑

## 架构

分为两层：

### 1. 控制平面 (当前软件)

中枢神经系统。管理以下内容：

- 代理注册表和组织结构图
- 任务分配和状态
- 预算和 Token 支出跟踪
- 公司知识库
- 目标层级 (公司 → 团队 → 代理 → 任务)
- 心跳监控 (Heartbeat) —— 了解代理何时在运行、何时空闲以及何时卡住

### 2. 执行服务 (适配器 Adapters)

代理在外部运行并向控制平面汇报。代理本质上就是被启动并执行工作的 Python 代码等。适配器负责连接不同的执行环境：

- **OpenClaw** —— 最初的适配器目标
- **心跳循环 (Heartbeat loop)** —— 简单的自定义 Python 脚本，它循环、签到并执行工作
- **其他** —— 任何可以调用 API 的运行时

控制平面本身**不运行**代理，它**编排**代理。代理在它们各自的环境中运行并“打电话回家 (phone home)”。

## 核心原则

你应该能够看着 Paperclip 就能一目了然地了解整个公司的状况——谁在做什么，花费了多少钱，以及它是否正在顺利运作。
