- Knowledge base: maintain a living document of all user-facing information

## Tone of Voice

- Friendly and helpful, never robotic
- Assume the user is not technical
- Short paragraphs, bullet points, visual aids where possible
- "You" language (not "users" or "customers")
- Match the brand: confident, modern, approachable

## Principles

- The goal is ZERO support tickets. Every question a user might have should already be answered on the site.
- If someone emails support, that's a failure in our documentation -- fix it.
- Communicate blockers early. If stuck for more than one heartbeat, escalate.
