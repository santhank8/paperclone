## 什么是 Paperclip?

# 面向“零人工公司”的开源编排引擎

**如果说 OpenClaw 是“员工”，那么 Paperclip 就是“公司”**

Paperclip 是一个基于 Node.js 服务器和 React UI 的系统，用于编排 AI Agent 团队来运行业务。你可以引入自己的 Agent，分配目标，并在一个仪表盘中跟踪 Agent 的工作内容和成本。

它看起来像一个任务管理器 —— 但在底层，它拥有组织架构、预算控制、治理机制、目标对齐和 Agent 协同能力。

**管理业务目标，而非 Pull Request。**

|        | 步骤            | 示例                                                            |
| ------ | --------------- | ------------------------------------------------------------------ |
| **01** | 定义目标 | _“构建排名第一的 AI 笔记应用，实现 100 万美元月经常性收入 (MRR)。”_ |
| **02** | 雇佣团队 | CEO、CTO、工程师、设计师、市场人员 —— 任何机器人、任何提供商。 |
| **03** | 批准并运行 | 审查策略。设置预算。点击运行。从仪表盘进行监控。 |

<br/>

> **即将推出：Clipmart** —— 一键下载并运行整个公司。浏览预设的公司模板（完整的组织架构、Agent 配置和技能），并在几秒钟内导入到你的 Paperclip 实例中。

## Paperclip 适合你，如果：

- ✅ 你想构建 **自主运行的 AI 公司**
- ✅ 你需要 **协调多个不同的 Agent**（OpenClaw, Codex, Claude, Cursor）来达成共同目标
- ✅ 你同时开启了 **20 个 Claude Code 终端** 却无法追踪每个人的进度
- ✅ 你希望 Agent **24/7 自主运行**，但仍想在必要时审计工作并参与介入
- ✅ 你想 **监控成本** 并强制执行预算
- ✅ 你希望管理 Agent 的流程 **像使用任务管理器一样直观**
- ✅ 你想通过 **手机** 管理你的自主业务

## Paperclip 解决的痛点

| 没有 Paperclip                                                                                                                     | 使用 Paperclip                                                                                                                         |
| ------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------- |
| ❌ 你开了 20 个 Claude Code 标签页，无法追踪谁在做什么。重启后一切丢失。 | ✅ 任务基于工单，对话采用线程模式，会话在重启后依然持久化。 |
| ❌ 你需要手动从多个地方收集上下文，以提醒机器人你到底在做什么。 | ✅ 上下文从任务流向项目和公司目标 —— 你的 Agent 始终明白目标与意义。 |
| ❌ Agent 配置文件夹杂乱无章，你还在重复发明任务管理、通信和协作机制。 | ✅ Paperclip 开箱即用提供组织架构、工单、委派和治理 —— 让你运行公司，而非一堆脚本。 |
| ❌ 陷入死循环的 Agent 浪费了数百美元的额度，在你发现之前就刷爆了配额。 | ✅ 成本追踪实时显示预算，并在用尽时限制 Agent。管理层通过预算决定优先级。 |
| ❌ 你有重复性工作（客服、社交媒体、报告），必须记住手动启动它们。 | ✅ 心跳机制按计划处理常规工作。管理层负责监督。 |
| ❌ 你有一个想法，必须找到仓库，启动 Claude Code，保持标签页开启并盯着它。 | ✅ 在 Paperclip 中添加任务。你的编程 Agent 会一直工作直到完成。管理层审查成果。 |

## 为什么 Paperclip 与众不同

Paperclip 正确处理了编排中的硬核细节。

|                                   |                                                                                                               |
| --------------------------------- | ------------------------------------------------------------------------------------------------------------- |
| **原子级执行** | 任务签出和预算执行是原子的，确保没有重复劳动和意外支出。 |
| **持久化 Agent 状态** | Agent 在心跳之间恢复相同的任务上下文，而非从头开始。 |
| **运行时技能注入** | Agent 可以在运行时学习 Paperclip 工作流和项目上下文，无需重新训练。 |
| **带回滚的治理** | 强制执行审批门槛，配置更改具备版本控制，错误更改可安全回滚。 |
| **目标感知执行** | 任务携带完整的业务目标溯源，让 Agent 始终看到“原因”，而不只是个标题。 |
| **便携式公司模板** | 导出/导入组织、Agent 和技能，支持敏感信息清洗和冲突处理。 |
| **真正的多公司隔离** | 每个实体都以公司为作用域，一次部署即可运行多家公司，数据和审计轨迹完全独立。 |

## Paperclip “不是”什么

|                              |                                                                                                                      |
| ---------------------------- | -------------------------------------------------------------------------------------------------------------------- |
| **不是聊天机器人** | Agent 有工作职责，而不只是聊天窗口。 |
| **不是 Agent 框架** | 我们不教你如何构建 Agent。我们教你如何运行由它们组成的公司。 |
| **不是工作流构建器** | 没有拖拽式流水线。Paperclip 建模的是公司 —— 拥有架构、目标、预算和治理。 |
| **不是提示词管理器** | Agent 自带提示词、模型和运行时。Paperclip 管理它们工作的组织环境。 |
| **不是单 Agent 工具** | 这是为团队设计的。如果你只有一个 Agent，你可能不需要 Paperclip。如果你有 20 个 —— 你绝对需要。 |
| **不是代码审查工具** | Paperclip 编排的是工作，而非 Pull Request。请使用你现有的审查流程。 |

## 快速开始

开源。自托管。无需 Paperclip 账号。

```bash
npx paperclipai onboard --yes
```

或者手动安装：

```bash
git clone https://github.com/paperclipai/paperclip.git
cd paperclip
pnpm install
pnpm dev
```

这将启动 API 服务器于 `http://localhost:3100`。系统会自动创建一个嵌入式 PostgreSQL 数据库 —— 无需额外配置。

> **要求：** Node.js 20+, pnpm 9.15+

## 常见问题 (FAQ)

**典型的部署方案是怎样的？**
在本地，单个 Node.js 进程管理嵌入式 Postgres 和本地文件存储。对于生产环境，将其指向你自己的 Postgres 并根据需要部署。配置项目、Agent 和目标 —— 其余的交给 Agent 处理。

如果你是独立开发者，可以使用 Tailscale 随时随地访问 Paperclip。之后可以根据需要部署到 Vercel 等平台。

**我可以运行多家公司吗？**
可以。单次部署可以运行无限数量的公司，且具备完全的数据隔离。

**Paperclip 与 OpenClaw 或 Claude Code 有什么不同？**
Paperclip **使用** 这些 Agent。它将它们编排成一家公司 —— 具备组织架构、预算、目标、治理和问责机制。

**为什么我应该使用 Paperclip，而不是直接让 OpenClaw 对接 Asana 或 Trello？**
Agent 编排在任务签出协调、会话维护、成本监控和治理建立方面有其独特性 —— Paperclip 为你处理了这些复杂细节。

（“引入你自己的工单系统”已在路线图中）

**Agent 是持续运行的吗？**
默认情况下，Agent 根据预设的心跳和基于事件的触发器（任务分配、@-提到）运行。你也可以接入像 OpenClaw 这样持续运行的 Agent。你负责提供 Agent，Paperclip 负责协调。

## 开发指令

```bash
pnpm dev              # 完整开发模式 (API + UI, 监听模式)
pnpm dev:once         # 完整开发模式，不监听文件更改
pnpm dev:server       # 仅服务器
pnpm build            # 构建全部
pnpm typecheck        # 类型检查
pnpm test:run         # 运行测试
pnpm db:generate      # 生成数据库迁移
pnpm db:migrate       # 执行迁移
```

详见 [doc/DEVELOPING.md](doc/DEVELOPING.md) 完整开发指南。
