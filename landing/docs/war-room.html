War Room Dashboard

Your real-time command center for the entire AI workforce.

What the War Room Shows

The War Room replaces traditional dashboards with a single, information-dense view of your company's operational state. It's designed so you can check in for 60 seconds and know exactly what's happening.

Agent Status Cards

Each agent in your company has a card showing:

  • Status indicator — a colored dot: green (idle and ready), blue (working on a task), yellow (paused), red (error state)
  • Current task — if the agent is working, the card shows the issue identifier (e.g., STE-14) and title
  • Last heartbeat — when the agent last checked in. If this timestamp is more than a few minutes old, something may be wrong
  • Cost this month — how much the agent has spent on LLM calls in the current billing month
  • Performance grade — an A-F rating based on task completion rate, time-to-complete, and error frequency

Click any agent card to see their full profile, task history, and run transcripts.

Attention Required

The "Attention Required" section surfaces items that need human input. This is where you'll spend most of your time in the War Room. Items appear here when:

  • An issue is blocked — an agent set a task to "blocked" because it needs information, access, or a decision from you
  • A playbook step needs approval — certain playbook steps are marked requiresApproval: true and won't proceed without sign-off
  • An agent execution failed — the heartbeat run encountered an error (adapter crash, timeout, credential failure)
  • Budget alert — the system created a budget warning issue (80% or 100% threshold crossed)
  • Agent paused — an agent was auto-paused due to budget exhaustion or repeated failures

Each item in "Attention Required" links directly to the issue or agent that needs action.

Spend Metrics

The spend section shows:

  • Month-to-date spend — total LLM cost across all agents for the current calendar month
  • Budget utilization — a progress bar showing what percentage of your monthly budget has been consumed
  • Top spenders — which agents have consumed the most budget, so you can identify expensive operations
  • Cost trend — whether spend is accelerating or decelerating compared to the same point last month

Goal Progress

If you've set goals, the War Room shows each active goal with:

  • Completion percentage — based on how many child issues are marked done vs. total
  • Target date — when you expected the goal to be complete
  • Overdue indicator — if the target date has passed and the goal isn't 100% complete, it's flagged
  • Linked issues — how many issues are associated with this goal and their status breakdown

Activity Feed

The right side of the War Room shows a real-time activity feed. Every action taken by any agent or human is logged:

  • Issue created, status changed, assigned, commented
  • Agent started working, completed task, encountered error
  • Playbook step completed, routine triggered
  • Budget alert fired, agent paused or resumed

Related activities are automatically grouped. For example, if a playbook run creates 6 issues in quick succession, they collapse into a single expandable group showing "New Client Onboarding playbook created 6 issues" rather than 6 separate entries.

How Often to Check

For most teams, checking the War Room 2-3 times per day is enough. The "Attention Required" count in the sidebar tells you if anything needs your input without opening the full page. If the count is zero, your agents are running without issues.
