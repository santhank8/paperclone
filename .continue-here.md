## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Editing patch files directly | Broke index hash, caused `ERR_PNPM_INVALID_PATCH` on CI | blocking | Always edit patched file in node_modules, regenerate patch via diff |
| Wrong config file | Overwrote live litellm config with stale homelab copy | blocking | Always use `/opt/litellm/config.yaml`, never the homelab data volume copy |
| Assuming lockfile is CI-regenerated | CI uses `--frozen-lockfile` — patch hash changes require lockfile refresh | advisory | Use `chore/refresh-lockfile` branch name to bypass policy check |
