## Paperclip 是什么？

# 零人力公司的开源编排平台

**如果 OpenClaw 是一名_员工_，那么 Paperclip 就是一家_公司_**

Paperclip 是一个 Node.js 服务端和 React UI，用于编排一支 AI 智能体团队来运营业务。接入你自己的智能体，分配目标，在一个仪表盘中跟踪智能体的工作和费用。

它看起来像一个任务管理器——但底层具备组织架构图、预算、治理、目标对齐和智能体协调能力。

**管理业务目标，而不是 Pull Request。**

|        | 步骤            | 示例                                                            |
| ------ | --------------- | --------------------------------------------------------------- |
| **01** | 定义目标        | _"打造排名第一的 AI 笔记应用，达到 100 万美元 MRR。"_             |
| **02** | 组建团队        | CEO、CTO、工程师、设计师、市场人员——任何机器人，任何提供商。       |
| **03** | 审批并运行      | 审查策略。设定预算。点击开始。从仪表盘监控。                      |

<br/>

> **即将推出：Clipmart** — 一键下载并运行完整的公司。浏览预构建的公司模板——完整的组织架构、智能体配置和技能——几秒钟内导入到你的 Paperclip 实例中。

## 适用场景

- ✅ 你想构建**自主运行的 AI 公司**
- ✅ 你需要**协调多种不同的智能体**（OpenClaw、Codex、Claude、Cursor）朝一个共同目标努力
- ✅ 你同时打开了 **20 个 Claude Code 终端**，已经无法追踪每个在做什么
- ✅ 你希望智能体**全天候自主运行**，同时仍能审计工作并在需要时介入
- ✅ 你想**监控费用**并执行预算
- ✅ 你需要一个**像任务管理器一样**的智能体管理流程
- ✅ 你想**用手机**管理你的自主业务

## Paperclip 解决的问题

| 没有 Paperclip                                                                                                    | 有了 Paperclip                                                                                                         |
| ----------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| ❌ 你打开了 20 个 Claude Code 标签页，无法追踪哪个在做什么。重启后一切丢失。                                        | ✅ 任务基于工单，对话有线索，会话跨重启持久化。                                                                         |
| ❌ 你手动从多处收集上下文来提醒机器人你实际在做什么。                                                                | ✅ 上下文从任务向上流经项目和公司目标——你的智能体始终知道做什么以及为什么。                                               |
| ❌ 智能体配置文件夹杂乱无章，你在重新发明任务管理、通信和智能体间的协调。                                            | ✅ Paperclip 开箱即用地提供组织架构、工单、委派和治理——让你运营公司，而不是管理一堆脚本。                                  |
| ❌ 失控的循环浪费了数百美元的 Token，在你发现之前就已用完配额。                                                      | ✅ 费用追踪展示 Token 预算，智能体超支时自动限流。管理层通过预算来确定优先级。                                            |
| ❌ 你有定期任务（客户支持、社交媒体、报告），每次都要记得手动启动。                                                   | ✅ 心跳机制按计划处理常规工作。管理层进行监督。                                                                          |
| ❌ 你有个想法，需要找到仓库，启动 Claude Code，保持标签页打开，一直盯着。                                            | ✅ 在 Paperclip 中添加任务。你的编码智能体持续工作直到完成。管理层审查他们的工作。                                        |

## Paperclip 的独特之处

Paperclip 正确处理了复杂的编排细节。

|                                   |                                                                                                               |
| --------------------------------- | ------------------------------------------------------------------------------------------------------------- |
| **原子化执行。**                   | 任务签出和预算执行是原子的，不会出现重复工作和失控开支。                                                        |
| **持久化智能体状态。**              | 智能体跨心跳恢复相同的任务上下文，而不是从头开始。                                                              |
| **运行时技能注入。**               | 智能体可以在运行时学习 Paperclip 工作流和项目上下文，无需重新训练。                                              |
| **可回滚的治理。**                 | 审批门控强制执行，配置变更有版本记录，错误的变更可以安全回滚。                                                    |
| **目标感知执行。**                 | 任务携带完整的目标祖先链，智能体始终能看到"为什么"，而不仅是标题。                                                |
| **可移植的公司模板。**              | 导出/导入组织、智能体和技能，具有密钥清洗和冲突处理能力。                                                        |
| **真正的多公司隔离。**              | 每个实体都有公司作用域，一套部署可以运行多家公司，拥有独立的数据和审计日志。                                      |

## Paperclip 不是什么

|                              |                                                                                                                      |
| ---------------------------- | -------------------------------------------------------------------------------------------------------------------- |
| **不是聊天机器人。**          | 智能体有工作岗位，而不是聊天窗口。                                                                                    |
| **不是智能体框架。**          | 我们不告诉你如何构建智能体。我们告诉你如何运营一家由它们组成的公司。                                                    |
| **不是工作流构建器。**        | 没有拖拽式管道。Paperclip 建模的是公司——有组织架构、目标、预算和治理。                                                  |
| **不是提示词管理器。**        | 智能体自带提示词、模型和运行时。Paperclip 管理它们工作的组织。                                                          |
| **不是单智能体工具。**        | 这是为团队打造的。如果你只有一个智能体，你可能不需要 Paperclip。如果你有二十个——你绝对需要。                              |
| **不是代码审查工具。**        | Paperclip 编排工作，而不是 Pull Request。自带你的审查流程。                                                              |

## 快速开始

开源。自托管。无需 Paperclip 账户。

```bash
npx paperclipai onboard --yes
```

或手动安装：

```bash
git clone https://github.com/paperclipai/paperclip.git
cd paperclip
pnpm install
pnpm dev
```

这将在 `http://localhost:3100` 启动 API 服务。会自动创建一个嵌入式 PostgreSQL 数据库——无需额外配置。

> **环境要求：** Node.js 20+, pnpm 9.15+

## 常见问题

**典型的部署是什么样的？**
本地环境下，单个 Node.js 进程管理嵌入式 Postgres 和本地文件存储。生产环境下，指向你自己的 Postgres 并按需部署。配置项目、智能体和目标——智能体会处理其余的一切。

如果你是独立创业者，可以使用 Tailscale 随时随地访问 Paperclip。之后需要时可以部署到 Vercel 等平台。

**可以运行多家公司吗？**
可以。单次部署可以运行无限数量的公司，数据完全隔离。

**Paperclip 和 OpenClaw 或 Claude Code 等智能体有什么区别？**
Paperclip _使用_这些智能体。它将它们编排成一家公司——拥有组织架构、预算、目标、治理和问责制。

**为什么要用 Paperclip 而不是直接让 OpenClaw 对接 Asana 或 Trello？**
智能体编排在协调谁签出了工作、如何维持会话、监控费用、建立治理等方面有许多微妙之处——Paperclip 为你处理这一切。

（自带工单系统功能已在路线图中）

**智能体会持续运行吗？**
默认情况下，智能体按计划心跳和基于事件的触发器（任务分配、@提及）运行。你也可以接入 OpenClaw 等持续运行的智能体。你提供智能体，Paperclip 负责协调。

## 开发

```bash
pnpm dev              # 完整开发环境（API + UI，监听模式）
pnpm dev:once         # 完整开发环境，不监听文件变更
pnpm dev:server       # 仅服务端
pnpm build            # 构建全部
pnpm typecheck        # 类型检查
pnpm test:run         # 运行测试
pnpm db:generate      # 生成数据库迁移
pnpm db:migrate       # 应用迁移
```

完整开发指南请参阅 [doc/DEVELOPING.md](doc/DEVELOPING.md)。
